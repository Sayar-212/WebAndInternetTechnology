Q1(I)________ operator executes a shell command from the Hive shell.1M
✓ Answer

The ! (bang/exclamation) operator executes a shell command from the Hive shell.

Example: !ls /user/hive/warehouse; — runs the ls command directly from within the Hive CLI without leaving the session.

Q1(II)The data being captured can be any form or structure. Which characteristic of Big Data are we talking about?1M
✓ Answer

Variety. It refers to the different types and forms of data — structured (tables, databases), semi-structured (JSON, XML), and unstructured (images, videos, social media posts, logs).

Variety is one of the 5 V's of Big Data: Volume, Velocity, Variety, Veracity, and Value.

Q1(III)Which NoSQL database stores data in nodes and edges?1M
✓ Answer

Graph database (e.g., Neo4j, Amazon Neptune, JanusGraph).

Data is stored as nodes (entities) and edges (relationships between entities). Properties can be attached to both nodes and edges. Ideal for social networks, recommendation engines, and fraud detection.

Q1(IV)Which term is used to denote the small subsets of a large file created by HDFS?1M
✓ Answer

Blocks. HDFS splits large files into fixed-size blocks (default 128 MB in Hadoop 2.x, was 64 MB in Hadoop 1.x). Each block is stored independently across different DataNodes for parallel processing and fault tolerance through replication (default replication factor = 3).

Q1(V)YARN stands for: ________1M
✓ Answer

Yet Another Resource Negotiator.

YARN is the resource management layer of Hadoop 2.x. It separates resource management from job scheduling/monitoring, allowing multiple processing engines (MapReduce, Spark, Tez) to run on the same cluster.

Q1(VI)Is HBase a NoSQL database?1M
✓ Answer

Yes. Apache HBase is a column-family NoSQL database built on top of HDFS. It is modelled after Google's Bigtable and provides real-time read/write random access to large datasets. It is schema-less, distributed, and horizontally scalable.

Q1(VII)________ step is performed by a data scientist after acquiring the data.1M
✓ Answer

Data Preprocessing / Data Cleaning.

After data acquisition, the data scientist cleans and preprocesses the raw data — handling missing values, removing duplicates, transforming formats, normalising data — before it can be used for analysis or model building.

Q1(VIII)NoSQL databases are used mainly for handling large volumes of ________ data.1M
✓ Answer

Unstructured (and semi-structured) data.

Unlike relational databases (RDBMS) which require predefined schemas, NoSQL databases can handle unstructured data such as JSON documents, key-value pairs, wide-column stores, and graph data — making them ideal for Big Data applications.

Q1(IX)What is the default file size of an HDFS data block?1M
✓ Answer

128 MB (in Hadoop 2.x and later). In Hadoop 1.x, the default block size was 64 MB.

The block size can be configured using the dfs.blocksize parameter in hdfs-site.xml. Larger blocks reduce NameNode memory usage but may waste space for small files.

Q1(X)A ________ node acts as the Slave and is responsible for executing a Task assigned to it by the JobTracker.1M
✓ Answer

TaskTracker node.

In Hadoop 1.x (MRv1), the TaskTracker runs on each slave node, accepts Map and Reduce tasks from the JobTracker, executes them in separate JVM processes, and reports progress/heartbeat back to the JobTracker. In Hadoop 2.x (YARN), this role is handled by the NodeManager.

Q1(XI)Most NoSQL databases support automatic ________ meaning that you get high availability and disaster recovery.1M
✓ Answer

Replication.

NoSQL databases automatically replicate data across multiple nodes/data centres, ensuring high availability (data is accessible even if some nodes fail) and disaster recovery (data is not lost due to hardware failures). Examples: Cassandra uses configurable replication factor, MongoDB has replica sets.

Q1(XII)The ________ and the EditLog are central data structures of HDFS.1M
✓ Answer

FsImage (File System Image).

FsImage: A snapshot of the entire HDFS file system metadata (directory structure, file-to-block mappings, permissions) at a point in time. EditLog: A transaction log that records every change made to the file system metadata after the last FsImage checkpoint. During startup, NameNode loads FsImage + replays EditLog to reconstruct current state.

Q3What is the role of Name Node in an HDFS cluster?5 Marks
✓ Answer

The NameNode is the master server in the HDFS architecture that manages the file system namespace and regulates client access to files.

Key Roles of NameNode

  • Metadata Management: Stores the file system tree and metadata — file names, permissions, block locations, replication factors — in memory (FsImage + EditLog).
  • Block Mapping: Maintains the mapping of files to their constituent blocks and which DataNodes store each block replica.
  • Client Request Handling: Receives file open/close/rename requests from clients and determines which DataNodes to read from or write to.
  • Heartbeat Monitoring: Receives periodic heartbeats and block reports from DataNodes to confirm they are alive and track block locations.
  • Replication Management: Ensures each block has the configured number of replicas; triggers re-replication when a DataNode goes down.
  • Namespace Operations: Handles directory creation, deletion, renaming, and permission management.

Q4Define Business Intelligence (BI).5 Marks
✓ Answer

Business Intelligence (BI) refers to the strategies, technologies, and tools used to collect, integrate, analyse, and present business data to support better decision-making.

Key Components of BI

  • Data Warehousing: Central repository that stores historical data from multiple sources for analysis.
  • ETL (Extract, Transform, Load): Process of extracting data from source systems, transforming it into a consistent format, and loading it into the warehouse.
  • OLAP (Online Analytical Processing): Enables multi-dimensional analysis — slicing, dicing, drill-down, roll-up on data cubes.
  • Reporting & Dashboards: Visual summaries of KPIs and metrics using tools like Tableau, Power BI, Looker.
  • Data Mining: Discovering hidden patterns and insights using statistical and ML techniques.

Benefits: Faster and data-driven decisions, identify trends and anomalies, competitive advantage, improved operational efficiency, and better customer insights.

BI vs Big Data Analytics: Traditional BI works on structured data in warehouses; Big Data analytics extends this to handle massive volumes of structured + unstructured data using tools like Hadoop, Spark.

Q5Explain the CAP theorem.5 Marks
✓ Answer

The CAP Theorem (Brewer's Theorem) states that a distributed system can guarantee at most two out of three properties simultaneously:

PropertyMeaning
C — ConsistencyEvery read receives the most recent write (all nodes see the same data at the same time)
A — AvailabilityEvery request gets a response (success/failure), even if some nodes are down
P — Partition ToleranceSystem continues to operate even when network partitions (communication failures between nodes) occur

CAP Trade-offs in NoSQL Databases

TypeGuaranteesExamples
CP (Consistency + Partition Tolerance)Consistent reads, may reject writes during partitionHBase, MongoDB, Redis
AP (Availability + Partition Tolerance)Always available, may serve stale dataCassandra, DynamoDB, CouchDB
CA (Consistency + Availability)Not practical in distributed systems (no partition tolerance)Traditional RDBMS (single-node)

Key Insight: Since network partitions are unavoidable in distributed systems, real-world systems must choose between CP or AP. Many NoSQL databases offer tunable consistency (e.g., Cassandra allows configuring consistency level per query).

Q7(a) Explain application of Big Data in Healthcare. (b) How do you deploy a Big Data solution? [6+9]15 Marks
✓ Answer

(a) Applications of Big Data in Healthcare [6 marks]

  • Electronic Health Records (EHR): Centralised patient records accessible across hospitals — medical history, prescriptions, lab results, imaging data. Enables faster diagnosis and reduces duplicate tests.
  • Predictive Analytics: ML models predict disease outbreaks (e.g., COVID-19 spread modelling), patient readmission risk, and treatment outcomes based on historical data.
  • Drug Discovery: Analysing genomic data and clinical trials to identify drug candidates faster. Reduces drug development time from 10+ years to a few years.
  • Medical Imaging: Deep learning on X-rays, MRIs, CT scans to detect tumours, fractures, and anomalies with high accuracy (often surpassing human radiologists).
  • Wearable Devices & IoT: Fitness trackers and smartwatches generate continuous health data (heart rate, blood oxygen, sleep patterns) — enabling remote patient monitoring.
  • Fraud Detection: Identifying fraudulent insurance claims by analysing billing patterns and anomalies.
  • Personalised Medicine: Tailoring treatment plans based on a patient's genetic profile, lifestyle, and historical response to medications.

(b) How to Deploy a Big Data Solution [9 marks]

  1. Identify Business Problem: Define the specific problem to solve — what questions need answering? What decisions will the data support?
  2. Data Source Identification: Identify all data sources — databases, APIs, IoT sensors, social media, logs. Assess data volume, velocity, variety.
  3. Choose Technology Stack:
    • Storage: HDFS, Amazon S3, Azure Blob Storage
    • Processing: MapReduce, Apache Spark, Apache Flink
    • Database: HBase, Cassandra, MongoDB
    • Query Engine: Hive, Pig, Presto
    • Orchestration: Apache Airflow, Oozie
  4. Data Ingestion: Set up pipelines to collect data — batch (Sqoop, Flume) or real-time (Kafka, Spark Streaming).
  5. Data Processing & ETL: Clean, transform, and enrich raw data. Handle missing values, duplicates, format conversion.
  6. Data Storage: Store processed data in appropriate format — Parquet/ORC for analytics, HBase for random access.
  7. Analysis & Modelling: Apply analytics — descriptive, diagnostic, predictive, or prescriptive. Build ML models if needed.
  8. Visualisation & Reporting: Deploy dashboards using Tableau, Grafana, or custom web apps for stakeholders.
  9. Monitoring & Scaling: Monitor cluster health (Ambari, Cloudera Manager), tune performance, scale horizontally by adding nodes as data grows.

Q8(a) What is NoSQL database? (b) Why is it used? (c) Advantages of NoSQL? (d) NoSQL vs Relational databases. [2+3+5+5]15 Marks
✓ Answer

(a) What is NoSQL Database? [2 marks]

NoSQL (Not Only SQL) is a category of database management systems that do not follow the traditional relational (table-based) model. They are designed to handle large volumes of unstructured and semi-structured data with flexible schemas, horizontal scalability, and high performance.

(b) Why is NoSQL Used? [3 marks]

  • Big Data: Traditional RDBMS cannot efficiently handle petabytes of data — NoSQL databases scale horizontally across commodity hardware.
  • Flexible Schema: Data models evolve rapidly in modern applications (social media, IoT). NoSQL allows schema-less or dynamic schema designs.
  • High Velocity: Real-time applications (gaming, social media feeds, IoT) need low-latency reads/writes — NoSQL databases like Cassandra and Redis excel at this.
  • Variety of Data: JSON documents, key-value pairs, graphs, wide-column data — NoSQL supports diverse data models.
  • Cloud & Distributed: NoSQL databases are designed for distributed environments and cloud deployment.

(c) Advantages of NoSQL [5 marks]

  • Horizontal Scalability: Add more nodes to the cluster instead of upgrading a single server. Scale out, not up.
  • Flexible Data Model: No rigid schema — documents, key-value, columnar, or graph models. Adapts to changing requirements.
  • High Performance: Optimised for specific access patterns — fast reads/writes for key-value stores, complex queries for graph databases.
  • High Availability: Built-in replication and automatic failover ensure data is always accessible.
  • Cost-Effective: Runs on commodity hardware; open-source options (MongoDB, Cassandra, HBase) reduce licensing costs.
  • Developer Friendly: JSON-like documents map naturally to programming language objects; easier to develop with agile methodologies.

(d) NoSQL vs Relational Databases [5 marks]

Q9(a) Important features of Hadoop. (b) Different modes of Hadoop. (c) Core components of Hadoop. [5+5+5]15 Marks
✓ Answer

(a) Important Features of Hadoop [5 marks]

  • Open Source: Apache-licensed, free to use. Large community support and ecosystem.
  • Distributed Storage & Processing: Data is stored and processed across multiple nodes in parallel.
  • Scalability: Easily scales from a single server to thousands of commodity machines.
  • Fault Tolerance: Data is replicated across multiple nodes (default 3). If a node fails, data is automatically recovered from replicas.
  • Data Locality: Processing moves to where the data resides rather than moving data to processing — reduces network bottleneck.
  • Cost-Effective: Uses commodity hardware instead of expensive specialised servers.
  • Flexible Data Handling: Can process structured, semi-structured, and unstructured data.
  • High Throughput: Optimised for large sequential reads/writes — ideal for batch processing of massive datasets.

(b) Different Modes in Which Hadoop Runs [5 marks]

  2. YARN (Yet Another Resource Negotiator): Resource management layer. Manages cluster resources and schedules jobs. Components: ResourceManager (master — allocates resources), NodeManager (slave — manages containers on each node), ApplicationMaster (per-application — negotiates resources).

Q10(a) What do you mean by schema-less databases? (b) Explain types of NoSQL databases. [3+12]15 Marks
✓ Answer

(a) Schema-less Databases [3 marks]

A schema-less database (or dynamic schema database) does not require a predefined structure before data can be inserted. Each record/document can have different fields and data types without altering a global table schema.

  • No need to define columns, data types, or relationships in advance.
  • New fields can be added to individual documents without affecting existing data.
  • Ideal for agile development where data models evolve frequently.
// MongoDB example — two documents in the same collection with different fields:
{ "name": "Alice", "age": 25, "email": "alice@mail.com" }
{ "name": "Bob", "phone": "9876543210", "address": "Kolkata" }
// No error — both are valid despite different structures.

(b) Types of NoSQL Databases [12 marks]

1. Key-Value Stores

  • Simplest NoSQL model — data stored as key-value pairs (like a dictionary/hash map).
  • Ultra-fast lookups by key; value can be any data type (string, JSON, binary).
  • Examples: Redis, Amazon DynamoDB, Riak, Memcached.
  • Use Cases: Session management, caching, user preferences, shopping carts.
Key: "user:1001"    → Value: {"name": "Alice", "role": "admin"}
Key: "session:xyz"  → Value: {"token": "abc123", "expiry": 3600}

2. Document Stores

  • Data stored as JSON/BSON/XML documents. Each document is self-contained with nested structures.
  • Supports indexing and querying on document fields.
  • Examples: MongoDB, CouchDB, Amazon DocumentDB.
  • Use Cases: Content management, user profiles, catalogues, real-time analytics.

3. Column-Family (Wide-Column) Stores

  • Data organised by columns rather than rows. Column families group related columns together.
  • Efficient for reading/writing large datasets where only specific columns are needed.
  • Examples: Apache Cassandra, HBase, Google Bigtable.
  • Use Cases: Time-series data, IoT sensor data, event logging, analytics on specific columns.

4. Graph Databases

  • Data stored as nodes (entities), edges (relationships), and properties. Optimised for traversing relationships.
  • Excellent for highly connected data where joins would be expensive in RDBMS.
  • Examples: Neo4j, Amazon Neptune, JanusGraph, ArangoDB.
  • Use Cases: Social networks, recommendation engines, fraud detection, knowledge graphs.

Q11(a) Job Scheduling in MapReduce — Fair Scheduler and Capacity Scheduler. (b) FileInputFormat class. (c) Types of MapReduce applications. [10+3+2]15 Marks
✓ Answer

(a) Job Scheduling in MapReduce [10 marks]

Job scheduling determines how multiple MapReduce jobs share cluster resources. Hadoop provides pluggable schedulers:

(i) The Fair Scheduler

  • Developed by Facebook. Aims to give every user/pool a fair share of cluster resources over time.
  • Jobs are organised into pools (queues). Each pool gets an equal share of resources by default.
  • If a pool's jobs use less than its fair share, surplus resources are redistributed to other pools.
  • When a pool needs more resources, it gradually reclaims them as tasks from other pools finish.
  • Preemption: If a pool has been starved for too long, it can preempt (kill) tasks from over-allocated pools to reclaim its fair share.
  • Supports priorities within pools and minimum share guarantees.

(ii) The Capacity Scheduler

  • Developed by Yahoo. Designed for multi-tenant clusters where organisations share a cluster.
  • Resources are divided into queues, each with a guaranteed minimum capacity (percentage of cluster).
  • Queues can use excess capacity from idle queues (elastic scheduling), but must return resources when needed.
  • Supports hierarchical queues (parent-child), ACLs (access control), and per-queue resource limits.
  • FIFO ordering within each queue by default, but supports priority ordering.

(b) FileInputFormat Class [3 marks]

FileInputFormat is the base class for all file-based InputFormats in Hadoop MapReduce. It handles:

  • Input Validation: Checks that the input paths exist and are accessible.
  • Input Splitting: Splits input files into InputSplits — each split is processed by one Map task. Default split size = HDFS block size.
  • Record Reading: Subclasses (TextInputFormat, KeyValueTextInputFormat, SequenceFileInputFormat) define how to read records from each split.
// Setting input format in driver code:
FileInputFormat.addInputPath(job, new Path("/input/data"));
job.setInputFormatClass(TextInputFormat.class); // default

(c) Types of MapReduce Applications [2 marks]

  • Batch Processing: Processing large datasets in one go — log analysis, ETL, data transformation.
  • Sorting & Searching: TeraSort, distributed grep, indexing.
  • Data Mining: Pattern discovery, clustering, classification on large datasets.
  • Machine Learning: Iterative algorithms (k-means, PageRank) over distributed data — though Spark is preferred for this now.
