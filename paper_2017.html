Multiple Choice Questions 10 × 1 = 10

Q1(i)What type of protocol is HTTP?1M
a) State aware
b) Stateful
c) Stateless
d) Connection oriented
✓ Answer: (c) Stateless

HTTP is stateless — each request is independent; the server retains no memory of previous requests. Sessions must be managed externally via cookies, tokens, or server-side session storage.

Q1(ii)Which is the intra-domain routing protocol?1M
a) RIP
b) OSPF
c) BGP
d) None
✓ Answer: (b) OSPF (also RIP qualifies — best answer is OSPF)

Both RIP and OSPF are intra-domain (IGP — Interior Gateway Protocols) used within an Autonomous System. BGP is an inter-domain (EGP) protocol. OSPF is the better/more modern choice. RIP is also an IGP but limited to 15 hops.

Q1(iii)Which is NOT an IPv6 address type?1M
a) Anycast
b) Multicast
c) Unicast
d) Broadcast
✓ Answer: (d) Broadcast

IPv6 supports Unicast (one-to-one), Multicast (one-to-many group), and Anycast (one-to-nearest). Broadcast does not exist in IPv6 — it is replaced by multicast addresses for efficiency.

Q1(iv)PERL stands for1M
a) Practical Extraction and Reading Language
b) Practical Extraction and Report Language
c) Practical Example and Report Language
d) Practical Expert on Report Language
✓ Answer: (b) Practical Extraction and Report Language

Perl was originally created by Larry Wall in 1987 as a Practical Extraction and Report Language for text processing and generating reports. It is now a general-purpose scripting language famous for its powerful regular expression support.

Q1(v)RIP is based on1M
a) Link State Routing
b) Distance Vector Routing
c) Path Vector Routing
d) None
✓ Answer: (b) Distance Vector Routing

RIP (Routing Information Protocol) uses the Bellman-Ford (distance vector) algorithm. Each router shares its entire routing table with direct neighbours. Metric = hop count (max 15). OSPF uses link state (Dijkstra). BGP uses path vector.

Q1(vi)Which statement about XML is true?1M
a) All the statements are true
b) All XML elements must have a closing tag
c) All XML elements must be in lower case
d) All XML documents must have a DTD
✓ Answer: (b) All XML elements must have a closing tag

Option (b) is the most accurate standalone truth. (c) is FALSE — XML is case-sensitive but tags can be any case. (d) is FALSE — a DTD is optional (document can be well-formed without one). So the correct single true statement is (b). Note: option (a) "all are true" is incorrect because (c) and (d) are false.

Q1(vii)Position of SSL in TCP/IP protocol suite1M
a) Transport and internet layer
b) Data link and physical layer
c) Application and transport layer
d) None
✓ Answer: (c) Between Application and Transport layer

SSL/TLS inserts itself between the Application layer and the Transport (TCP) layer. Applications pass data to SSL, which encrypts it before handing it to TCP. This is why HTTPS = HTTP + SSL/TLS.

Q1(viii)Which HTML attribute defines inline styles?1M
a) Class
b) Styles
c) Style
d) Font
✓ Answer: (c) style

The style attribute applies CSS directly on an HTML element: <p style="color:red; font-weight:bold">. There is no styles or font attribute for this purpose.

Q1(ix)Cookie is created and sent by1M
a) Web browser
b) Web server
c) Application server
d) Strut platform
✓ Answer: (b) Web server

The web server creates the cookie and sends it to the browser via the Set-Cookie HTTP response header. The browser stores it and returns it in the Cookie request header on subsequent requests to the same domain.

Q1(x)Maximum size of data portion of IP datagram is1M
a) 65555B
b) 65535B
c) 65515B
d) None
✓ Answer: (c) 65515B

The Total Length field in the IPv4 header is 16 bits → maximum = 65535 bytes. The minimum IP header = 20 bytes. Maximum IP options = 40 bytes → max header = 60 bytes. Max data = 65535 − 20 = 65515 bytes (with minimum header). Some questions accept 65515 or 65535-20=65515.

The Internet is a global, publicly accessible network of interconnected computers and networks that use the TCP/IP protocol suite to communicate. It provides services like WWW, email, FTP, VoIP, and streaming. No single organisation owns it — it is a decentralised network of networks.

XML Schema (XSD) (2 marks)

An XML Schema (XSD — XML Schema Definition) is a W3C-recommended language for defining the structure, content model, and data types of XML documents. It replaces DTD with much richer functionality.

<!-- Simple XSD example -->
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">

    <xs:complexType>

        <xs:element name="name" type="xs:string"/>
        <xs:element name="age"  type="xs:integer"/>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>

Q7a) Firewall — what + needs   b) Types of firewall   c) Packet filtering   d) Proxy purpose   e) NAT functions4+2+2+3+4 = 15
✓ Answer

a) Firewall — Definition and Basic Needs (4 marks)

A firewall is a network security system (hardware/software) that monitors and controls incoming/outgoing traffic based on predetermined security rules, creating a barrier between trusted internal networks and untrusted external networks.

Basic needs:

  • Protect internal network from external attacks (hackers, malware).
  • Control access — block unauthorized access to internal resources.
  • Log and monitor all network traffic for audit/forensics.
  • Prevent data exfiltration — stop sensitive data leaving the network.
  • Guard against DoS/DDoS attacks.
  • Enforce organisational security policy.

b) Types of Firewall (2 marks)

  • Packet Filtering: Network layer, checks IP/port headers.
  • Stateful Inspection: Tracks connection state; more intelligent than packet filtering.
  • Proxy / Application Firewall: Layer 7, deep content inspection.
  • Next-Generation Firewall (NGFW): Combines DPI, IPS, application awareness, SSL inspection.

c) Packet Filtering (2 marks)

Examines each packet independently based on: source/destination IP, protocol (TCP/UDP/ICMP), port numbers, and direction. Uses ACL rules — processed top to bottom; first match wins. Stateless — no connection tracking. Fast but cannot inspect payload or detect fragmentation attacks.

d) Purpose of Proxy (3 marks)

  • Intermediary: Client connects to proxy; proxy connects to server — neither side communicates directly.
  • Anonymity: Hides client's real IP from the server.
  • Content filtering: Blocks malicious URLs, scans content for viruses.
  • Caching: Stores frequently requested pages — reduces latency and bandwidth.
  • Access control: Restricts which sites users can access.

e) Functions of NAT (4 marks)

  • Address translation: Converts private IPs (192.168.x.x) to public IP for internet communication.
  • IP address conservation: Many devices share one public IP (PAT/NAT overload).
  • Security: External hosts cannot directly initiate connections to internal hosts — hides internal topology.
  • Types: Static (1:1), Dynamic (pool), PAT (many:1 using ports).
  • Translation table: Maintains mapping of private:port ↔ public:port for all active connections.

c) OSPF Link Types (5 marks)

  • Point-to-Point: Direct link between exactly two routers. No DR/BDR election needed. E.g., serial WAN links.
  • Broadcast (Transient): Multi-access network with many routers (e.g., Ethernet LAN). DR (Designated Router) and BDR elected to reduce LSA flooding.
  • Non-Broadcast Multi-Access (NBMA): Like Frame Relay — multiple routers but no broadcast capability. DR/BDR still used. Manual neighbour configuration needed.
  • Point-to-Multipoint: Collection of point-to-point links. No DR election. Flexible for partial-mesh topologies.
  • Virtual Link: Logical tunnel through a non-backbone area to connect to area 0 (backbone). Used for discontiguous backbone areas.
Q9a) Class B default mask + broadcast + Class A private range   b) Dynamic vs static routing   c) GET vs POST   d) TCP vs UDP4+4+4+3 = 15
✓ Answer

a) Class B Info + Class A Private Range (4 marks)

  • Class B default mask: 255.255.0.0 (/16)
  • Class B broadcast: Host bits all 1s — e.g., for network 172.16.0.0/16 → broadcast = 172.16.255.255
  • Class A private IP range: 10.0.0.0 – 10.255.255.255 (10.0.0.0/8)

b) Dynamic vs Static Routing (4 marks)

Dynamic routing is preferred in continuously changing networks because:

  • Automatically updates routing tables when topology changes (link failure, new router).
  • No manual intervention required — routers exchange routing information via protocols (OSPF, RIP, BGP).
  • Adapts to failures in real-time — finds alternate paths automatically.
  • Scales well for large, complex networks.

Static routing requires manual update of every router's table when topology changes — impractical in dynamic environments.

d) Multicast Addressing + Transport Gateway (1+3 marks)

Multicast: One packet delivered to a group of interested receivers simultaneously. Class D addresses (224.0.0.0–239.255.255.255). Routers replicate only where needed. More efficient than broadcast (non-members ignored) and unicast (one copy each).

Transport Gateway: A device that connects two networks using different transport-layer protocols (e.g., TCP ↔ ISO TP4). It terminates the transport connection on one side, re-encapsulates the data in the protocol of the other network, and re-establishes a connection. Acts as a protocol translator at Layer 4.

Q11Short Notes (any 3): a) Proxy Firewall   b) TCP 3-way handshake   c) Java Socket   d) HTML Image Map   e) Path Vector Routing3 × 5 = 15
✓ Answer

a) Proxy Firewall

Application-layer firewall; acts as intermediary; performs deep content inspection; hides internal IPs; caches content; blocks malicious URLs. Slower than packet filtering but far more secure. Needs separate proxy per protocol (HTTP proxy, FTP proxy, SMTP proxy).

b) TCP Three-Way Handshake

Client                         Server
  │── SYN (seq=x) ─────────────►│  Wants to connect
  │◄── SYN-ACK (seq=y,ack=x+1) ─│  OK, here's my seq
  │── ACK (ack=y+1) ────────────►│  Acknowledged
  │════════ Data Transfer ════════│

Step 1: Client sends SYN with its ISN (x). Step 2: Server responds SYN-ACK with own ISN (y) and ACK=x+1. Step 3: Client sends ACK=y+1. Both sides now know each other's sequence numbers. Connection is established and bidirectional data transfer begins.

e) Path Vector Routing

Each routing entry contains the complete path (sequence of ASes) to reach the destination. Used by BGP (Border Gateway Protocol) for inter-domain routing on the internet. Loop prevention: if own AS number appears in the path, reject the route. Supports policy-based routing — can avoid certain ASes for business or political reasons. More flexible than distance vector; more scalable than link state for inter-domain use.
