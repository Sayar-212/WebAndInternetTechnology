Q1(V)How do large companies monitor and track cargo deliveries to their customers?1M
✓ Answer

Large companies track cargo using multiple technologies:

  • GPS tracking: Real-time location of delivery vehicles and containers via GPS devices — data fed to a TMS (Transportation Management System).
  • Barcode / RFID scanning: At each fulfilment touchpoint (warehouse pickup, sortation centre, delivery hub) items are scanned — status updated in the OMS.
  • EDI 856 (ASN): Suppliers send Advance Ship Notices electronically — companies know shipment details before arrival.
  • Carrier API integration: FedEx, DHL, BlueDart APIs feed tracking events directly into the company's customer portal and mobile app.
  • IoT sensors: For cold chain or high-value cargo — sensors monitor temperature, humidity, shocks and transmit data via cellular/satellite.
  • Customer-facing tracking portal: Customers receive tracking links (SMS/email) — see real-time status on map.
Q1(VI)Give an idea about 'Stored-value card'.1M
✓ Answer

A stored-value card is a payment card that has a pre-loaded monetary value stored on the card itself (on a chip) or in a linked account — not connected to a bank account or credit line. The value decreases with each purchase. Types: closed-loop (usable only at one retailer — Starbucks card, Amazon gift card, Domino's voucher) and open-loop (usable anywhere the card network is accepted — Visa prepaid cards, travel forex cards). Examples: metro cards, toll cards (FASTag), gift cards, prepaid travel cards, meal vouchers.

Q1(VII)What is the full form of VAN and what purpose is it used for?1M
✓ Answer

VAN = Value Added Network. A VAN is a third-party private network service that acts as an intermediary "electronic post office" for EDI transactions between trading partners. Purpose: provides secure store-and-forward messaging (holds EDI messages until the recipient is ready to retrieve them), protocol conversion, message tracking and audit trail, mailbox services, and transmission acknowledgements. Charges: per-transaction or per-kilocharacter fees. Examples: GEIS (now IBM), Sterling Commerce, SPSCOMMERCE, Kleinschmidt. Modern alternative: AS2 over internet (eliminates VAN fees).

Q1(VIII)Explain what is meant by 'Certificate Revocation List (CRL)'.1M
✓ Answer

A CRL (Certificate Revocation List) is a digitally signed list maintained by a Certificate Authority (CA) that contains the serial numbers of digital certificates that have been revoked before their expiry date — because the private key was compromised, the certificate was issued incorrectly, or the organisation no longer exists. When a browser establishes an HTTPS connection, it checks whether the server's certificate appears on the CA's CRL. If it does, the connection is rejected. Modern alternative to CRL: OCSP (Online Certificate Status Protocol) — real-time single-certificate status check rather than downloading the full CRL list.

Q2Explain with an example how cart reminders can be used for E-marketing. [5]5 Marks
✓ Answer

Cart abandonment is one of e-commerce's biggest revenue leaks — on average 70% of shoppers who add items to a cart leave without purchasing. Cart reminder emails are automated e-marketing messages triggered when a logged-in customer adds products to their cart but doesn't complete checkout within a defined time window (typically 1–3 hours).

How it Works

Customer adds laptop to cart → doesn't checkout for 2 hours
    ↓
Automated trigger fires (email platform: Klaviyo / Mailchimp)
    ↓
Email 1 (2 hrs later): "You left something behind! 💻"
  - Shows the abandoned laptop with image and price
  - CTA: "Complete your purchase"

Email 2 (24 hrs later, if still no purchase):
  - "Still thinking it over?"
  - May include a limited-time discount (5% off)
  - Reviews/ratings of the product for social proof

Email 3 (48 hrs later):
  - "Last chance — limited stock remaining"
  - Stronger urgency cue

Results

  • Cart reminder emails recover 5–15% of abandoned carts.
  • Average order value from cart recovery emails is typically higher than regular purchases.
  • Push notifications and WhatsApp reminders (for mobile app users) work similarly.

Real Example

A Flipkart customer adds a smartphone to cart but leaves. An hour later they receive an email: "Your OnePlus 12 is waiting! ₹54,999 — only 3 left in stock. Tap to complete your order." This combines personalisation, social proof (stock scarcity), and urgency — proven conversion drivers.

Q3Identify the different steps in EDI implementation. [5]5 Marks
✓ Answer

Implementing EDI in an organisation involves these sequential steps:

  1. Business case and needs analysis: Identify which trading partners require EDI, which document types need to be exchanged, expected transaction volumes, and ROI justification.
  2. Select EDI standard: Decide on the standard to use — ANSI X12 (if US partners), UN/EDIFACT (international), or industry-specific (ODETTE for automotive, HL7 for healthcare). Must match what partners require.
  3. Select EDI software and communication method: Choose translation software (IBM Sterling, OpenText, Cleo) and communication protocol (AS2 over HTTPS is the modern standard; VAN is the legacy option).
  4. Trading partner setup: Exchange technical details with each trading partner — ISA/GS IDs (X12), AS2 certificates and URLs, agreed message types, and functional acknowledgement requirements.
  5. Data mapping: Map internal application data fields to EDI standard segment/element positions and vice versa. This is the most time-consuming step — maps differ per partner and document type.
  6. Integration with internal systems: Connect the EDI translator to the ERP/order management system so received EDI messages automatically create records (PO received → sales order created) and outbound EDI is generated from system events.
  7. Testing: Test with each trading partner — send and receive test transactions; verify acknowledgements; confirm data maps correctly into both systems. Use ISA test indicator (T = test, P = production).
  8. Go-live and monitoring: Switch to production (ISA indicator P); monitor message status dashboards; set up alerts for failed transmissions; ensure acknowledgements arrive within agreed SLAs.
Q4Discuss the threats of E-Commerce. [5]5 Marks
✓ Answer

e-Commerce faces threats from multiple directions — technical, financial, and human:

  • Phishing attacks: Fraudulent emails and websites impersonate trusted brands (banks, Amazon, IRCTC) to steal login credentials and card details. India sees millions of phishing attempts monthly.
  • Credit card / CNP fraud: Stolen card details used for Card Not Present (CNP) online purchases. Chargeback costs the merchant even when not at fault.
  • DDoS (Distributed Denial of Service): Flooding the server with traffic from thousands of botnets — website goes offline, revenue lost. Peak shopping periods (Diwali, Big Billion Day) are prime targets.
  • SQL injection: Attackers inject malicious SQL into web form inputs to extract, modify, or delete database records — stealing customer data, order history, and payment information.
  • Cross-Site Scripting (XSS): Malicious scripts injected into web pages that execute in other users' browsers — steal session cookies and redirect users to phishing pages.
  • Man-in-the-Middle (MITM): Attacker intercepts communication between browser and server on unsecured networks — reads or alters data in transit. Mitigated by HTTPS/TLS.
  • Account takeover (ATO): Credential stuffing — using leaked username/password lists to take over customer accounts, then making fraudulent purchases.
  • Malware and ransomware: Infects e-commerce servers or employee devices — steals data or encrypts systems demanding ransom payment.
  • Fake reviews and counterfeit goods: Damage brand reputation, erode customer trust, and can result in regulatory action.

Q6Explain Customer-premises equipment (CPE). Give examples. [5]5 Marks
✓ Answer

CPE (Customer Premises Equipment) refers to any terminal equipment, network device, or hardware located at the customer's premises (home or office) that is used to connect to a service provider's network — as opposed to equipment located at the service provider's facility.

Characteristics

  • Physically located on the customer's side of the network demarcation point.
  • May be owned by the customer or provided/leased by the ISP/telecom.
  • The customer is responsible for its operation and maintenance.
  • Connects to the ISP or carrier network via the "last mile" connection.

Examples of CPE

  • Broadband router/modem: Jio Fiber ONT (Optical Network Terminal), Airtel DSL modem — connects home/office to the internet.
  • Wi-Fi router: TP-Link, Netgear, D-Link — provides wireless internet access within premises.
  • IP telephone: VoIP desk phone connected to the corporate PBX via internet.
  • Set-top box: Tata Sky DTH box, IPTV set-top box — receives TV services at customer's home.
  • POS terminal: Payment terminal at a retail shop connected to the acquirer bank via internet.
  • Corporate firewall/router: An enterprise's perimeter router/firewall — on customer premises, connecting to ISP.
  • Smart meter: Electricity meter that reports consumption data to the utility company via cellular/internet.

In e-commerce context: CPE is what customers use to access e-commerce websites — their home router and device. Poor CPE quality (slow modem, weak Wi-Fi) directly impacts the shopping experience and conversion rates.

Q7(a) E-Commerce business models by transaction type [8]   (b) Examples [4]   (c) Challenges of E-Governance in India [3]15 Marks
✓ Answer

(a) E-Commerce Business Models Based on Transaction Type [8]

1. B2C — Business to Consumer

A business sells products or services directly to individual consumers. The most common and visible form of e-commerce. The business manages product sourcing, inventory, website, payments, and delivery. Revenue: product sales, subscription fees, commissions.

2. B2B — Business to Business

Commercial transactions between two businesses — manufacturer to wholesaler, wholesaler to retailer, or any business-to-business supply relationship. Typically involves: higher order values, credit terms, negotiated pricing, ERP/EDI integration, account management. Revenue model: bulk sales, contract-based pricing, platform fees.

3. C2C — Consumer to Consumer

Individual consumers sell to other individual consumers through an online platform that acts as intermediary. The platform provides listing, payment escrow, and dispute resolution. Revenue: listing fees, transaction commissions, premium features.

4. C2B — Consumer to Business

Individual consumers offer products or services to businesses. Reversal of traditional model — consumer creates value that businesses buy. Examples: freelance platforms, influencer marketing (individuals sell their reach to brands), stock photo contributors.

5. B2G — Business to Government

Businesses sell goods and services to government agencies electronically — e-procurement, e-tendering. Government is the buyer; businesses compete for contracts. Examples: defence procurement, IT services, office supplies to government departments.

6. G2C — Government to Consumer

Government delivers digital services to citizens — tax filing, licence applications, social benefits, utility bill payment. While not "commerce" in the traditional sense, it involves electronic transactions between government and citizens.

7. D2C — Direct to Consumer

Manufacturers sell directly to end consumers, bypassing traditional retail intermediaries (distributors, wholesalers, retailers). Enabled by e-commerce — brands own the customer relationship and data, and capture full retail margin. Fast-growing model.

(c) Challenges of E-Governance in India [3]

  • Digital divide: Rural areas have limited internet access and device ownership — a large population cannot access e-governance services. Only ~55% of India has internet access.
  • Digital literacy: Many citizens (especially elderly and rural) lack the skills to use online portals, fill digital forms, or navigate apps — creating exclusion.
  • Language barrier: Most government portals are primarily in English — India has 22 official languages. Inadequate regional language support limits adoption.
  • Infrastructure reliability: Power outages, slow internet, server downtime frustrate users and reduce trust in e-governance systems.
  • Cybersecurity: Government databases containing Aadhaar, PAN, health, and tax data are high-value targets for hackers — breaches have occurred.
  • Inter-department integration: Different government departments use different, often incompatible systems — citizen still has to visit multiple portals for related services.
  • Resistance to change: Government employees accustomed to manual processes resist digital transformation — implementation without buy-in fails.
Q8(a) Supply Chain Management [5]   (b) Merits of ERP before Supply Chain [5]   (c) Benefits of SC + ERP integration [5]15 Marks
✓ Answer

(a) Supply Chain Management [5]

Supply Chain Management (SCM) is the active management of all activities and relationships involved in sourcing raw materials, transforming them into finished products, and delivering them to the end customer — in the most efficient, cost-effective, and responsive manner possible.

A supply chain spans multiple organisations and functions:

Raw Material     Component      Manufacturer   Distributor   Retailer   End
Suppliers    →   Suppliers   →  (Production) → (Warehouse) → (Store) → Customer
                                      ↑_____________________________________________|
                                           Returns / Reverse Logistics

SCM activities: Demand planning, procurement, production scheduling, inventory management, warehousing, transportation, order management, returns processing, supplier performance management.

Goal: Deliver the right product, to the right customer, at the right time, in the right quantity, at the lowest possible total cost — while maintaining quality and flexibility.

(b) Merits of Implementing ERP Before Supply Chain [5]

  • Foundation of master data: ERP establishes clean, standardised master data (suppliers, materials, customers, locations) that all SCM applications depend on. Without this foundation, SCM systems have inconsistent, unreliable data.
  • Process standardisation: ERP enforces consistent processes across all business units — SCM implementations on top of standardised ERP processes are far smoother than trying to connect SCM to fragmented legacy systems.
  • Financial integration: Every SCM transaction (PO, goods receipt, shipment) needs to create accounting entries. ERP provides the financial backbone — SCM operations automatically flow into accounts without manual reconciliation.
  • Single source of inventory truth: ERP's inventory management module provides the real-time stock data that SCM systems (demand planning, replenishment) need to function correctly.
  • Reduced integration complexity: SCM tools integrate with one ERP system rather than multiple disparate legacy applications — dramatically simpler architecture, lower cost, fewer failure points.
  • Change management: Staff already familiar with ERP processes adapt more easily to SCM tools that extend the same platform (e.g., SAP APO/IBP extending SAP ERP).

(c) Benefits of Integrating Supply Chain and ERP Systems [5]

  • End-to-end visibility: From supplier PO to customer delivery — a single integrated view of the entire supply chain in real time, without manual data aggregation.
  • Automated order-to-fulfil cycle: Customer order in ERP → triggers inventory check → reserve stock → trigger warehouse pick → update shipping → generate invoice — all automatically without re-keying.
  • Accurate demand-driven replenishment: ERP sales order data feeds directly into supply chain planning tools — demand-driven procurement and production scheduling eliminates both stockouts and over-production.
  • Financial accuracy: Every supply chain event (receipt, shipment, return) immediately and automatically updates financial accounts — no month-end reconciliation backlog.
  • Supplier collaboration: Integrated system shares production schedules and demand forecasts with suppliers in real time — enabling JIT delivery and reducing safety stock requirements.
  • Performance management: Integrated KPI dashboards — OTIF (On Time In Full), inventory turns, fill rate, procurement cost — all from one data source, enabling continuous improvement.
  • Returns management: Customer return triggers automatic inventory update, quality inspection workflow, supplier credit note, and customer refund — all in one integrated process.
Q9(a) Security issues related to E-payments [9]   (b) Cryptocurrency as e-payment mechanism [6]15 Marks
✓ Answer

(a) Security Issues Related to E-Payments [9]

1. Authentication Risk

Verifying the identity of both payer and payee is challenging online. Stolen credentials (username/password) enable account takeover. Solutions: multi-factor authentication (MFA), biometric authentication, OTP, device fingerprinting.

2. Confidentiality / Data Interception

Payment data (card numbers, bank credentials) transmitted over networks can be intercepted if not encrypted. Solution: TLS 1.3 encrypts all HTTPS traffic; end-to-end encryption for sensitive fields.

3. Integrity Violations

Payment amounts or beneficiary details tampered in transit — a MITM attack changes ₹100 to ₹10,000 or redirects payment to attacker's account. Solution: digital signatures, message authentication codes (HMAC), TLS integrity checks.

4. Non-repudiation Failure

Merchants or customers deny completed transactions — "I never made that payment." Solution: digital signatures provide cryptographic proof of transaction; bank logs provide audit trail.

5. Phishing and Social Engineering

Fraudulent emails/SMS mimic banks or payment apps — trick users into entering credentials on fake sites. India's "KYC update" SMS scams steal banking credentials. Solution: user education, 2FA, anti-phishing filters.

6. Card Skimming and CNP Fraud

Stolen card data used for Card Not Present (CNP) online transactions. Magecart attacks inject skimmer scripts into checkout pages — harvest card details in real time. Solution: CVV verification, 3D Secure (OTP), tokenisation, PCI-DSS compliance.

7. Denial of Service on Payment Gateway

DDoS attack on payment gateway — no transactions can process — revenue loss and reputational damage during peak shopping periods. Solution: cloud-based DDoS protection, redundant payment gateway providers.

8. Insufficient Authorisation Controls

Weak transaction limits, no velocity checks — attacker makes multiple small fraudulent transactions below alert thresholds. Solution: real-time fraud scoring AI, transaction velocity limits, behavioural analytics.

9. Regulatory / Compliance Risk

Non-compliance with RBI regulations (Payment Aggregator Guidelines), PCI-DSS, and data localisation requirements exposes businesses to fines, licence cancellation, and reputational damage.

(b) Cryptocurrency as an e-Payment Mechanism [6]

Cryptocurrency (Bitcoin, Ethereum, USDC, Binance Coin) is a decentralised digital currency that runs on blockchain technology — a distributed, immutable ledger of all transactions.

How it works as e-payment:

Customer holds Bitcoin in a digital wallet (Coinbase, MetaMask)
    ↓
Merchant displays Bitcoin payment address or QR code
    ↓
Customer sends Bitcoin from their wallet to merchant's address
    ↓
Transaction broadcast to the Bitcoin network
    ↓
Miners validate the transaction (Proof of Work)
    ↓
Transaction confirmed in ~10 minutes (Bitcoin) or ~15 sec (Ethereum)
    ↓
Merchant's wallet balance updated — payment complete

Advantages as e-payment:

  • No chargebacks: Cryptocurrency transactions are irreversible — eliminates chargeback fraud that costs merchants billions annually.
  • Low cross-border fees: International transfers cost a fraction of SWIFT/SEPA fees — ideal for global e-commerce.
  • No intermediary: Peer-to-peer — no bank, no payment processor, no account freeze risk.
  • Financial inclusion: Accessible to the unbanked — only requires internet and a smartphone.
  • Stablecoins (USDC, USDT): Pegged to USD — eliminates price volatility concern for everyday transactions.

Challenges:

  • Price volatility: Bitcoin's value swings wildly — a payment accepted today may be worth 30% less tomorrow (stablecoins address this).
  • Regulatory uncertainty: RBI has been cautious about crypto payments in India — legal status unclear.
  • User complexity: Managing private keys, wallet addresses, and gas fees is too technical for mainstream consumers.
  • Transaction speed: Bitcoin takes 10+ minutes to confirm; high network congestion raises fees.
  • Tax implications: Each crypto payment is a taxable event in India (30% crypto tax) — complex accounting.

Verdict: Cryptocurrency has real potential for cross-border e-commerce, particularly using stablecoins on fast networks (Solana, Polygon). As regulatory frameworks mature and UX improves, crypto will likely become a mainstream e-payment option — but mass adoption is still some years away for Indian e-commerce.

Keys usedOne shared secret key for both encryption and decryptionTwo linked keys: public (shared) + private (secret)
Key sharingMust be securely shared before communication — the key distribution problemPublic key freely distributed; private key never shared

(b) Services Provided by SSL/TLS [5]

  • 1. Authentication (Server authentication): The SSL certificate verifies the server's identity — the client confirms it is communicating with the genuine server (e.g., actual amazon.in, not a phishing clone). Mutual authentication (client + server) is possible with client certificates.
  • 2. Confidentiality (Encryption): All data exchanged between browser and server is encrypted using symmetric session keys (AES-256 in modern TLS) — even if intercepted, the data is unreadable without the session key.
  • 3. Integrity: MAC (Message Authentication Code) appended to every record — detects any tampering or corruption of data in transit. If a MITM alters a single bit, the MAC check fails and the connection is terminated.
  • 4. Key Exchange: TLS handshake securely establishes session keys between client and server — using RSA or ECDH (Elliptic Curve Diffie-Hellman) key exchange algorithm. Ensures forward secrecy (past sessions cannot be decrypted even if private key is later compromised).
  • 5. Non-repudiation (via digital certificates): The server's certificate (signed by a CA) provides proof of identity that cannot be forged — combined with audit logs, creates a traceable, non-repudiable communication record.
  • 6. Protection against replay attacks: TLS uses sequence numbers and session IDs — prevents an attacker from recording and re-sending valid transactions to cause duplicate actions.

(c) RSA Algorithm [5]

RSA (Rivest-Shamir-Adleman) — Asymmetric Encryption

KEY GENERATION:
  1. Select two large prime numbers: P and Q
  2. Compute modulus:   N = P × Q
  3. Compute totient:   φ(N) = (P−1) × (Q−1)
  4. Choose public exponent E:
        1 < E < φ(N)  and  gcd(E, φ(N)) = 1
  5. Compute private exponent D:
        E × D ≡ 1 (mod φ(N))
        [Solve using Extended Euclidean Algorithm]

  Public key  : (E, N)   ← share freely
  Private key : (D, N)   ← keep secret

ENCRYPTION (sender uses recipient's PUBLIC key):
  Ciphertext C = M^E mod N

DECRYPTION (recipient uses their PRIVATE key):
  Plaintext  M = C^D mod N

DIGITAL SIGNATURE (sender uses their PRIVATE key):
  hash = SHA-256(Message)
  Signature = hash^D mod N
  [Recipient verifies: hash^E mod N == SHA-256(received message)]

WORKED EXAMPLE (P=17, Q=29):
  N    = 17 × 29 = 493
  φ(N) = 16 × 28 = 448
  E    = 3        [gcd(3, 448) = 1 ✓]
  D    = 299      [3 × 299 = 897 = 2×448+1 → 897 mod 448 = 1 ✓]

  Encrypt M=5:  C = 5³ mod 493 = 125
  Decrypt C=125: M = 125²⁹⁹ mod 493 = 5 ✓

SECURITY:
  Breaking RSA = factoring N into P and Q
  For N ≥ 2048 bits → computationally infeasible with
  classical computers (would take longer than the universe's age)
