Q1(i)Which one of the following is NOT an E-Commerce?1M

b) B2G
c) B2D

✓ Answer: (c) — B2D

B2D (Business-to-Developer) is not a standard e-commerce category. The recognised categories are: B2C (Business-to-Consumer), B2B (Business-to-Business), B2G (Business-to-Government), C2C (Consumer-to-Consumer), C2B (Consumer-to-Business), and G2C (Government-to-Citizen). B2D is sometimes used informally for developer-targeted APIs/tools but is not a standard e-commerce model.

Q1(ii)The default port of HTTP is1M
a) 81
b) 80
c) 70
d) 71
✓ Answer: (b) — Port 80

HTTP (HyperText Transfer Protocol) uses default port 80. HTTPS uses port 443. Other well-known ports: FTP (21), SSH (22), SMTP (25), DNS (53), POP3 (110), IMAP (143). When a URL has no explicit port number, browsers assume port 80 for HTTP and 443 for HTTPS.

An ISP is a company providing internet access and related services (web hosting, email, domain registration) to individuals and organisations. Examples: Jio, Airtel, BSNL, ACT Fibernet. ISPs connect customers to the internet backbone through DSL, fiber, cable, or mobile networks.

✓ Answer: (d) — All of these

e-Commerce evolved through all three technology eras: EDP (early EDI, batch transactions), MIS (inter-company networks, VANs), and Internet (web-based stores, online payments, mobile commerce). Modern e-commerce is the culmination of all three.

Q1(v)Friction in a transaction is defined as1M

b) Loss
c) Profit

✓ Answer: (a) — Overhead

Transaction friction = any overhead, barrier, or inefficiency that increases the cost or time of completing a transaction. e-Commerce reduces friction: instant price comparison, one-click checkout, automated payments. "Frictionless commerce" is the ideal — zero overhead between buyer intent and completed purchase.

Q1(vi)Flood attack by attackers happens on1M
a) Router
b) Hub
c) Switch
d) Server
✓ Answer: (d) — Server

A flood attack (DoS/DDoS) targets a server — overwhelming it with massive traffic (HTTP requests, SYN packets, UDP packets) until its resources are exhausted and it cannot serve legitimate users. The attacker's goal is to take the web server, DNS server, or payment gateway offline.

Q1(vii)Which one of the following is an ASCII protocol?1M
a) SNMP
b) SMTP
c) SET
d) PGP
✓ Answer: (b) — SMTP

SMTP (Simple Mail Transfer Protocol) uses human-readable ASCII text commands and responses (HELO, MAIL FROM, RCPT TO, DATA, QUIT). This makes it easy to debug but historically easy to forge. SNMP uses binary BER encoding; SET and PGP use binary/ASN.1 encoding.

Credit card payment is primarily associated with B2C (Business-to-Consumer) e-commerce — individual consumers paying businesses for goods/services. B2B typically uses credit terms, POs, and EDI-based invoicing rather than credit cards. C2C platforms like eBay use payment intermediaries (PayPal), but the credit card payment system model is fundamentally B2C.

Q1(x)DES stands for1M
a) Data Encryption Standard
b) Digital Encryption Standard
c) Double Encryption Standard
d) Data Encryption System
✓ Answer: (a) — Data Encryption Standard

DES (Data Encryption Standard) is a symmetric block cipher developed by IBM and adopted as a US federal standard in 1977. It uses a 56-bit key and 64-bit blocks with a 16-round Feistel network. Now considered insecure (56-bit key brute-forced in 1998) — replaced by 3DES and then AES.

Q2Define EDI. What are the components of EDI?5 Marks
✓ Answer

EDI (Electronic Data Interchange) is the direct computer-to-computer exchange of standardised business documents between organisations in a machine-readable format — eliminating paper, manual data entry, and postal delays. Both parties must use an agreed standard (ANSI X12 or EDIFACT).

Components of EDI

  • EDI Standards: Define the exact message structure and data elements — ANSI X12 (USA), UN/EDIFACT (international), HL7 (healthcare), ODETTE (automotive).
  • EDI Translation Software: Maps internal application data to/from the standard EDI format. Converts ERP flat files to X12 850 PO format for sending; converts received X12 810 invoice back to ERP format.
  • Communication Network: Transmission medium — VAN (Value Added Network), AS2 (HTTP-based, most common today), SFTP, AS4 (newer web-services based).
  • VAN (Value Added Network): Third-party secure store-and-forward network — acts as a "post office" holding messages until recipient retrieves them. Provides auditability and guaranteed delivery.
  • Trading Partner Agreement (TPA): Legal and technical agreement between two companies defining which EDI messages they'll exchange, response times, acknowledgement requirements, and data element usage conventions.
  • Functional Acknowledgement (997/CONTRL): Automated receipt confirmation — confirms the message was received and is syntactically valid (not necessarily semantically correct).
Q3Explain how SSL protocol is used for secure transaction. Explain the advantages of e-commerce.5 Marks
✓ Answer

SSL/TLS for Secure e-Commerce Transactions

SSL (Secure Sockets Layer) — now superseded by TLS (Transport Layer Security) — encrypts all data between the user's browser and the web server. It is the foundation of HTTPS.

TLS Handshake Process:

1. CLIENT HELLO: Browser sends supported TLS versions,
   cipher suites, random number.
2. SERVER HELLO: Server responds with chosen cipher,
   sends its SSL certificate (with public key + CA signature).
3. CERTIFICATE VERIFY: Browser verifies certificate
   against trusted CA store — confirms it's talking to
   the genuine website.
4. KEY EXCHANGE: Browser generates a pre-master secret,
   encrypts it with server's public key → sends to server.
5. SESSION KEY: Both sides derive the same symmetric
   session key from the pre-master secret.
6. ENCRYPTED CHANNEL: All subsequent data (form fields,
   card numbers, passwords) encrypted with AES session key.
7. FINISHED: Handshake complete — secure channel established.

What SSL protects in e-commerce: Login credentials, credit card numbers, personal data, session tokens — all encrypted in transit; cannot be read even if intercepted.

Advantages of e-Commerce

  • 24×7 global availability — no store hours, no geographic limits
  • Lower operational cost vs physical stores
  • Wider product selection — no shelf-space constraints
  • Personalised shopping experience via recommendation engines
  • Instant delivery for digital products
  • Easy price comparison for consumers
  • Rich customer analytics for businesses
  • Convenient — shop from home, office, or commute
Q4Discuss virtual auction. Differences between virtual auction and reverse auction.5 Marks
✓ Answer

Virtual Auction

A virtual (online) auction is an electronic marketplace where sellers list items and multiple buyers bid against each other — the highest bidder wins at auction close. Operates through the brokerage e-commerce model.

  • Process: Seller lists item with opening bid and end time → buyers place ascending bids → highest bid at closing time wins → payment and delivery arranged.
  • Types: English auction (ascending bids — eBay), Dutch auction (price descends until buyer accepts), sealed-bid auction (all bids submitted privately).
  • Examples: eBay, Sotheby's online, GovPlanet (equipment), government e-auctions.
  • Benefits: Price discovery through competition, access to rare/unique items, global buyer pool for sellers.

Q6What role does SCM play in B2B e-commerce?5 Marks
✓ Answer

SCM and B2B e-commerce are deeply intertwined — B2B e-commerce is primarily the electronic face of supply chain transactions:

  • Automated procurement: B2B portal or EDI-connected SCM automatically generates POs when stock falls below reorder level — no manual intervention. Buyer's ERP directly connects to supplier's B2B portal.
  • Catalogue management: B2B e-commerce portals provide punchout catalogues integrated with buyer's procurement system — buyers browse supplier catalogues within their own procurement workflow.
  • Order visibility: B2B portals give buyers real-time visibility into order status, shipment tracking, and expected delivery — reduces supply uncertainty.
  • Supplier performance management: B2B e-commerce platforms track on-time delivery, quality defects, and price compliance — suppliers rated and managed on performance data.
  • Invoice and payment automation: EDI-based B2B e-commerce automates invoice matching and payment triggering — shortens payment cycles, improves supplier relationships.
  • Demand signal sharing: B2B portals share rolling demand forecasts (DELFOR) with suppliers — enabling supplier capacity planning and JIT delivery.
  • Reduced lead times: Electronic B2B transactions eliminate postal and manual processing delays — SCM cycle times compressed from days/weeks to hours/minutes.

Q7Types of digital documents · Risk in e-Payment · Smart Cards and Credit Cards in e-commerce15 Marks
✓ Answer

Types of Digital Documents

  • Electronic Purchase Orders (e-PO): Formally placed orders in EDI (X12 850) or XML format — trigger automated fulfilment in supplier's ERP.
  • Electronic Invoices (e-Invoice): Machine-readable invoices in EDI (X12 810), XML, or PDF/A — automatically matched against POs in buyer's AP system.
  • Digital Contracts: Legally binding agreements signed with digital signatures (IT Act 2000) — e-commerce terms & conditions, supplier contracts.
  • Digital Certificates (X.509): CA-issued documents binding a public key to an identity — used in HTTPS, SET, code signing.
  • e-Receipts: Transaction confirmation documents — order confirmations, payment receipts, GST e-invoices.
  • Electronic Cheques (e-Cheques): Digital equivalent of a paper cheque — issued over internet, signed digitally, cleared through banking network.
  • Digital Licences: Software and content licences delivered and verified electronically — DRM-protected ebooks, software activation keys.

How Risk is Handled in e-Payment Systems

  • Encryption (TLS/SSL): All payment data encrypted in transit — prevents interception of card numbers and credentials.
  • Tokenisation: Card number replaced by a random token for storage and subsequent transactions — even if token is stolen, it's useless without the token vault.
  • 3D Secure (3DS): Additional authentication step — OTP, biometric, or app approval — verifies cardholder identity before transaction completes.
  • CVV/CVC verification: 3-digit code on card back never stored by merchants — reduces CNP (Card Not Present) fraud.
  • Fraud detection AI: Real-time transaction scoring — flags unusual patterns (foreign IP, large amount, new device) for additional verification or blocking.
  • PCI-DSS compliance: Payment Card Industry Data Security Standard — mandates encryption, access control, vulnerability scanning for all entities handling card data.
  • Chargeback system: Customer dispute mechanism — bank reverses fraudulent charges, protecting consumer from financial loss.

Smart Cards and Credit Cards in e-Commerce

Smart Cards in e-commerce:

  • EMV chip cards (contact smart cards) — used at POS terminals; chip generates a unique cryptogram per transaction preventing cloning.
  • Contactless smart cards / NFC — tap-to-pay at physical retailers; increasingly integrated with mobile payments (Apple Pay, Google Pay).
  • Digital wallet cards — stored in phone's secure element (a smart card chip inside the phone).
  • Stored-value smart cards — transit passes (Oyster, Mumbai Metro), loyalty cards — store balance on-chip.

Credit Cards in e-commerce:

  • Most common payment method for online purchases — Visa, Mastercard, Amex, RuPay.
  • Card number, expiry, and CVV entered at checkout → processed via payment gateway → authorised by card network and issuing bank.
  • Protected by 3D Secure, CVV verification, and real-time fraud scoring.
  • Chargebacks provide consumer protection — key reason for their dominance in online shopping trust.
  • Card-on-file (stored tokenised cards) enables one-click repeat purchases — improves conversion rates.

b) Working Principle of DES Algorithm

DES (Data Encryption Standard) is a symmetric block cipher that encrypts 64-bit plaintext blocks using a 56-bit key through 16 rounds of a Feistel network.

INPUT: 64-bit plaintext block

STEP 1: Initial Permutation (IP)
  Rearranges the 64 bits according to a fixed permutation table.

STEP 2: Split into L0 (left 32 bits) and R0 (right 32 bits)

STEP 3: 16 Rounds of Feistel encryption
  For each round i (1 to 16):
    L(i) = R(i−1)
    R(i) = L(i−1) XOR f(R(i−1), K(i))

  The function f involves:
    a) Expansion permutation: R(i−1) expanded from 32 → 48 bits
    b) Key mixing: XOR with 48-bit round subkey K(i)
    c) S-box substitution: 8 S-boxes compress 48 → 32 bits
       (this is the non-linear heart of DES — provides confusion)
    d) P-box permutation: rearranges 32 bits (provides diffusion)

  Key schedule: 56-bit key → 16 × 48-bit round subkeys K(1)...K(16)
  via PC-1 permutation + left circular shifts + PC-2 permutation

STEP 4: Swap: (R16, L16) → 64-bit pre-output

STEP 5: Final Permutation (IP⁻¹ — inverse of IP)

OUTPUT: 64-bit ciphertext block

DECRYPTION: Same algorithm, but subkeys applied in REVERSE order (K16...K1)

Weaknesses: 56-bit key → 2^56 ≈ 72 quadrillion possible keys → broken by brute force in 1998 (EFF Deep Crack). Replaced by 3DES (triple-apply DES with 3 keys = 168-bit effective security) and ultimately AES (2001).

EAN Coding System for EDI Messages

EAN (European Article Number) — now part of the GS1 system — provides standardised product and location identification codes used extensively in EDI messages.

  • EAN-13: 13-digit product identifier (barcode on consumer goods). Structure: 2–3 digit country prefix + 4–5 digit manufacturer code + 5 digit product code + 1 check digit.
  • EAN-8: Shortened 8-digit version for small packaging.
  • GLN (Global Location Number): 13-digit identifier for business locations — used in EDI to identify ship-to, ship-from, and bill-to addresses.
  • SSCC (Serial Shipping Container Code): 18-digit code on pallets/cartons — used in EDI ASN (DESADV) to identify each shipment container.
  • Use in EDI: EAN/GTIN product codes appear in EDI ORDERS (purchase order), DESADV (despatch advice), INVOIC (invoice) messages identifying products. GLN codes identify trading partner locations in EDI envelopes.
EDI INVOIC line item example:
LIN+1++5012345678900:EN'   (EN = EAN product code: 5012345678900)
QTY+47:100:PCE'             (100 pieces ordered)
MOA+203:500.00'             (line value £500)

Protecting Online Website Operations from Hackers

  • HTTPS/TLS: Encrypt all traffic — prevents MITM, eavesdropping, and credential theft.
  • Web Application Firewall (WAF): Filters HTTP traffic — blocks SQL injection, XSS, CSRF, path traversal, and other OWASP Top 10 attacks.
  • Input validation: Sanitise and validate ALL user inputs server-side — never trust client-submitted data.
  • Prepared statements / parameterised queries: Prevent SQL injection at the database layer.
  • Strong authentication: Multi-factor authentication for admin accounts; brute-force protection (CAPTCHA, account lockout, rate limiting).
  • Regular patching: Keep web server, CMS, plugins, and frameworks updated — most attacks exploit known vulnerabilities in unpatched software.
  • DDoS protection: CDN with DDoS scrubbing (Cloudflare, AWS Shield); rate limiting; anycast routing to absorb attack traffic.
  • Security scanning: Regular penetration testing, DAST (Dynamic Application Security Testing), vulnerability scans.
  • Principle of least privilege: Database user has only SELECT/INSERT on required tables — not full admin access.
  • Intrusion Detection/Prevention System (IDS/IPS): Monitors for anomalous traffic patterns and attack signatures.
  • Security headers: CSP (Content Security Policy), HSTS, X-Frame-Options, X-Content-Type-Options — prevent browser-based attacks.
  • Backups: Encrypted, off-site backups — rapid recovery in case of ransomware or data corruption.
Q11Short notes on any three: a) RSA   b) Trade Cycle & stages   c) Data Warehouse   d) Decision Support System   e) Collaborative Computing15 Marks
✓ Answer

a) RSA Algorithm

See 2014-15 Q10 for full RSA explanation. Summary: asymmetric encryption based on integer factorisation. Key generation: choose primes P, Q → N=PQ → φ(N)=(P−1)(Q−1) → choose E coprime to φ(N) → find D using Extended Euclidean. Encrypt: C=M^E mod N. Decrypt: M=C^D mod N. Used in HTTPS key exchange, digital signatures, SET protocol. Security relies on the infeasibility of factoring large N.

b) Trade Cycle and Different Stages

The trade cycle is the complete sequence of steps in a commercial transaction between a buyer and seller. In e-commerce context:

STAGES OF A TRADE CYCLE:

1. PRE-SALE:
   • Information search (customer browses catalogue, compares prices)
   • Product enquiry / RFQ (B2B)
   • Negotiation / Quote (B2B)

2. EXECUTION (ORDER):
   • Purchase Order placed (EDI 850 / web form)
   • Order acknowledgement (EDI 997 / 855)
   • Inventory reservation

3. SETTLEMENT (DELIVERY & PAYMENT):
   • Goods shipped — ASN sent (EDI 856)
   • Goods received and inspected
   • Invoice raised (EDI 810)
   • Payment made (EFT / EDI 820)
   • Remittance advice (EDI 820)

4. POST-SALE (AFTER-SALES):
   • Customer support / helpdesk
   • Returns / warranty claims (EDI 180)
   • Reorder trigger

Three types of trade cycles: Repeat/commodity cycle (routine JIT orders), Credit/large order cycle (negotiated B2B), Cash/low-value cycle (instant retail payment).

c) Data Warehouse

See 2010-11 Q11(d) and 2012-13 Q11(b) for full detail. Summary: centralised repository of integrated, historical, cleaned data from multiple operational systems (ERP, CRM, e-commerce) — optimised for analytical queries (OLAP). Key components: ETL pipeline, star/snowflake schema, OLAP cubes, BI tools. Characteristics: subject-oriented, integrated, non-volatile, time-variant (Inmon's 4). Examples: Amazon Redshift, Google BigQuery, Snowflake, SAP BW.

d) Decision Support System (DSS)

See 2012-13 Q11(e) for full detail. Summary: information system supporting managerial decision-making through data collection, analysis, and what-if modelling. Components: data management, model management, user interface. Types: model-driven (optimisation), data-driven (OLAP, data mining), knowledge-driven (expert system). In e-commerce: pricing DSS, demand forecasting, churn prediction, fraud scoring, inventory replenishment decisions.

e) Collaborative Computing

See 2011-12 Q11(d) for full detail. Summary: using technology to enable multiple people/systems in different locations to work together — video conferencing, shared document editing, project management tools, wikis. In e-commerce: cross-functional teams collaborate on promotions; suppliers and buyers co-design products; CPFR (Collaborative Planning Forecasting & Replenishment) in SCM; customer service knowledge base sharing. Enables global 24×7 teamwork without physical co-location.
