Q1(iii) HTML documents are saved in 1 Mark

D) None of these
✓ Answer: (A) ASCII text

HTML files are plain text files (saved with .html or .htm extension) encoded in ASCII (or UTF-8, which is ASCII-compatible). They can be opened and edited by any plain text editor like Notepad.

✓ Answer: (A) Default mode

In Telnet's default mode, the client terminal driver handles line editing locally. The client echoes each character to the screen but buffers the input until the user presses Enter, then sends the entire line to the server. This reduces network traffic.

a) Types of Lists in HTML (3 marks)

HTML supports three main types of lists:

1. Unordered List <ul> — displays items with bullet points.

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

2. Ordered List <ol> — displays items with numbers or letters.

<ol type="1">          <!-- 1,2,3 (default) -->
  <li>Step 1: Open editor</li>
  <li>Step 2: Write HTML</li>
  <li>Step 3: Save and open in browser</li>

<!-- type="A" for A,B,C  |  type="i" for i,ii,iii -->

3. Definition List <dl> — displays term-definition pairs.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language — used to structure web pages.</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets — used to style web pages.</dd>
</dl>

b) Use of <ol> and <ul> (2 marks)

  • <ul> (Unordered List): Used when the order of items does not matter — e.g., a list of ingredients, navigation menu items.
  • <ol> (Ordered List): Used when the sequence/order matters — e.g., step-by-step instructions, ranked items, numbered procedures.
  • Both use <li> (list item) tags for each entry.
  • Lists can be nested — an <ul> inside an <ol> or vice versa.
Q3 a) Usage of :hover, :active and :focus dynamic pseudo-classes    b) What is cookie? 3+2 = 5
✓ Answer

a) CSS Dynamic Pseudo-classes (3 marks)

Pseudo-classes apply styles based on user interaction state — they do not exist in the HTML source, they are dynamic.

:hover — Applies when the mouse pointer is over an element.

a:hover {
  color: #ff6600;
  text-decoration: underline;
}
button:hover {
  background: #6c63ff;
  transform: scale(1.05);
}

:active — Applies at the moment an element is being clicked/activated (mousedown).

button:active {
  background: #444;
  transform: scale(0.97);   /* pressed-in effect */
}
a:active { color: red; }

:focus — Applies when an element receives keyboard focus (tab key or click on input).

input:focus {
  outline: 2px solid #6c63ff;
  background: #f0f0ff;
}
textarea:focus { border-color: blue; }

b) Cookie (2 marks)

A cookie is a small piece of text data (≤ 4KB) that a web server creates and sends to the client's browser via the Set-Cookie HTTP response header.

  • The browser stores the cookie locally and sends it back to the server with every subsequent HTTP request to the same domain.
  • Used for: session management (login state), user preferences, shopping carts, tracking.
  • Attributes: Expires, Path, Domain, Secure (HTTPS only), HttpOnly (no JS access).
Q4 a) Implement the Perl regex: a*bc    b) What is spoofing? 3+2 = 5
✓ Answer

a) Perl Regular Expression: a*bc (3 marks)

The regex a*bc matches any string containing zero or more a's followed immediately by bc.

Matches: bc, abc, aabc, aaabc, xaaabc (contains the pattern)

Does NOT match: ac, ab, b, abbc

#!/usr/bin/perl
use strict;
use warnings;

my @test = ("bc", "abc", "aabc", "aaabc", "ac", "b", "abbc", "xaaabcy");

foreach my $str (@test) {
    if ($str =~ /a*bc/) {
        print "'$str'  => MATCHES\n";
    } else {
        print "'$str'  => No match\n";
    }
}

# Output:
# 'bc'      => MATCHES    (zero a's before bc)
# 'abc'     => MATCHES    (one a before bc)
# 'aabc'    => MATCHES    (two a's before bc)
# 'aaabc'   => MATCHES    (three a's before bc)
# 'ac'      => No match
# 'b'       => No match
# 'abbc'    => No match
# 'xaaabcy' => MATCHES    (pattern found within string)

To match the whole string, use anchors: /^a*bc$/

b) Spoofing (2 marks)

Spoofing is a cyberattack where an attacker impersonates a trusted entity (IP address, email sender, website, MAC address) to gain unauthorized access or intercept data.

  • IP Spoofing: Faking the source IP address in packets to impersonate another machine.
  • Email Spoofing: Sending emails with a forged sender address.
  • DNS Spoofing: Inserting false DNS records to redirect users to malicious sites.
  • ARP Spoofing: Sending fake ARP replies to associate attacker's MAC with a legitimate IP.
Q5 a) What is Denial-of-Service attack?    b) Discuss briefly about proxy firewall. 2+3 = 5
✓ Answer

a) Denial-of-Service (DoS) Attack (2 marks)

A DoS attack is a cyberattack that overwhelms a server, network, or service with excessive requests or traffic, making it unavailable to legitimate users.

  • DDoS (Distributed DoS): Attack launched from thousands of compromised machines (botnet) simultaneously — much harder to block.
  • Techniques: SYN Flood (half-open TCP connections), Ping of Death, UDP Flood, HTTP Flood.
  • Impact: Service downtime, financial loss, reputation damage.

b) Proxy Firewall (3 marks)

A proxy firewall (also called an application-layer firewall or application gateway) operates at the Application Layer (Layer 7) of the OSI model.

  • Acts as a complete intermediary: the client connects to the proxy, and the proxy connects to the actual server. The two sides never communicate directly.
  • Performs deep packet inspection — examines the actual content/payload of packets, not just headers.
  • Can inspect HTTP, FTP, SMTP traffic and block malicious content.
  • Advantages: Very high security, hides internal network topology, caches content.
  • Disadvantages: Slower than packet filtering, requires separate proxy for each protocol.

Q7 a) DNS meaning + Recursive Resolution with diagram    b) NVT significance in Telnet + Telnet operational modes (2+5)+(3+5) = 15
✓ Answer

a) DNS — Domain Name System (2 marks)

DNS (Domain Name System) is a hierarchical, distributed naming system that translates human-readable domain names (e.g., www.google.com) into machine-readable IP addresses (e.g., 142.250.195.196). It acts as the "phone book" of the internet.

DNS Hierarchy: Root servers → TLD servers (.com, .in) → Authoritative name servers → Local resolver.

DNS Recursive Resolution with Diagram (5 marks)

In recursive resolution, the client asks its local resolver to do all the work and return a final answer.

  Client Browser
       │
       │  1. Query: "What is IP of www.example.com?"
       ▼
  Local DNS Resolver (ISP)
       │
       │  2. Asks Root Name Server: "Who handles .com?"
       ▼
  Root Name Server
       │
       │  3. Returns: "Ask TLD server: 192.5.6.30"
       ▼
  Local DNS Resolver
       │
       │  4. Asks TLD (.com) Name Server: "Who handles example.com?"
       ▼
  TLD Name Server (.com)
       │
       │  5. Returns: "Ask authoritative server: 205.251.196.1"
       ▼
  Local DNS Resolver
       │
       │  6. Asks Authoritative Name Server: "IP of www.example.com?"
       ▼
  Authoritative Name Server (example.com)
       │
       │  7. Returns: "93.184.216.34"
       ▼
  Local DNS Resolver
       │
       │  8. Caches result, returns IP to client
       ▼
  Client Browser  ──► Connects to 93.184.216.34

Key point: The local resolver does all the iterative querying. The client only makes one query and gets back a final answer — that is why it is called "recursive" from the client's perspective.

b) Significance of NVT in TELNET (3 marks)

NVT (Network Virtual Terminal) is a virtual device defined by Telnet to provide a standard interface between different computers and operating systems.

  • Different systems have different keyboard layouts, character sets, and terminal behaviours.
  • NVT defines a common character set (7-bit ASCII) and a set of standard control functions that both client and server understand.
  • The client converts its local terminal input into NVT format before sending; the server converts NVT format into its local format.
  • This ensures interoperability — a Windows machine can Telnet into a Unix server without compatibility issues.

Q8 a) HTTP meaning   b) URL + components   c) GET vs POST   d) FTP transmission modes 2+(1+4)+4+4 = 15

a) HTTP — HyperText Transfer Protocol (2 marks)

HTTP (HyperText Transfer Protocol) is an application-layer protocol that defines the rules for transferring hypertext documents (HTML, images, videos) between a web client (browser) and a web server over the internet.

  • Stateless: Each request is independent — server doesn't remember previous requests.
  • Port: Default port 80; HTTPS (encrypted) uses port 443.
  • Model: Request–Response. Client sends a request; server sends a response.

b) URL — Uniform Resource Locator (1+4 marks)

A URL is the complete address used to access a specific resource on the internet.

d) FTP File Transmission Modes (4 marks)

FTP (File Transfer Protocol) supports three data transmission modes across its data connection:

  • 1. Stream Mode (default): Data is transmitted as a continuous stream of bytes. No special structure. The end of file is signaled by closing the connection. Simple and efficient.
  • 2. Block Mode: Data is divided into blocks. Each block has a 3-byte header containing a descriptor byte (flags) and a 2-byte byte count. Supports restart/resume of interrupted transfers via markers.
  • 3. Compressed Mode: Uses run-length encoding to compress repetitive data before transmission. A compressed byte sequence replaces repeated characters. Less commonly used today as the OS/application handles compression.
Q9 a) Classless addressing + NAT working    b) Subnet, masking, default vs subnet mask    c) First and last address of 205.16.37.39/28 15
✓ Answer

a) Classless Addressing (CIDR) + How NAT works

Classless Inter-Domain Routing (CIDR) eliminates the rigid class boundaries of classful addressing (Class A/B/C). Addresses are written as IP/prefix-length (e.g., 192.168.1.0/24). Any number of bits can be used for the network portion, allowing flexible allocation and reduced IP wastage.

How NAT (Network Address Translation) works:

  • A router maintains a NAT translation table mapping private IP:port ↔ public IP:port.
  • When a private host (e.g., 192.168.1.5) sends a packet to the internet, the NAT router replaces the source IP with its public IP and records the mapping.
  • When a reply arrives at the public IP, the router looks up the NAT table and forwards the packet to the correct private host.
  Private LAN            NAT Router              Internet
  192.168.1.5:3456 ──►  203.45.12.1:5001 ──►  Server 8.8.8.8
  192.168.1.5:3456 ◄──  203.45.12.1:5001 ◄──  Server 8.8.8.8

  NAT Table:
  Private IP:Port        Public IP:Port
  192.168.1.5:3456  ↔   203.45.12.1:5001

b) Subnet, Masking, Default Mask vs Subnet Mask

Subnet: A logically subdivided segment of an IP network. Subnetting divides a large network into smaller, more manageable sub-networks.

Why masking is necessary: A subnet mask tells a device which portion of an IP address is the network part and which is the host part. Without it, devices cannot determine if a destination is local or remote, and routing cannot work.

Q10 a) Why MIME is used with SMTP    b) Mail Transfer Agent responsibility    c) SMTP email delivery stages with diagram 3+6+6 = 15
✓ Answer

a) Why MIME is used as an extension of SMTP (3 marks)

SMTP (Simple Mail Transfer Protocol) was originally designed to transfer only 7-bit ASCII text. It cannot natively handle:

  • Non-ASCII characters (e.g., Hindi, Chinese, Arabic text)
  • Binary attachments (images, PDFs, Word documents)
  • Audio/video files
  • Multi-part messages (text + attachment together)

MIME (Multipurpose Internet Mail Extensions) extends SMTP by:

  • Encoding binary data as ASCII using Base64 or Quoted-Printable encoding.
  • Adding header fields: MIME-Version, Content-Type, Content-Transfer-Encoding.
  • Supporting multipart messages — multiple sections (text body + attachments) in one email.

b) Mail Transfer Agent (MTA) Responsibilities (6 marks)

An MTA (Mail Transfer Agent) is server software responsible for routing and delivering email messages between mail servers.

  • Receiving mail: Accepts email from MUAs (Mail User Agents) or other MTAs via SMTP.
  • DNS MX lookup: Queries DNS for the MX (Mail Exchanger) record of the recipient's domain to find the destination mail server.
  • Routing: Determines the next hop for the message and forwards it accordingly.
  • Queuing: If destination server is unavailable, stores the message in a queue and retries delivery at intervals.
  • Bouncing: If delivery permanently fails, sends a Non-Delivery Report (NDR) back to the sender.
  • Relaying: Passes the message through intermediate MTAs toward the final destination.
  • Examples: Postfix, Sendmail, Microsoft Exchange, Exim.

Q11 Short Notes (any 3): a) TCP 3-way handshake   b) Path vector routing   c) Proxy Firewall   d) HTML Image Map   e) XML tree 3 × 5 = 15
✓ Answer

a) Three-way Handshaking of TCP Connection (5 marks)

TCP establishes a reliable connection using a three-way handshake before any data transfer.

  Client                                    Server
    │                                          │
    │──── SYN (seq=x) ───────────────────────►│  Step 1: Client sends SYN
    │                                          │  (synchronize, wants to connect)
    │◄─── SYN-ACK (seq=y, ack=x+1) ──────────│  Step 2: Server sends SYN-ACK
    │                                          │  (acknowledges client, sends own SYN)
    │──── ACK (ack=y+1) ─────────────────────►│  Step 3: Client acknowledges
    │                                          │
    │════════ Data Transfer Begins ════════════│
  • Step 1 (SYN): Client sends a TCP segment with SYN flag set and its Initial Sequence Number (ISN = x).
  • Step 2 (SYN-ACK): Server acknowledges (ACK = x+1) and sends its own SYN with ISN = y.
  • Step 3 (ACK): Client acknowledges server's SYN (ACK = y+1). Connection established.

After three-way handshake, both sides have agreed on sequence numbers and the connection is reliable.

b) Path Vector Routing (5 marks)

Path Vector Routing is a routing algorithm where each routing table entry contains not just the destination and distance, but also the complete path (sequence of nodes) to reach the destination.

  • Used by BGP (Border Gateway Protocol) — the routing protocol of the internet between Autonomous Systems (AS).
  • Loop prevention: Before accepting a route, a router checks if its own AS number appears in the path — if yes, it rejects the route (loop detected).
  • Policy routing: Can enforce policies — e.g., avoid routing through certain ASes.
  • More flexible than distance vector (avoids count-to-infinity) and more scalable than link state for inter-domain routing.

c) Proxy Firewall (5 marks)

A proxy firewall sits between client and server, intercepting all application-layer traffic. Neither side communicates directly — all traffic flows through the proxy.

  • Operates at Layer 7 (Application Layer) — understands HTTP, FTP, SMTP protocols.
  • Performs deep content inspection — can scan for viruses, block URLs, filter keywords.
  • Hides internal IP addresses — server only sees proxy's IP.
  • Can cache frequently accessed content for performance.
  • Types: HTTP proxy, SOCKS proxy, transparent proxy, reverse proxy.
  • Disadvantage: Higher latency, needs a separate proxy per application protocol.

d) HTML Image Map (5 marks)

An Image Map allows different areas (regions) of a single image to act as separate hyperlinks. Clicking different zones of the image navigates to different URLs.

<!-- Image linked to a map -->
<img src="world-map.jpg"
     alt="World Map"
     usemap="#worldmap"

<map name="worldmap">
  <!-- Rectangular area -->
  <area shape="rect"
        coords="50,100,200,250"
        href="india.html"
        alt="India">

  <!-- Circular area -->
  <area shape="circle"
        coords="350,150,80"
        href="usa.html"
        alt="USA">

  <!-- Polygonal area -->
  <area shape="poly"
        coords="400,50,500,50,450,150"
        href="europe.html"
        alt="Europe">
</map>

Shapes: rect (x1,y1,x2,y2), circle (cx,cy,radius), poly (x1,y1,x2,y2,...), default (entire image).

e) XML Tree (5 marks)

An XML tree is the hierarchical tree structure that represents an XML document. Every XML document has exactly one root element, and all other elements are children or descendants of it.

XML Document:
<?xml version="1.0"?>
<university>
  <department name="CSE">
    <student>
      <name>Alice</name>
      <roll>101</roll>
    </student>
    <student>
      <name>Bob</name>
      <roll>102</roll>
    </student>
  </department>
</university>

Tree Representation:
university  (root)
└── department [@name="CSE"]
    ├── student
    │   ├── name  → "Alice"
    │   └── roll  → "101"
    └── student
        ├── name  → "Bob"
        └── roll  → "102"

The DOM parser builds exactly this tree in memory, allowing traversal using parentNode, childNodes, firstChild, nextSibling etc.
