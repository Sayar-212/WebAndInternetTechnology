Q1(i)The correct HTML tag for the biggest heading is1 Mark
a) <h6>
b) <heading small>
c) <small heading>
d) <h1>
✓ Answer: (d) <h1>

HTML headings range from <h1> (largest/most important) to <h6> (smallest). <h1> is typically used for the main page title. There are no tags like <heading small> or <small heading> in HTML.

Q1(ii)Which technology is NOT a part of J2EE architecture?1 Mark
a) JSP
b) Servlet
c) EJB
d) COBRA
✓ Answer: (d) COBRA (should be CORBA)

J2EE (Java 2 Enterprise Edition) includes JSP (JavaServer Pages), Servlet, EJB (Enterprise JavaBeans), JDBC, JMS, JNDI, etc. CORBA (Common Object Request Broker Architecture) is an OMG standard — not a J2EE component. The option "COBRA" is a typo for CORBA.

Q1(iii)EJB contains1 Mark
a) Unique interface
b) Four interfaces
c) Two interfaces
d) No interface
✓ Answer: (c) Two interfaces

An EJB (Enterprise JavaBean) traditionally consists of two interfaces: the Home interface (for creating/finding/removing bean instances) and the Remote interface (for defining business methods accessible by remote clients). In EJB 3.x this was simplified, but the classic answer is two interfaces.

Q1(iv)The JSP tags are interpreted by1 Mark
a) JSP compiler
b) JSP engine
c) JSP debugger
d) None of these
✓ Answer: (b) JSP engine

The JSP engine (part of the web container, e.g., Tomcat) processes JSP tags. It translates the JSP file into a Servlet, then compiles and executes it. The JSP engine handles all JSP-specific tags like <% %>, <%= %>, <%@ %>.

Q1(v)Which protocol is stateless in nature?1 Mark
a) FTP
b) HTTP
c) TCP
d) UDP
✓ Answer: (b) HTTP

HTTP is stateless — each request/response pair is completely independent. The server does not retain any memory of previous requests from the same client. Session state must be managed explicitly via cookies, sessions, or tokens. TCP is connection-oriented (stateful). FTP maintains a session.

Q1(vi)Variables are declared in JavaScript using which of the following?1 Mark
a) dim
b) var
c) function
d) variable
✓ Answer: (b) var

In JavaScript, variables are declared using var (function-scoped), let (block-scoped, ES6+), or const (block-scoped constant, ES6+). dim is used in Visual Basic. Example: var x = 10; let name = "Alice"; const PI = 3.14;

Q1(vii)After a servlet is destroyed, garbage collection is performed by1 Mark
a) The server
b) The servlet
c) HTTP
d) JVM
✓ Answer: (d) JVM

After destroy() is called, the servlet object becomes eligible for garbage collection. Java's JVM (Java Virtual Machine) has an automatic garbage collector that reclaims memory from objects that are no longer referenced. Neither the server, HTTP, nor the servlet itself performs GC.

Q1(viii)The ____ in XML is equivalent to the schema in database.1 Mark
a) Element
b) Attribute
c) Parser
d) DTD
✓ Answer: (d) DTD

DTD (Document Type Definition) defines the legal structure, elements, attributes, and their relationships for an XML document — just like a database schema defines the structure of tables. A document is "valid" if it conforms to its DTD. XML Schema (XSD) is a more powerful modern alternative.

Q1(ix)An XML parser is used to1 Mark
a) Display contents of XML
b) Read but not interpret
c) Read and interpret
d) None of these
✓ Answer: (c) Read and interpret

An XML parser reads and interprets an XML document — it checks well-formedness, optionally validates against a DTD/Schema, and makes the document's structure and content available to the application through a DOM tree or SAX events.

Q1(x)Which statement is true about XML?1 Mark
a) XML tags are case sensitive
b) XML documents must have a root tag
c) XML elements must be properly closed
d) All the statements are true
✓ Answer: (d) All the statements are true

All three are fundamental XML rules: (a) <Name> and <name> are different elements in XML. (b) Every XML document must have exactly one root element enclosing all others. (c) Every opening tag must have a matching closing tag (or be self-closed like <br/>).

Q2Why dynamic webpage? Advantages of stylesheet? How to link stylesheet to webpage?5 Marks
✓ Answer

Why Dynamic Webpages?

A dynamic webpage generates content at runtime based on user input, database queries, or time — unlike static pages that always show the same content. Dynamic pages enable: personalised content, real-time data updates, interactive forms, e-commerce, social media feeds, and user-specific dashboards — all without the user needing to reload the entire page.

Advantages of Using Stylesheets

  • Separation of concerns: HTML handles structure; CSS handles presentation — easier to maintain.
  • Consistency: One CSS file can style hundreds of pages uniformly.
  • Reduced HTML size: No inline style attributes on every element.
  • Easy updates: Change one CSS rule to update style across all pages instantly.
  • Responsive design: Media queries adapt layout to screen size.
  • Reusability: Same stylesheet shared across multiple websites.

How to Link a Stylesheet

<!-- 1. External CSS (recommended) — inside <head> -->
<link rel="stylesheet" type="text/css" href="styles.css">

<!-- 2. Internal CSS — inside <head> -->
<style>
  body { background: #fff; font-family: Arial; }
  h1   { color: #6c63ff; }
</style>

<!-- 3. Inline CSS — on individual element -->
<p style="color: red; font-size: 16px;">Hello</p>

<!-- 4. Import inside CSS file -->
<style> @import url("other.css"); </style>

Q4JavaScript in client-side data validation with examples. Need for client-side validation.5 Marks
✓ Answer

Need for Client-Side Validation

  • Provides instant feedback to the user without a round-trip to the server.
  • Reduces server load — invalid data never reaches the server.
  • Improves user experience — errors highlighted immediately.
  • Saves bandwidth — no unnecessary form submissions.

JavaScript Validation Examples

<form onsubmit="return validateForm()">
  Name:  <input type="text"  id="name">
  Email: <input type="text"  id="email">
  Age:   <input type="number" id="age">
  <input type="submit" value="Submit">
</form>

<script>
function validateForm() {
  var name  = document.getElementById("name").value.trim();
  var email = document.getElementById("email").value.trim();
  var age   = document.getElementById("age").value;

  // 1. Empty check
  if (name === "") {
    alert("Name is required!"); return false;
  }

  // 2. Email format check using regex
  var emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!emailRegex.test(email)) {
    alert("Enter a valid email!"); return false;
  }

  // 3. Range check
  if (age < 18 || age > 100) {
    alert("Age must be between 18 and 100!"); return false;
  }

  return true; // form submits only if all valid
}
</script>

Note: Client-side validation improves UX but should always be backed by server-side validation — JavaScript can be disabled by users.

Q5What is Servlet? Servlet API? Lifecycle of a Servlet.5 Marks
✓ Answer

What is a Servlet?

A Servlet is a Java class that runs on a web/application server (inside a Servlet Container like Tomcat) and handles HTTP requests to generate dynamic web content. It extends javax.servlet.http.HttpServlet.

Servlet API

The Servlet API is provided by two packages:

  • javax.servlet — generic interfaces: Servlet, ServletRequest, ServletResponse, ServletConfig, ServletContext
  • javax.servlet.http — HTTP-specific: HttpServlet, HttpServletRequest, HttpServletResponse, HttpSession, Cookie

Servlet Lifecycle

  1. LOADING       — Container loads Servlet class (.class) into JVM
  2. INSTANTIATION — Container creates ONE instance of the Servlet
         │
         ▼
  3. init(ServletConfig)
     — Called ONCE on first request (or at startup)
     — Initialize DB connections, load config params
         │
         ▼
  4. service(HttpServletRequest, HttpServletResponse)  ← called for EVERY request
     — Dispatches to doGet(), doPost(), doPut(), doDelete() etc.
     — Multiple threads can call service() simultaneously
         │
         ▼
  5. destroy()
     — Called ONCE before the servlet is unloaded
     — Release resources (close DB connections, clean up)
// Minimal Servlet example
import javax.servlet.*; import javax.servlet.http.*; import java.io.*;
public class HelloServlet extends HttpServlet {
    public void doGet(HttpServletRequest req, HttpServletResponse res)
            throws IOException {
        res.setContentType("text/html");
        PrintWriter out = res.getWriter();
        out.println("<h1>Hello from Servlet!</h1>");
    }
}

Q6a) Internal valid XML with example   b) Well-formed XML rules   c) DOM vs SAX   d) Explain DTD declarations6+2+3+4 = 15
✓ Answer

a) Internal Valid XML Document (6 marks)

A valid XML document is one that is well-formed AND conforms to an associated DTD or Schema. An internal DTD is embedded within the XML file itself inside the <!DOCTYPE> declaration.

<?xml version="1.0" encoding="UTF-8"?>

<!DOCTYPE library [
  <!ELEMENT library (book+)>
  <!ELEMENT book (title, author, price)>
  <!ELEMENT title  (#PCDATA)>
  <!ELEMENT author (#PCDATA)>
  <!ELEMENT price  (#PCDATA)>
  <!ATTLIST book isbn CDATA #REQUIRED
                 genre CDATA #IMPLIED>
]>

<library>
  <book isbn="978-0134685991" genre="Programming">
    <title>Effective Java</title>
    <author>Joshua Bloch</author>
    <price>599</price>
  </book>
  <book isbn="978-0596516178" genre="Web">
    <title>JavaScript: The Good Parts</title>
    <author>Douglas Crockford</author>
    <price>450</price>
  </book>
</library>

This document is valid because: it has one root <library>, all elements match the DTD definitions, required attribute isbn is present on all <book> elements.

b) Rules for Well-Formed XML Document (2 marks)

  • Must have exactly one root element enclosing all other elements.
  • All elements must have a closing tag (or be self-closed: <br/>).
  • Tags are case-sensitive — <Name> ≠ <name>.
  • Elements must be properly nested — no crossing tags.
  • Attribute values must be in quotes (single or double).
  • Special characters must use entities: &amp; &lt; &gt; &quot; &apos;
  • XML declaration should appear on the first line.

c) DOM vs SAX Parser (3 marks)

FeatureDOMSAX
ApproachTree-based — loads entire documentEvent-driven — streaming, sequential
MemoryHigh — whole doc in RAMLow — processes node by node
SpeedSlower for large filesFaster for large files
AccessRandom — navigate any nodeForward-only, no backtracking
ModificationCan read and write/modifyRead-only
Best useSmall/medium docs, frequent navigationLarge files, memory-constrained systems

d) Explain DTD Declarations (2+2 marks)

(i) <!ELEMENT section (title+, para(note | code*)>

  • Declares an element named section.
  • It must contain: one or more title elements (+ = one or more), followed by para elements.
  • Each para contains either note OR zero-or-more code elements (| = choice, * = zero or more).

(ii) <!ATTLIST sign signaturefile CDATA #IMPLIED email CDATA #REQUIRED>

  • Declares attributes for element sign.
  • signaturefile: type CDATA (character data, any string), #IMPLIED = optional attribute.
  • email: type CDATA, #REQUIRED = this attribute MUST be present in every <sign> element.

Q8Congestion Policy, Congestion Avoidance, and Congestion Detection in TCP with diagrams.3 × 5 = 15
✓ Answer

1. Congestion Policy — Slow Start (5 marks)

When a TCP connection begins (or after timeout), it enters Slow Start — the congestion window (cwnd) starts small and grows exponentially.

  • Initially: cwnd = 1 MSS (Maximum Segment Size)
  • Each ACK received → cwnd += 1 MSS → doubles per RTT (exponential)
  • Continues until cwnd ≥ ssthresh (slow start threshold)
cwnd
 16│                    *
  8│             *
  4│       *
  2│   *
  1│*
   └──────────────────── RTT
    (Doubles each round — Slow Start)

2. Congestion Avoidance (5 marks)

Once cwnd reaches ssthresh, TCP switches to Congestion Avoidance — linear (additive) growth.

  • Each full RTT → cwnd += 1 MSS (linear growth)
  • This is AIMD: Additive Increase, Multiplicative Decrease
  • Gently probes for more bandwidth without causing congestion
cwnd
  │                        * * *
  │                   * *
  │              * *                ← Congestion Avoidance (linear)
ssthresh──────*──────────────────────
  │     *  *   ← Slow Start (exponential)
  │  *
  │*
  └──────────────────────────── RTT

3. Congestion Detection (5 marks)

TCP detects congestion through two signals:

Signal 1 — Timeout:

  • Retransmission timer expires → packet assumed lost due to severe congestion
  • ssthresh = cwnd / 2
  • cwnd = 1 MSS — restart Slow Start from scratch

Signal 2 — 3 Duplicate ACKs (Fast Retransmit):

  • Receiver gets out-of-order segments and sends duplicate ACKs for the last in-order segment
  • 3 dup-ACKs = mild congestion (some packets still getting through)
  • ssthresh = cwnd / 2, cwnd = ssthresh (Fast Recovery — no reset to 1)
  • Retransmit the missing segment immediately without waiting for timeout
  After Timeout:          After 3 dup-ACKs:
  cwnd                    cwnd
   │\                      │\
   │ \   ssthresh/2        │ \   ssthresh/2
   │  \──────────          │  *── (stays at ssthresh, Fast Recovery)
   │   \                   │
   1────*── slow start     └── no reset to 1

/28 → 28 network bits, 4 host bits
Subnet mask: 255.255.255.240

Last octet = 39
Blocks of 16: 0, 16, 32, 48...
39 falls in block starting at 32

Network (first) address : 205.16.37.32
Broadcast (last) address: 205.16.37.47
Usable hosts: 2^4 - 2 = 14  (.33 to .46)

Subnet: A logically subdivided segment of a network, created by borrowing bits from the host portion of an IP address.

Is masking required? Yes — without a subnet mask, a device cannot determine which portion of an IP address is the network part and which is the host part. Routing decisions depend entirely on the mask.

b) DoS Attack (4 marks)

A Denial of Service (DoS) attack floods a target with traffic/requests to exhaust its resources (CPU, memory, bandwidth), making it unavailable to legitimate users. Types: SYN Flood, UDP Flood, Ping of Death, HTTP Flood. DDoS uses thousands of compromised hosts simultaneously. Defence: rate limiting, firewalls, CDN, anti-DDoS services.

RTP — Real-time Transport Protocol (4 marks)

RTP carries real-time audio/video over IP (UDP). Header contains: Version, Payload Type (codec), Sequence Number (for ordering/loss detection), Timestamp (synchronisation/jitter), SSRC (stream identifier). Used with RTCP for QoS monitoring (packet loss, jitter, delay stats).

NVT Utility in Telnet (2 marks)

NVT (Network Virtual Terminal) provides a standardised, universal terminal interface for Telnet. Different systems have different terminal types — NVT acts as a common intermediate format so any client terminal can communicate with any server, regardless of their native terminal implementations.

Q10Short Notes (any 3): a) IPTV   b) Web Crawler   c) IP Security   d) Proxy Firewall   e) HTML DOM3 × 5 = 15
✓ Answer

a) IPTV (5 marks)

IPTV (Internet Protocol Television) delivers television content over IP networks (internet or managed private networks) instead of traditional cable or satellite.

  • Live TV: Uses IP Multicast + RTP — one stream sent to all group members efficiently.
  • VoD (Video on Demand): Uses unicast — individual stream per viewer.
  • Time-shifted TV: Catch-up/pause live TV.
  • Protocols used: IGMP (multicast group management), RTP (stream delivery), RTSP (stream control), HTTP (VoD streaming).
  • Advantages: Interactive features (EPG, pause, rewind), personalised content, global access, cheaper than cable.
  • Examples: Netflix, Amazon Prime, Hotstar, YouTube TV, Jio TV.

b) Web Crawler (5 marks)

An automated bot that systematically browses the web to index pages for search engines. Process: Start from seed URLs → download page → extract links → add to queue → index content → repeat. Respects robots.txt. Examples: Googlebot, Bingbot. Challenges: duplicate detection, dynamic JavaScript pages, rate limiting, infinite URL spaces.

c) IP Security (IPSec) (5 marks)

IPSec is a suite of protocols providing security at the IP layer — authentication, integrity, and encryption of IP packets.

  • AH (Authentication Header): Provides integrity and source authentication. Does not encrypt.
  • ESP (Encapsulating Security Payload): Provides encryption + integrity + authentication.
  • Transport Mode: Encrypts payload only; original IP header intact.
  • Tunnel Mode: Encrypts entire original IP packet; new outer header added. Used in VPNs.
  • IKE (Internet Key Exchange): Negotiates security associations and manages keys.

d) Proxy Firewall (5 marks)

Operates at Application Layer (Layer 7). All client-server communication passes through the proxy — neither side connects directly. Deep packet inspection of HTTP/FTP/SMTP content. Hides internal IP addresses. Caches content for performance. Can block URLs, scan for malware. Disadvantage: higher latency, one proxy per protocol.

e) HTML DOM (5 marks)

The HTML DOM (Document Object Model) is a programming interface that represents an HTML document as a tree of objects. JavaScript uses the DOM to dynamically access and manipulate the page.
