Q1(I)Explain RTP and mention the use of RTP.1M
✓ Answer

RTP (Real-time Transport Protocol) is a transport protocol designed to deliver real-time audio and video over IP networks using UDP.

Uses: VoIP phone calls, video conferencing (Zoom, Teams), live streaming, IPTV, WebRTC. Provides sequence numbers (ordering), timestamps (synchronisation), and payload type identification (codec).

Q1(II)Expand the term WWW and mention its importance with respect to Internet.1M
✓ Answer

WWW = World Wide Web. A system of interlinked hypertext documents and resources accessed via the internet using web browsers and HTTP.

Importance: Enables global information sharing, e-commerce, online education, social media, cloud services, and universal access to knowledge through hyperlinks and URLs.

Q1(III)Write the full form of CSS and mention its significance.1M
✓ Answer

CSS = Cascading Style Sheets. A stylesheet language that controls the presentation of HTML documents.

Significance: Separates content from design; controls layout, colours, fonts, animations; enables responsive design; ensures consistent look across pages; reduces HTML clutter.

Q1(VII)What is FTP? What are the advantages of using FTP?1M
✓ Answer

FTP (File Transfer Protocol) is an application-layer protocol for transferring files between client and server over TCP (port 21 for control, port 20 for data).

Advantages: Fast large file transfer, resume interrupted transfers, transfer multiple files/directories, user authentication, widely supported by all operating systems.

Q1(VIII)Expand the term HTML and mention the use of HTML with respect to websites.1M
✓ Answer

HTML = HyperText Markup Language. The standard markup language for creating web pages.

Use: Defines the structure and content of webpages using elements/tags — headings, paragraphs, links, images, tables, forms. It is the skeleton of every website; CSS styles it and JavaScript makes it interactive.

Q1(IX)What is DNS? Mention the importance of DNS.1M
✓ Answer

DNS (Domain Name System) translates human-readable domain names (e.g., www.google.com) into IP addresses (e.g., 142.250.80.46) that computers use to communicate.

Importance: Acts as the "phone book" of the internet; users don't need to remember IP addresses; enables load balancing and failover via multiple IP records; essential for email routing (MX records).

Q1(X)What do you mean by denial of service attacks?1M
✓ Answer

A DoS (Denial of Service) attack floods a server or network with excessive traffic, exhausting its resources (CPU, memory, bandwidth) and making it unavailable to legitimate users.

DDoS (Distributed DoS): Uses thousands of compromised machines (botnet) to attack simultaneously — much harder to block. Examples: SYN flood, UDP flood, HTTP flood.

Q1(XI)Explain the PageRank technique with respect to web page searching.1M
✓ Answer

PageRank is Google's original algorithm for ranking web pages in search results based on link structure.

Core idea: A page is important if many high-quality pages link to it. Formula: PR(A) = (1−d)/N + d × Σ(PR(linking pages) / out-links). Higher PageRank → higher position in search results. Foundation of modern SEO.

Q2Define the term 'Cookie' and explain the process to create and store cookies.5 Marks
✓ Answer

A cookie is a small text file (≤4 KB) stored in the user's browser containing key-value pairs, set by the web server to maintain state across HTTP's stateless requests.

Creation & Storage Process

1. Client → HTTP Request → Server
2. Server processes request
3. Server → HTTP Response with header:
   Set-Cookie: session_id=abc123; Expires=Wed, 31 Dec 2025; Path=/; HttpOnly; Secure
4. Browser stores cookie in its cookie jar (per domain)
5. Client → subsequent requests include:
   Cookie: session_id=abc123
// Server-side (Node.js example)
res.setHeader("Set-Cookie", "user=Alice; Max-Age=3600; Path=/; HttpOnly");

// Client-side JavaScript
document.cookie = "theme=dark; expires=Fri, 31 Dec 2025 23:59:59 GMT; path=/";
console.log(document.cookie);  // read all cookies

Types: Session cookies (deleted on browser close), Persistent cookies (with expiry), Secure (HTTPS only), HttpOnly (no JavaScript access — prevents XSS theft).

Q3What is code-viruses? Briefly explain the working process of code-virus.5 Marks
✓ Answer

A code virus is malicious software that attaches its code to legitimate executable programs. When the infected program runs, the virus code executes.

Working Process

  1. Infection: Virus injects its code into a host executable (.exe, .dll) or boot sector.
  2. Dormancy: Virus may remain hidden until a trigger condition (date, key press, program count).
  3. Activation: User runs the infected program — virus code executes first.
  4. Replication: Virus searches for other executables to infect, spreading to new files.
  5. Payload delivery: Destructive action — delete files, corrupt data, display messages, steal data, encrypt files (ransomware).

Types: File infectors, Boot sector viruses, Macro viruses (Word/Excel), Polymorphic viruses (change signature to evade antivirus), Worms (self-replicating without host).

Q5What is CGI? Provide an example of CGI script.5 Marks
✓ Answer

CGI (Common Gateway Interface) is a standard protocol that allows a web server to execute external programs (scripts) and return their output as dynamic web content to the browser.

Browser → HTTP Request → Web Server → CGI Script (Perl/Python)
                              ↑              │
                              └──── Output ──┘
                         (HTML sent back to browser)
#!/usr/bin/perl
# File: /cgi-bin/hello.pl
use CGI qw(:standard);
use strict;

print header("text/html");          # HTTP header
print start_html("CGI Demo");
print h1("Hello, World from CGI!");

my $name = param("name") || "Guest";
print p("Welcome, ", b($name), "!");

print end_html();

<!-- HTML form that calls the CGI script -->
<form action="/cgi-bin/hello.pl" method="get">
  <label>Your name: <input type="text" name="name"></label>
  <input type="submit" value="Greet Me">
</form>

CGI is largely replaced by modern frameworks (Node.js, PHP, Python Flask/Django) but the concept of server-side script generating dynamic HTML remains fundamental.

Q6Explain Java Socket. Provide an appropriate example to demonstrate the Java Socket.5 Marks
✓ Answer

A Java Socket is an endpoint for two-way communication between programs over a TCP/IP network. ServerSocket listens for connections; Socket connects client to server.

Server Side

import java.net.*;
import java.io.*;

public class EchoServer {
    public static void main(String[] args) throws Exception {
        ServerSocket server = new ServerSocket(5000);
        System.out.println("Server listening on port 5000...");

        Socket client = server.accept();  // blocks until client connects
        System.out.println("Client connected: " + client.getInetAddress());

        BufferedReader in  = new BufferedReader(
            new InputStreamReader(client.getInputStream()));
        PrintWriter out = new PrintWriter(client.getOutputStream(), true);

        String msg = in.readLine();
        System.out.println("Received: " + msg);
        out.println("Echo: " + msg);   // send response

        client.close();
        server.close();
    }
}

Client Side

import java.net.*;
import java.io.*;

public class EchoClient {
    public static void main(String[] args) throws Exception {
        Socket socket = new Socket("localhost", 5000);

        PrintWriter out = new PrintWriter(socket.getOutputStream(), true);
        BufferedReader in = new BufferedReader(
            new InputStreamReader(socket.getInputStream()));

        out.println("Hello Server!");
        System.out.println("Server says: " + in.readLine());

        socket.close();
    }
}

Flow: Server binds to port → Client connects → TCP 3-way handshake → Data exchanged via streams → Connection closed.

Q8(a) What is code-viruses? (b) What is Trojan horses? (c) What is spoofing? [5+5+5]15 Marks
✓ Answer

(a) Code-Viruses [5 marks]

A virus is malicious code that attaches to legitimate programs and replicates when the host program is executed.

  • File infector: Attaches to .exe, .com files.
  • Boot sector virus: Infects the Master Boot Record (MBR).
  • Macro virus: Written in VBA — infects Word/Excel documents.
  • Polymorphic virus: Changes its code signature on each infection to evade antivirus.

Lifecycle: Infection → Dormancy → Activation → Replication → Payload. Requires user action (running infected file) to spread.

(b) Trojan Horses [5 marks]

A Trojan horse disguises itself as legitimate, useful software to trick users into installing it. Unlike viruses, Trojans do not self-replicate.

TypeBehaviourExample
Backdoor TrojanOpens hidden remote access portAttacker controls PC remotely
Downloader TrojanDownloads and installs other malwareInstalls ransomware after entry
Banking TrojanSteals banking credentialsZeus, Emotet
Ransom TrojanEncrypts files, demands paymentWannaCry variant
Spy TrojanRecords keystrokes, screenshotsKeylogger disguised as game

Named after the Greek Trojan Horse — appears harmless but contains hidden threat inside.

(c) Spoofing [5 marks]

Spoofing is impersonating a trusted entity (device, user, IP, email) to gain unauthorised access or deceive victims.

TypeWhat is fakedAttack goal
IP SpoofingSource IP address in packet headerBypass IP-based access control, DDoS amplification
Email SpoofingFrom: address in email headerPhishing — trick user into clicking malicious link
DNS SpoofingDNS response (cache poisoning)Redirect users to fake website
ARP SpoofingMAC address in ARP replyMan-in-the-middle on local network
Caller ID SpoofingPhone number displaySocial engineering phone scams
Website SpoofingFake website mimicking real oneSteal login credentials (phishing)

Prevention: SPF/DKIM for email, HTTPS + certificate validation, ARP inspection, DNSSEC, firewall rules, user awareness training.

Q9(a) What is POP3? (b) What is SMTP? (c) What is HTTP? [5+5+5]15 Marks
✓ Answer

(a) POP3 — Post Office Protocol v3 [5 marks]

Application-layer protocol for retrieving email from a mail server to a local client. Port 110 (995 with SSL/TLS).

Client                          Mail Server
  │── Connect port 110 ──────────────────────►│
  │── USER alice@example.com ────────────────►│
  │── PASS secret ───────────────────────────►│
  │── STAT ──────────────────────────────────►│  (message count)
  │── LIST ──────────────────────────────────►│  (list messages)
  │── RETR 1 ──────────────────────────────►│  (download msg 1)
  │── DELE 1 ──────────────────────────────►│  (mark for delete)
  │── QUIT ──────────────────────────────────►│  (commit deletes)
  • Downloads email to local device; typically deletes from server.
  • Simple, works offline after download.
  • Single-device oriented — contrast with IMAP (keeps mail on server, multi-device sync).

(b) SMTP — Simple Mail Transfer Protocol [5 marks]

Application-layer protocol for sending email between mail servers and from client to server. Port 25 (587 for submission, 465 for SMTPS).

Sender MUA → Sender MTA → Recipient MTA → Recipient MDA → Recipient MUA

SMTP Session:
  Client: EHLO mail.example.com
  Server: 250 Hello
  Client: MAIL FROM:<alice@example.com>
  Server: 250 OK
  Client: RCPT TO:<bob@other.com>
  Server: 250 OK
  Client: DATA
  Server: 354 Start mail input
  Client: Subject: Hello\n\nMessage body.\n.
  Server: 250 Message accepted
  Client: QUIT
  • Push protocol — only sends, never retrieves mail.
  • Uses MIME extension for attachments and non-ASCII content.
  • MTA (Mail Transfer Agent) routes mail using DNS MX records.

(c) HTTP — HyperText Transfer Protocol [5 marks]

Application-layer protocol for the World Wide Web. Stateless request-response model. Port 80 (443 for HTTPS).

GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...</html>
MethodPurpose
GETRetrieve resource
POSTSubmit data / create resource
PUTUpdate/replace resource
DELETERemove resource
HEADGET without body (headers only)

Status codes: 1xx Informational, 2xx Success (200 OK), 3xx Redirect (301), 4xx Client Error (404 Not Found), 5xx Server Error (500).

Relationship: SMTP sends email; POP3/IMAP retrieves email; HTTP serves web pages. All three are application-layer protocols.

(b) World Wide Web [5 marks]

The World Wide Web (WWW) is an information system built on top of the internet where documents and resources are identified by URLs, linked by hyperlinks, and accessed via web browsers using HTTP/HTTPS.

  • Invented: 1989 by Tim Berners-Lee at CERN.
  • Components: Web browsers (client), web servers (host content), HTTP protocol (communication), HTML (content format), URLs (addressing).
  • Hypertext: Documents linked to each other — click a link to jump to another page anywhere in the world.
  • Not the same as the Internet: Internet is the physical/network infrastructure; WWW is a service running on it.
  • Impact: Revolutionised communication, commerce, education, entertainment, and information access globally.

(c) What is a Network? [5 marks]

A network is a collection of interconnected devices (computers, servers, routers, switches, printers) that communicate and share resources using communication protocols.

Types of Networks

TypeRangeExample
LAN (Local Area Network)Building/campusOffice Wi-Fi, college lab
MAN (Metropolitan Area Network)CityCity-wide cable network
WAN (Wide Area Network)Country/globalThe Internet
PAN (Personal Area Network)Personal devicesBluetooth headset to phone

Network Components

  • Nodes: End devices — PCs, phones, servers.
  • Links: Wired (Ethernet, fiber) or wireless (Wi-Fi, cellular).
  • Routers: Forward packets between networks.
  • Switches: Connect devices within a LAN.
  • Protocols: TCP/IP, HTTP, DNS — rules for communication.
