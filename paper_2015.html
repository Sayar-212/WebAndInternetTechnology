Q1(i) A web server by default runs on port. 1 Mark
A) 23
B) 25
C) 80
D) 22
✓ Answer: (C) 80

Web servers (HTTP) run on port 80 by default. HTTPS uses port 443. Other ports: 23 = Telnet, 25 = SMTP, 22 = SSH.

Q1(ii) Which of the following methods is used to get information of a resource? 1 Mark
A) TOP
B) HEAD
C) GET
D) GRAB
✓ Answer: (C) GET

The HTTP GET method requests a representation of a resource without modifying it. It is idempotent and the most common HTTP method. HEAD is similar but retrieves only the headers, not the body.

Q1(iii) DHTML is for 1 Mark
A) Model
B) View
C) Controller
D) Servlet
✓ Answer: (B) View

DHTML (Dynamic HTML) is a combination of HTML, CSS, JavaScript, and DOM used to create dynamic, interactive web pages. It handles the presentation/view layer — changing what the user sees without reloading the page.

Q1(iv) HTTP status code is 1 Mark
A) Two-digit code
B) Three-digit code
C) Four-digit code
D) Five-digit code
✓ Answer: (B) Three-digit code

HTTP status codes are 3-digit numbers: 1xx (Informational), 2xx (Success), 3xx (Redirection), 4xx (Client Error), 5xx (Server Error). Examples: 200 OK, 404 Not Found, 500 Internal Server Error.

Q1(v) A remote object must be an instance of 1 Mark
A) java.rmi.RemoteObject
B) java.io.Serializable
C) java.lang.Cloneable
D) java.rmi.Remote
✓ Answer: (D) java.rmi.Remote

Any object that is to be accessed remotely via RMI must implement the java.rmi.Remote interface. This is a marker interface — it has no methods but signals to the RMI runtime that this object can be invoked remotely. All methods in the interface must declare throws RemoteException.

D) FTP
✓ Answer: (B) UDP

UDP (User Datagram Protocol) operates at the Transport Layer. TCP is also a transport layer protocol. HTTP, SMTP, and FTP are Application Layer protocols that use TCP/UDP as their underlying transport.

Q3 a) What is a firewall?    b) Briefly explain any two types of firewall. 1+4 = 5
✓ Answer

a) Firewall (1 mark)

A firewall is a network security device (hardware or software) that monitors and controls incoming and outgoing network traffic based on a predefined set of security rules. It creates a barrier between a trusted internal network and untrusted external networks (like the internet).

b) Two Types of Firewall (4 marks)

1. Packet Filtering Firewall

  • Operates at the Network Layer (Layer 3).
  • Inspects each packet individually based on: source/destination IP, protocol, source/destination port.
  • Uses Access Control Lists (ACLs) to allow or deny packets.
  • Pros: Fast, low overhead, transparent to users.
  • Cons: Stateless — cannot track connection context. Vulnerable to IP spoofing. Cannot inspect payload content.
Rule Table Example:
Src IP       | Protocol | Dst Port | Action
-------------|----------|----------|-------
10.0.0.0/24  | TCP      | 80       | ALLOW
10.0.0.0/24  | TCP      | 443      | ALLOW
ANY          | ANY      | ANY      | DENY

2. Stateful Inspection Firewall (Dynamic Packet Filtering)

  • Operates at Layer 3 and Layer 4 (Network + Transport).
  • Maintains a state table tracking the state of active connections (NEW, ESTABLISHED, RELATED).
  • Only allows packets that are part of an established, legitimate connection.
  • Pros: Much more secure than simple packet filtering. Can detect and block SYN flood attacks.
  • Cons: Slower than packet filtering. Cannot inspect application-layer content.
Q4 a) What is network congestion?    b) Describe congestion control strategies. 2+3 = 5
✓ Answer

a) Network Congestion (2 marks)

Network congestion occurs when the volume of data (packets) injected into a network exceeds the network's capacity to handle it. Router buffers overflow, packets are dropped, retransmissions increase, and network performance degrades drastically.

  • Symptoms: High latency, packet loss, reduced throughput.
  • Cause: Too many senders sending too much data too fast; bottleneck links; slow routers.

b) Congestion Control Strategies (3 marks)

1. Slow Start (TCP)

  • TCP starts with a small congestion window (cwnd = 1 MSS).
  • Doubles cwnd every RTT (exponential growth) until ssthresh (slow start threshold) is reached.

2. Congestion Avoidance

  • After reaching ssthresh, cwnd increases by 1 MSS per RTT (linear growth) instead of doubling.
  • Prevents the network from being overwhelmed.

3. Fast Retransmit & Fast Recovery

  • If 3 duplicate ACKs received → retransmit lost segment without waiting for timeout.
  • ssthresh = cwnd/2, cwnd = ssthresh (halved, not reset to 1).

4. Timeout Detection

  • On timeout: ssthresh = cwnd/2, cwnd reset to 1 → restart slow start.

Q6 Describe different types of TCP Timers. 5 Marks
✓ Answer

TCP uses four main timers to manage reliable transmission and connection state:

1. Retransmission Timer (RTO)

  • Started when a segment is sent.
  • If no ACK received before timer expires → segment is retransmitted.
  • RTO is dynamically calculated using RTT (Round-Trip Time) measurements: RTO = SRTT + 4 × RTTVAR
  • Uses exponential backoff — doubles on each timeout to avoid flooding a congested network.

2. Persistence Timer

  • Prevents a deadlock situation when the receiver advertises a zero window (buffer full).
  • When receiver sends Window Size = 0, sender starts persistence timer.
  • When it expires, sender transmits a Window Probe segment (1 byte) to ask if window has opened.
  • Receiver responds with current window size.

3. Keepalive Timer

  • Detects idle/dead connections.
  • If no data exchanged for 2 hours (default), sender transmits a keepalive probe.
  • If no response after several probes → connection is considered dead and closed.
  • Prevents half-open connections from consuming server resources indefinitely.

4. Time-Wait Timer (2MSL Timer)

  • Started when a connection enters the TIME_WAIT state (after sending the final ACK in connection termination).
  • Duration: 2 × MSL (Maximum Segment Lifetime, typically 2 minutes → timer = 4 minutes).
  • Purpose: Ensures any delayed duplicate packets from the old connection are discarded before a new connection with the same socket pair can be established.

Types of XML Parsers (4 marks)

1. DOM Parser (Document Object Model)

  • Loads the entire XML document into memory as a tree structure.
  • Allows random access — navigate to any node, forward or backward.
  • Supports reading AND writing/modifying the document.
  • Con: High memory usage for large files.

2. SAX Parser (Simple API for XML)

  • Event-driven, streaming parser — processes XML node by node.
  • Does not load the entire document into memory.
  • Fires events (startElement, endElement, characters) as it reads.
  • Con: Read-only, forward-only, more complex to code.

3. StAX Parser (Streaming API for XML)

  • Pull-based streaming parser — application pulls events rather than being pushed.
  • More efficient than SAX for selective processing.

Q9 a) Basic principles of security + Active vs Passive attack   b) Virus vs Worm + IP Sniffing + IP Spoofing + DoS 4+4+2+2+3 = 15
✓ Answer

a) Basic Principles of Security (4 marks)

  • Confidentiality: Information is accessible only to those authorised to access it. Achieved through encryption (AES, RSA).
  • Integrity: Data has not been altered during transmission or storage. Achieved via hashing (MD5, SHA-256) and digital signatures.
  • Availability: Systems and data are accessible to authorised users when needed. Protected by redundancy, backups, DDoS mitigation.
  • Authentication: Verifying the identity of a user or system. Achieved via passwords, certificates, biometrics, MFA.
  • Non-repudiation: Ensuring a sender cannot deny sending a message. Achieved via digital signatures.

IP Sniffing (2 marks)

IP Sniffing (Packet Sniffing) is a passive attack where an attacker uses software (a sniffer) to capture and inspect network packets passing through a network segment. Tools: Wireshark, tcpdump. Used to steal passwords, session tokens, or sensitive data transmitted in plaintext. Defence: Use encryption (HTTPS, TLS).

DoS — Denial of Service (3 marks)

A DoS attack floods a target server/network with traffic, exhausting its resources (CPU, memory, bandwidth) and making it unavailable to legitimate users.

  • SYN Flood: Send many TCP SYN packets without completing the 3-way handshake → server fills up with half-open connections.
  • UDP Flood: Send UDP packets to random ports → server responds with "port unreachable" packets until overwhelmed.
  • HTTP Flood: Send large numbers of HTTP GET/POST requests to exhaust web server resources.
  • DDoS: Coordinated DoS from thousands of machines (botnet) — much harder to stop.
  • Defence: Rate limiting, traffic filtering, CDN, cloud-based DDoS mitigation (Cloudflare).

c) NVT — What and Why? (1+2 marks)

NVT (Network Virtual Terminal) is a conceptual terminal defined by the Telnet protocol that represents a universal, bidirectional character device with a standard character set (7-bit ASCII).

Why NVTs are required:

  • Different computers and operating systems use different terminal types, character sets, and control codes (e.g., Unix vs Windows vs IBM mainframe).
  • NVT acts as an intermediary standard — the client translates its local terminal data into NVT format before sending; the server translates NVT into its own local format.
  • This ensures interoperability — any machine can Telnet into any other machine regardless of their native terminal implementations.
Q11 Short Notes (any 3): a) Web Crawler   b) RTP   c) NAT   d) SSL   e) SMTP 3 × 5 = 15
✓ Answer

a) Web Crawler (5 marks)

A web crawler (spider/bot) is an automated program that systematically browses the web to discover and index web pages for search engines.

Algorithm:

1. Start with a list of seed URLs
2. For each URL in queue:
   a. Download the HTML page
   b. Parse and extract all hyperlinks (<a href="...">)
   c. Add unvisited links to the queue
   d. Store/index the page content
3. Repeat until queue is empty or limit reached
4. Respects robots.txt (pages that opt out of crawling)

Used by Google (Googlebot), Bing (Bingbot). Challenges: duplicate detection, JavaScript-rendered pages, rate limiting (avoid overloading servers), deep web (pages behind logins).

b) RTP — Real-time Transport Protocol (5 marks)

RTP is an application-layer protocol (RFC 3550) designed for delivering real-time audio and video over IP networks. It runs over UDP for low latency.

RTP Header Fields:

 ┌────┬───┬───┬──────┬───┬──────────────────────────────────────┐
 │ V  │ P │ X │ CC   │ M │  Payload Type (PT)                   │
 ├────┴───┴───┴──────┴───┴──────────────────────────────────────┤
 │              Sequence Number (16 bits)                        │
 ├───────────────────────────────────────────────────────────────┤
 │              Timestamp (32 bits)                              │
 ├───────────────────────────────────────────────────────────────┤
 │              SSRC — Synchronization Source (32 bits)         │
 └───────────────────────────────────────────────────────────────┘
  • Sequence Number: Orders packets at receiver; detects loss.
  • Timestamp: Synchronizes audio/video; calculates jitter.
  • PT (Payload Type): Identifies the codec (G.711, H.264, VP8).
  • SSRC: Unique identifier for each RTP stream source.
  • RTP is accompanied by RTCP which provides QoS reports (packet loss, jitter, delay).

c) NAT — Network Address Translation (5 marks)

NAT is a technique where a router modifies IP addresses in packet headers while they are in transit — mapping private IP addresses to a public IP address.

Types of NAT:

  • Static NAT: One private IP ↔ one public IP (permanent mapping).
  • Dynamic NAT: Pool of public IPs dynamically assigned to private IPs.
  • PAT/NAT Overload: Many private IPs share one public IP, differentiated by port numbers. Most common in home routers.

192.168.1.2:1234  ↔  203.0.113.1:4001  ↔  8.8.8.8:53
192.168.1.3:5678  ↔  203.0.113.1:4002  ↔  8.8.8.8:80

Benefits: Conserves IPv4 addresses; hides internal network topology; provides firewall-like security.
Limitation: Breaks end-to-end connectivity; complicates some protocols (FTP, SIP).

d) SSL — Secure Sockets Layer (5 marks)

SSL (Secure Sockets Layer), now superseded by TLS (Transport Layer Security), provides encrypted communication between client and server. It sits between the Application layer and Transport layer.

SSL Handshake Process:

Client                              Server
  │── ClientHello (cipher suites) ──►│
  │◄── ServerHello (chosen cipher) ──│
  │◄── Server Certificate (pub key) ─│
  │── ClientKeyExchange (session key)►│
  │── ChangeCipherSpec ──────────────►│
  │◄── ChangeCipherSpec ──────────────│
  │════ Encrypted Data Exchange ══════│
  • Uses asymmetric encryption (RSA) for key exchange, symmetric (AES) for data.
  • Provides: Confidentiality (encryption), Integrity (MAC), Authentication (certificates).
  • HTTPS = HTTP over TLS/SSL.

e) SMTP — Simple Mail Transfer Protocol (5 marks)

SMTP is the standard application-layer protocol for sending email messages between mail servers. Port: 25 (server-to-server), 587 (client submission), 465 (SMTPS).

SMTP Commands:

EHLO   sender.com          → Identify sending server
MAIL FROM: <alice@a.com>  → Specify sender
RCPT TO:  <bob@b.com>     → Specify recipient
DATA                        → Begin message body
Subject: Hello Bob

Message content here.
.                           → Single dot = end of message
QUIT                        → Close connection

Response Codes: 220 (ready), 250 (OK), 354 (start input), 550 (user not found), 421 (server unavailable).
Limitation: SMTP only handles 7-bit ASCII — MIME extends it for attachments and non-ASCII text.
