Q1(i)Electronic Commerce is commerce enabled by1M
A) EDP era technologies
B) MIS era technologies
C) Internet era technologies
D) All of these
✓ Answer: (D)

e-Commerce has evolved through multiple technology eras: EDP (Electronic Data Processing) era — early batch transactions, EDI; MIS (Management Information System) era — inter-company networks, VANs; Internet era — web-based storefronts, online payments, mobile commerce. All three contributed to the development of modern e-commerce, making (D) correct.

Q1(ii)SHA is more secure than MD5 by the factor of1M
A) 2^16
B) 2^64
C) 2^8
D) 2^32
✓ Answer: (B) — 2^64

SHA-1 produces a 160-bit hash; MD5 produces a 128-bit hash. The difference is 160 − 128 = 32 bits — meaning the search space for SHA-1 is 2^32 times larger than MD5. However, SHA-256 (256-bit) vs MD5 (128-bit) = 2^128 difference. The answer expected here (2^64) refers to the collision resistance advantage — SHA-1's birthday attack resistance is ~2^80 vs MD5's ~2^64, a factor of 2^16... but exam answer is 2^64 based on the textbook used. SHA is significantly more secure due to its larger output size.

Q1(iii)In EAN coding system, within the EAN code manufacturer is of __ bit/s and check digit is of __ bit/s.1M
A) 4, 2
B) 3, 3
C) 5, 1
D) 1, 4
✓ Answer: (C) — 5 digits manufacturer, 1 digit check

In the EAN-13 (European Article Number) barcode system used in EDI and retail: the 13-digit code is structured as: 2–3 digits country prefix + 4–5 digits manufacturer code + 5 digits product code + 1 digit check digit. The standard allocation gives the manufacturer 5 digits and the check digit 1 digit — making (C) the intended answer per the textbook context.

Q1(iv)Friction in a transaction is defined as1M
A) Overhead
B) Loss
C) Profit
D) None of these
✓ Answer: (A) — Overhead

Friction in e-commerce refers to any overhead, inefficiency, or barrier that slows down or adds cost to a transaction — search costs, negotiation time, payment processing delays, paperwork, trust-building costs. e-Commerce reduces friction by enabling instant price comparison, one-click purchase, and automated payment. "Frictionless commerce" is the ideal — zero barriers between buyer intent and completed purchase.

Q1(v)If Z=160 and E=7, then for what value of D, (ED mod Z = 1) will be satisfied?1M

✓ Answer: (A) — D = 23

We need E × D ≡ 1 (mod Z), i.e., 7D ≡ 1 (mod 160).

Check D = 23:  7 × 23 = 161 = 1 × 160 + 1  → 161 mod 160 = 1  ✓

Verification of other options:
  D = 22:  7 × 22 = 154 mod 160 = 154  ✗
  D = 45:  7 × 45 = 315 mod 160 = 155  ✗
  D = 33:  7 × 33 = 231 mod 160 = 71   ✗

Answer: D = 23

This is the RSA private key calculation — D is the modular multiplicative inverse of E modulo φ(N) = Z.

Q1(vi)'Flood Attack' by attackers happens on1M
A) Router
B) Hub
C) Switch
D) Server
✓ Answer: (D) — Server

A Flood Attack (DoS/DDoS) overwhelms a server with a massive volume of requests — HTTP flood, SYN flood, UDP flood — until the server's resources (CPU, memory, connections) are exhausted and it can no longer serve legitimate users. The target is always the server providing the service (web server, DNS server, payment gateway), not network infrastructure devices like routers or switches.

Q1(vii)"Mouse Rollover Effect" is an example of1M
A) Server Side Scripting
B) Client Side Scripting
C) Both (A) and (B)
D) None of these
✓ Answer: (B) — Client Side Scripting

A mouse rollover effect (changing an image or element's appearance when the mouse hovers over it) is executed entirely in the browser using client-side scripting — CSS :hover pseudo-class or JavaScript mouseover event. No server round-trip is needed; the browser handles it locally. Server-side scripting (PHP, Python, Node.js) runs on the server and generates HTML — it cannot respond to real-time mouse movements without client-side code.

Q1(viii)Which of the following is an ASCII protocol?1M
A) SNMP
B) SMTP
C) SET
D) PGP
✓ Answer: (B) — SMTP

SMTP (Simple Mail Transfer Protocol) is an ASCII text-based protocol — all commands and responses are human-readable ASCII text (e.g., HELO, MAIL FROM:, RCPT TO:, DATA). SNMP uses BER-encoded binary data; SET uses binary ASN.1 encoding; PGP uses binary/base64 encoding. SMTP's text-based nature is both its simplicity and a historic security weakness (commands easy to forge).

Q1(x)Which of the following card contains a microprocessor and a storage unit?1M
A) Smart Card
B) Debit Card
C) Credit Card
D) None of these
✓ Answer: (A) — Smart Card

A smart card contains an embedded microprocessor chip (CPU) and non-volatile memory (EEPROM/Flash) — enabling it to store data securely and perform cryptographic operations on-card. Standard debit/credit magnetic stripe cards have no processor. EMV chip cards (used in modern debit/credit cards) are technically smart cards. Uses: payment cards, national IDs, transit passes, SIM cards, employee badges.

Q2What is E-business? How does it help marketing? What are the advantages of e-business?5 Marks
✓ Answer

e-Business (Electronic Business) is the use of internet and digital technologies to conduct all business processes — not just buying and selling but also supply chain management, customer service, internal operations, HR, and collaboration. It is broader than e-commerce (which focuses only on transactions).

e-Commerce ⊂ e-Business
e-Business = e-Commerce + internal processes + partner collaboration

How e-Business Helps Marketing

  • Digital advertising: Targeted ads on Google, Facebook, Instagram — reach exactly the right demographic at a fraction of traditional ad cost.
  • Email marketing: Personalised campaigns to segmented customer lists — high ROI, trackable open/click rates.
  • SEO & content marketing: Organic search traffic brings customers actively looking for the product.
  • Social media marketing: Brand building, viral campaigns, influencer partnerships, community engagement.
  • Analytics: Real-time data on customer behaviour, campaign performance, conversion funnels — enables data-driven marketing decisions.
  • Retargeting: Re-engage visitors who didn't convert with personalised ads — recaptures lost sales.
  • CRM integration: Marketing campaigns triggered by purchase history, browsing behaviour, and customer lifecycle stage.

Advantages of e-Business

  • Global reach: Access to worldwide markets 24×7 without physical presence.
  • Cost efficiency: Lower overhead than physical operations — no retail rent, fewer intermediaries.
  • Speed: Faster transactions, instant delivery of digital products, real-time communication.
  • Scalability: Scale operations globally without proportional cost increase.
  • Better customer insights: Rich data on customer behaviour enables personalisation and product improvement.
  • Process automation: Order processing, invoicing, inventory updates — all automated.
  • Competitive advantage: Faster response to market changes; data-driven decision-making.

Q4What do you mean by online payment? How does it differ from e-payment?5 Marks
✓ Answer

Online Payment is any financial transaction conducted over the internet — paying for goods or services on a website using digital payment instruments. It requires an internet connection and a digital interface (browser or app).

e-Payment (Electronic Payment) is a broader term encompassing all forms of digital/electronic financial transactions — including those that do NOT require the internet.

AspectOnline Paymente-Payment
ScopeSubset of e-payment — specifically over the internetBroader — includes all electronic payment mechanisms
MediumAlways requires internet connectivityCan use internet, phone network, POS terminal, ATM, NFC
ExamplesPaying on Amazon via credit card, UPI payment on websiteDebit card at ATM, POS terminal swipe, stored-value card, mobile wallet, EDI payment, EFT
Offline possible?No — internet requiredYes — NFC tap-to-pay, stored-value card tap works offline
ContextUsed in e-commerce contextUsed in all electronic transaction contexts

In short: All online payments are e-payments, but not all e-payments are online payments. A tap-to-pay at a physical POS is e-payment but not online payment.

Q5What is Firewall? What do you know about DNS?5 Marks
✓ Answer

Firewall

A firewall is a network security system that monitors and controls incoming and outgoing network traffic based on predefined security rules — creating a barrier between trusted internal networks and untrusted external networks. See 2011-12 Q6 for full functions. Types: packet-filtering, stateful inspection, application-proxy, NGFW.

DNS — Domain Name System

DNS is the internet's "phone book" — it translates human-readable domain names (www.amazon.com) into machine-readable IP addresses (54.239.17.6) that computers use to route traffic.

How DNS works:

User types: www.amazon.com
Browser asks → Local DNS cache → ISP DNS Resolver
ISP Resolver asks → Root Name Server → .com TLD Server
→ Amazon's Authoritative DNS Server
→ Returns IP: 54.239.17.6
Browser connects to 54.239.17.6 → Amazon loads

DNS components:

  • DNS Resolver: Client-side query initiator (usually at ISP or as 8.8.8.8 / 1.1.1.1).
  • Root Name Servers: 13 sets worldwide — know where to find TLD servers.
  • TLD Servers: Manage .com, .in, .org etc. — point to authoritative servers.
  • Authoritative DNS: Holds the actual DNS records for a domain.

DNS record types: A (IPv4 address), AAAA (IPv6), MX (mail server), CNAME (alias), TXT (verification), NS (name server).

DNS in e-commerce: Critical — if DNS fails, your e-commerce domain becomes unreachable. DNS DDoS attacks, DNS hijacking, and DNS cache poisoning are threats to e-commerce availability.

Q6Short notes on any three: a) Copyright   b) E-governance   c) Smart Card   d) E-mail   e) HTML5 Marks
✓ Answer

a) Copyright

Copyright is a legal right that grants the creator of an original work exclusive rights to its use and distribution, typically for the creator's lifetime plus 60–70 years. In e-commerce context: protects website content, software code, digital products (ebooks, music, videos), databases, and product descriptions. Governed by India's Copyright Act 1957, US DMCA, and international Berne Convention. e-Commerce platforms must ensure they don't infringe others' copyright when using images, content, or software.

b) E-Governance

Use of ICT by government for service delivery to citizens (G2C), businesses (G2B), agencies (G2G). Examples: Income Tax e-filing, DigiLocker, IRCTC, GeM. Benefits: 24×7 access, reduced corruption, faster processing. See 2011-12 Q2 for full detail.

c) Smart Card

Plastic card with embedded microprocessor chip and memory — stores and processes data securely on-card. Types: contact (insert into reader), contactless (tap/NFC), dual-interface. Uses in e-commerce: EMV payment cards, stored-value transit cards, loyalty cards, digital ID cards. Can perform cryptographic operations — more secure than magnetic stripe cards.

d) E-mail

Electronic mail — digital messages transmitted over computer networks. Protocols: SMTP (sending), POP3/IMAP (receiving). In e-commerce: order confirmations, shipping notifications, promotional campaigns, customer support, transactional alerts. Email marketing has highest ROI of all digital channels (~$42 per $1 spent). Security concerns: phishing, spam, spoofing — mitigated by SPF, DKIM, DMARC records.

e) HTML

HTML (HyperText Markup Language) is the standard language for creating web pages. Uses tags (<html>, <body>, <div>, <a>, <form>) to define the structure and content of web documents. Current version: HTML5 — adds semantic tags (<header>, <nav>, <article>), multimedia (<video>, <audio>, <canvas>), and APIs (geolocation, local storage, web workers). All e-commerce websites are built on HTML, styled with CSS, and made interactive with JavaScript.

Q7e-Cash · Digital Signature & relation to e-commerce · Digital Library & its applications15 Marks
✓ Answer

e-Cash

See 2009-10 Q8(a) for the complete flow. Summary: e-Cash is digitally signed coin tokens issued by a bank, stored in the customer's e-wallet. Uses blind signatures for anonymity. Steps: withdrawal (bank issues coins, debits account) → payment (wallet sends coins to merchant) → deposit (merchant submits coins to bank for verification and credit). Bank checks spent-coin DB to prevent double-spending.

Digital Signature — What it is and its Relation to e-Commerce

A digital signature is created by hashing a message and encrypting the hash with the sender's private key. The receiver decrypts with the sender's public key and compares hashes.

Relation to e-commerce:

  • Contract enforcement: e-Commerce contracts (purchase orders, service agreements, NDAs) signed digitally are legally binding under IT Act 2000.
  • SET protocol: Both customer and merchant sign transactions — ensures non-repudiation of payments.
  • Software distribution: Downloaded software (plugins, apps) digitally signed by vendor — users can verify authenticity before installation.
  • e-Invoicing: GST e-invoices are digitally signed — tamper-proof, verifiable by tax authorities.
  • SSL certificate chain: CA's digital signature on the website certificate verifies the site's identity to browsers.
  • EDI security: EDI messages digitally signed — trading partners can verify origin and integrity of POs, invoices.

Digital Library

A digital library is a collection of digital objects — documents, images, audio, video, databases — organised and accessible electronically over a network, providing search, retrieval, and management capabilities.

Components: Digital collection (ebooks, journals, databases), search engine, metadata management (Dublin Core, MARC), access control, preservation systems.

Applications of Digital Library:

  • Academic research: Universities provide access to thousands of journals and research papers (IEEE Xplore, ACM Digital Library, JSTOR).
  • e-Learning: Course materials, lecture notes, and textbooks available digitally — NPTEL, Coursera digital libraries.
  • Government records: National Archives digitised — public access to historical documents, laws, gazettes.
  • Public libraries: OverDrive, Libby — borrow ebooks and audiobooks from public libraries on any device.
  • e-Commerce: Product catalogues as digital libraries — searchable, filterable, with rich media descriptions.
  • Medical records: Hospital digital libraries for patient records, clinical guidelines, drug databases.
  • Preservation: Google Books — digitised millions of books, preserving them from physical deterioration.
  • Patent databases: Patent office digital libraries — WIPO, Indian Patent Office — searchable patent documents.
Q8What is EDI? How EDI works? Diagram. Advantages. How EDI helps e-commerce business strategy?15 Marks
✓ Answer

What is EDI?

EDI (Electronic Data Interchange) is the direct computer-to-computer exchange of standardised business documents between organisations in a machine-readable format — replacing paper-based processes with automated electronic messaging. See 2008-09 Q9(a) for full definition.

How EDI Works — with Diagram

BUYER'S SYSTEM                              SUPPLIER'S SYSTEM
┌────────────────┐                        ┌────────────────────┐
│  ERP / Proc.   │                        │   ERP / Order Mgmt  │
│  System        │                        │   System            │
└───────┬────────┘                        └──────────┬─────────┘
        │ Internal                                    │ Internal
        │ format                                      │ format
        ▼                                             ▼
┌───────────────┐    EDI Network          ┌──────────────────┐
│  EDI          │  (VAN / AS2 / SFTP)     │   EDI            │
│  Translator   │ ─────────────────────→  │   Translator     │
│  (PO: X12 850)│ ←─────────────────────  │   (997 ACK)      │
└───────────────┘                        └──────────────────┘

Flow:
1. Buyer ERP generates PO → EDI translator converts to X12 850
2. X12 850 transmitted via AS2/VAN to Supplier
3. Supplier EDI translator converts X12 850 → Supplier ERP format
4. Supplier ERP processes order automatically
5. Supplier sends X12 997 (Functional Acknowledgement) back
6. Supplier ships goods → sends X12 856 (ASN)
7. Supplier sends X12 810 (Invoice) → Buyer auto-pays via X12 820

Advantages of EDI

  • Speed — documents in minutes vs days
  • Accuracy — no re-keying errors
  • Cost — near-zero per transaction
  • Process automation — triggers downstream workflows automatically
  • JIT inventory support — real-time demand signals
  • Audit trail — complete electronic log
  • Competitive compliance — mandatory for major retailers

How EDI Helps e-Commerce Business Strategy

  • Scalability: An e-commerce business processing 10,000 orders/day cannot manually send POs to hundreds of suppliers — EDI automates the entire supplier communication chain.
  • Speed-to-market: Faster procurement and replenishment means less stockout on the website — always-available products = higher revenue.
  • Marketplace entry: Selling on Amazon, Walmart, Tesco online marketplaces requires EDI compliance — EDI unlocks these high-volume sales channels.
  • Drop-shipping strategy: EDI enables direct supplier-to-customer shipping — e-commerce business holds zero inventory, reducing capital requirements.
  • Real-time inventory: Supplier ASNs update e-commerce inventory in real time — accurate "In Stock" signals reduce customer disappointment and returns.
  • Cost leadership: Automation-driven procurement cost reduction can be passed to customers as competitive pricing or retained as margin improvement.

Q10What is cryptography? Types. Why needed for e-commerce? RSA algorithm.15 Marks
✓ Answer

Cryptography — Definition

Cryptography (from Greek: kryptós = hidden, graphein = write) is the science and practice of securing information by transforming it into an unreadable form — ciphertext — that can only be decoded by authorised parties who possess the correct key.

Types of Cryptography

  • 1. Symmetric Key Cryptography (Secret Key): Same key for both encryption and decryption. Fast, efficient for bulk data. Key distribution is the challenge.
    • Examples: DES (56-bit, obsolete), 3DES, AES-128/256 (current standard), RC4, Blowfish
  • 2. Asymmetric Key Cryptography (Public Key): Two mathematically linked keys — public key (shared openly) and private key (secret). Solves key distribution; slower.
    • Examples: RSA, ECC (Elliptic Curve), Diffie-Hellman, ElGamal, DSA
  • 3. Hash Functions (One-way): No key — produces fixed-length "fingerprint" of any input. Cannot be reversed.
    • Examples: MD5 (128-bit, deprecated), SHA-1 (160-bit, deprecated), SHA-256, SHA-3
  • 4. Hybrid Cryptography: Combines asymmetric (for key exchange) + symmetric (for bulk data) — used in TLS/HTTPS, PGP, SET.

Why Cryptography is Needed for e-Commerce

  • Payment security: Credit card and bank details must be encrypted in transit (TLS) and at rest (AES) — PCI-DSS mandates this.
  • Authentication: Digital certificates (built on RSA/ECC) verify website identity — prevents phishing and MITM attacks.
  • Data integrity: Hash functions ensure orders/payments arrive unaltered — any tampering detected instantly.
  • Non-repudiation: Digital signatures provide legal proof of transactions — buyers cannot deny purchases, sellers cannot deny sales.
  • User accounts: Passwords stored as salted hashes — even if database breached, passwords remain protected.
  • Session security: TLS session keys protect user sessions — prevents session hijacking.

RSA Algorithm — Full Explanation

RSA (Rivest-Shamir-Adleman, 1977) is the dominant asymmetric encryption algorithm, based on the integer factorisation problem.

KEY GENERATION:
  1. Choose two large primes: P and Q
  2. Compute: N = P × Q  (modulus)
  3. Compute: φ(N) = (P−1)(Q−1)  (Euler's totient)
  4. Choose E: 1 < E < φ(N), gcd(E, φ(N)) = 1
     (E is the public exponent — commonly 65537)
  5. Find D: E × D ≡ 1 mod φ(N)
     (D is the private exponent — Extended Euclidean Algorithm)

  Public key:  (E, N)
  Private key: (D, N)

ENCRYPTION (using recipient's public key):
  C = M^E mod N

DECRYPTION (using recipient's private key):
  M = C^D mod N

DIGITAL SIGNATURE (using sender's private key):
  Signature = Hash(M)^D mod N
  Verify:    Hash(M) = Signature^E mod N

WORKED EXAMPLE (P=17, Q=29):
  N = 493, φ(N) = 448, E = 3, D = 299
  Encrypt M=10: C = 10^3 mod 493 = 1000 mod 493 = 14
  Decrypt:      M = 14^299 mod 493 = 10 ✓

Security: Breaking RSA requires factoring N into P and Q — computationally infeasible for N ≥ 2048 bits with current computing power. RSA-2048 would take the universe's lifetime to brute-force classically.
