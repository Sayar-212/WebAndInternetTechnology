Q1(i)HTML documents are saved in1M
a) ASCII text
b) Machine language codes
c) Special binary format
d) None of these
✓ Answer: (a) ASCII text

HTML files are plain text files encoded in ASCII/UTF-8, saved with a .html or .htm extension. They can be opened and edited in any text editor like Notepad or VS Code.

Q1(ii)Position of SSL in TCP/IP protocol suite1M
a) Transport and internet layers
b) Data link and physical layers
c) Application and transport layers
d) None
✓ Answer: (c) Between Application and Transport layers

SSL/TLS intercepts data from the Application layer (HTTP, FTP), encrypts it, then passes it to TCP (Transport layer). HTTPS = HTTP + TLS on port 443.

Q1(iii)The _____ in XML is equivalent to the schema in database1M
a) Element
b) Attribute
c) Parser
d) DTD
✓ Answer: (d) DTD

DTD (Document Type Definition) defines the legal structure of an XML document — what elements are allowed, their order, and attribute types — just like a database schema defines table structure and constraints.

Q1(iv)Which syntax is used to describe elements in CSS?1M
a) Protectors
b) Selectors
c) Both
d) None
✓ Answer: (b) Selectors

CSS uses selectors to target HTML elements: p { color: red; } — the p is the selector. Types: element, class (.cls), id (#id), attribute, pseudo-class (:hover).

Q1(v)In Telnet, client echoes but doesn't send until whole line completed in1M
a) Default mode
b) Character mode
c) Server mode
d) None
✓ Answer: (a) Default mode

In default mode, the Telnet client buffers the full line locally, echoes characters on screen, but only transmits after Enter is pressed. This reduces network traffic compared to character mode.

Q1(vi)HTTP status code is1M
a) Two-digit
b) Three-digit
c) Four-digit
d) Five-digit
✓ Answer: (b) Three-digit code

HTTP uses 3-digit codes: 1xx Informational, 2xx Success (200 OK), 3xx Redirect (301), 4xx Client Error (404 Not Found), 5xx Server Error (500).

Q1(vii)Cookies are stored in1M
a) Web Server
b) Web Browser
c) Database Server
d) Application Server
✓ Answer: (b) Web Browser

Cookies are stored in the client's web browser (in its local cookie storage). The server creates the cookie and instructs the browser to store it via the Set-Cookie header. The browser sends it back with every subsequent request.

Q1(viii)IPv4 HLEN=12, Total Length=40000 — data field length?1M
a) 39,988
b) 40,012
c) 40,048
d) 39,952
✓ Answer: (d) 39,952
HLEN = 12 → Header size = 12 × 4 = 48 bytes
Total Length = 40,000 bytes
Data length = 40,000 − 48 = 39,952 bytes
Q1(ix)Which is correct regarding the meta tag in HTML?1M
a) <meta>...</meta>
b) <meta name=""/>
c) <metadata>...</metadata>
d) <metadata name=""/>
✓ Answer: (b) <meta name=""/>

The <meta> tag is a self-closing void element. Correct usage: <meta name="description" content="Page info"> or <meta charset="UTF-8">. There is no <metadata> tag in HTML.

Q1(x)In IPv6, option length is1M
a) 4
b) 5
c) 3
d) Infinite (variable)
✓ Answer: (d) Infinite / Variable

IPv6 uses extension headers instead of a fixed options field. Extension headers can be chained and their total length is variable — theoretically unlimited. Each extension header has its own Length field in units of 8 bytes.

Q2Explain the lifecycle of a Java Servlet.5 Marks
✓ Answer
  1. Loading       → JVM loads the Servlet .class file
  2. Instantiation → Container creates ONE servlet instance
         ↓
  3. init(ServletConfig config)
     ▸ Called ONCE — when servlet is first requested (or at startup)
     ▸ Used to: load config params, open DB connections, initialise resources
         ↓
  4. service(HttpServletRequest req, HttpServletResponse res)
     ▸ Called for EVERY incoming HTTP request
     ▸ Dispatches to: doGet(), doPost(), doPut(), doDelete()
     ▸ Multiple threads can call this simultaneously (servlet is single instance)
         ↓
  5. destroy()
     ▸ Called ONCE before container removes the servlet
     ▸ Used to: close DB connections, save state, release resources
import javax.servlet.*; import javax.servlet.http.*; import java.io.*;
public class LifecycleDemo extends HttpServlet {
    public void init() { System.out.println("Servlet initialised"); }

    public void doGet(HttpServletRequest req, HttpServletResponse res)
            throws IOException {
        res.setContentType("text/html");
        res.getWriter().println("<h2>Request served!</h2>");
    }

    public void destroy() { System.out.println("Servlet destroyed"); }
}
Q3a) DoS attack   b) Proxy firewall2+3 = 5
✓ Answer

a) Denial-of-Service Attack (2 marks)

A DoS attack overwhelms a server/network with excessive traffic, exhausting resources (CPU, memory, bandwidth) and making it unavailable to legitimate users. DDoS uses botnets (thousands of compromised machines) for a distributed attack. Common techniques: SYN Flood, UDP Flood, HTTP Flood, Ping of Death.

b) Proxy Firewall (3 marks)

Operates at Application Layer (Layer 7). Acts as a complete intermediary — client connects to proxy, proxy connects to server. No direct client-server connection. Performs deep content inspection (scans HTTP/FTP/SMTP payloads). Hides internal IPs. Caches content. Blocks malicious URLs and scans for viruses. Disadvantage: higher latency; needs a dedicated proxy per application protocol.

Q6a) What is cookie?   b) Why is HTTP stateless?2+3 = 5
✓ Answer

a) Cookie (2 marks)

A cookie is a small text file (≤4KB) containing key-value pairs that the web server sends to the browser via the Set-Cookie response header. The browser stores it and includes it in all future requests to the same domain via the Cookie header. Used for: session management, preferences, shopping carts, and tracking.

b) Why HTTP is Stateless (3 marks)

HTTP is stateless — each request-response pair is completely independent. The server does not retain any memory of previous requests from the same client.

  • By design: HTTP was designed for simplicity and scalability. Maintaining state for every client would require enormous server memory.
  • Each request is self-contained: It includes all the information the server needs (URL, headers, body) — no prior context needed.
  • Benefit: Server can be scaled horizontally — any server can handle any request since there's no session affinity.
  • Workaround: Session state IS needed for web apps — achieved via cookies, URL rewriting, hidden form fields, or server-side session objects.

Q8a) Web container + servlet engine   b) MTA responsibility   c) JSP factorial program4+6+5 = 15
✓ Answer

a) Web Container and Servlet Engine (4 marks)

Web Container (Servlet Container): The runtime environment that manages the execution of Servlets and JSPs. It provides: lifecycle management (init/service/destroy), HTTP request/response handling, session management, security, and deployment descriptor (web.xml) processing. Examples: Apache Tomcat, Jetty, GlassFish.

Servlet Engine: The component inside the web container responsible specifically for loading servlet classes, instantiating servlets, invoking their lifecycle methods, and managing their thread pool. In most literature, "Servlet Engine" and "Servlet Container" are used interchangeably.

b) Responsibilities of MTA (6 marks)

An MTA (Mail Transfer Agent) is software responsible for routing and transferring email between mail servers:

  • Accept email: Receives messages from MUAs or other MTAs via SMTP.
  • DNS MX lookup: Queries DNS for the recipient domain's MX (Mail Exchanger) record to find the target mail server.
  • Route and relay: Forwards messages through intermediate MTAs toward the final destination.
  • Queuing: If destination server is unavailable, stores message and retries at intervals (exponential backoff).
  • Bounce messages: Sends NDR (Non-Delivery Report) to sender if permanent delivery failure occurs.
  • Spam filtering: Many MTAs perform anti-spam and antivirus checks on incoming mail.
  • Examples: Postfix, Sendmail, Microsoft Exchange, Exim.

a) Steps to Create a DSN (6 marks)

A DSN (Data Source Name) is a configuration used by ODBC to connect applications to a database. Steps to create a DSN on Windows:

  • 1. Open Control Panel → Administrative Tools → ODBC Data Sources (or search "ODBC" in Start Menu).
  • 2. Choose DSN type: User DSN (current user only), System DSN (all users), or File DSN (stored as file).
  • 3. Click Add → select the appropriate database driver (e.g., MySQL ODBC Driver, SQL Server).
  • 4. Click Finish → Enter: DSN Name, Description, Server address, Database name, Port.
  • 5. Enter authentication credentials (username, password).
  • 6. Click Test Connection to verify the DSN works.
  • 7. Click OK to save. DSN is now available for JDBC/ODBC applications.

Q11Short Notes (any 3): a) HTML Image Map   b) VPN   c) SMTP   d) Gateway   e) Proxy Firewall3 × 5 = 15
✓ Answer

a) HTML Image Map (5 marks)

An image map allows different regions of a single image to be clickable hyperlinks using the <map> and <area> tags.

<img src="world.jpg" usemap="#worldmap" alt="World Map">
<map name="worldmap">
  <area shape="rect"   coords="50,50,200,150" href="asia.html"    alt="Asia">
  <area shape="circle" coords="350,200,60"    href="europe.html"  alt="Europe">
  <area shape="poly"   coords="100,300,250,280,200,400" href="africa.html" alt="Africa">
</map>

Shapes: rect(x1,y1,x2,y2), circle(cx,cy,r), poly(x1,y1...), default(whole image).

b) VPN — Virtual Private Network (5 marks)

A VPN creates a secure, encrypted tunnel over a public network (internet), allowing remote users to access a private network as if they were physically present.

  • Tunneling protocols: IPSec, SSL/TLS, PPTP, L2TP, OpenVPN, WireGuard.
  • Benefits: Encrypted communication, hides IP/location, secure remote access, bypass geo-restrictions.
  • Types: Remote Access VPN (user-to-network), Site-to-Site VPN (network-to-network).
  • Components: VPN client, VPN gateway/server, authentication server, tunneling protocol.

c) SMTP (5 marks)

SMTP (Simple Mail Transfer Protocol) is the standard protocol for sending email. Port 25 (server-to-server), 587 (submission), 465 (SMTPS). Commands: EHLO, MAIL FROM, RCPT TO, DATA, QUIT. Uses 7-bit ASCII; MIME extends it for attachments. Push protocol — only for sending, not retrieving (use POP3/IMAP for retrieval).

d) Gateway (5 marks)

A gateway is a network node that connects two networks using different communication protocols, data formats, or architectures — acting as a protocol translator.

  • Default gateway: The router that forwards packets from a local network to the internet.
  • Application gateway: Translates between application protocols (e.g., email gateway between SMTP and X.400).
  • Transport gateway: Connects networks with different transport protocols (TCP ↔ ISO TP4).
  • Gateways operate at Layer 4–7 (unlike routers which operate at Layer 3).

e) Proxy Firewall (5 marks)

Operates at Application Layer. Intercepts all traffic — client never talks directly to server. Deep content inspection; hides internal IPs; caches content; blocks malicious URLs; scans payloads for viruses. More secure than packet filtering but slower. Requires separate proxy per protocol.
