Q1(i)EDI requires1M
a) Representation of common business documents in computer readable form
b) Data entry operators by receivers
c) Special value added networks
d) Special hardware at co-operating business premises
✓ Answer: (a)

EDI (Electronic Data Interchange) works by converting standard business documents (invoices, purchase orders, shipping notices) into a computer-readable, standardised electronic format so two businesses can exchange them directly between their systems — without re-keying data. No special hardware or mandatory VAN is strictly required.

Q1(ii)By security in e-Commerce we mean — I. protecting data from unauthorised access, II. preventing disasters, III. authenticating messages, IV. protecting messages on internet from unauthorised reading1M
a) I, II
b) II, III
c) III, IV
d) I, III, IV
✓ Answer: (d)

e-Commerce security covers: I — confidentiality (protecting data stores), III — authentication/integrity (verifying message origin and content), and IV — message confidentiality in transit (encryption over internet). Statement II (preventing disasters) relates to disaster recovery / business continuity, not e-commerce security per se.

Q1(iii)A firewall may be implemented in1M
a) Routers which connect intranet to internet
b) Bridges used in an intranet
c) Expensive modem
d) User's application programs
✓ Answer: (a)

A firewall is most commonly implemented in the router/gateway that sits at the boundary between an internal network (intranet) and the public internet. It inspects and filters packets at that choke point. While software firewalls exist on end-user machines, the canonical architectural placement is at the intranet-to-internet router.

Q1(iv)Public key encryption method is a system1M
a) Which uses a set of public keys one for each participant
b) In which each person has two keys — a private key (known to him only) and a public key (publicised)
c) Which uses the RSA coding system
d) Which is a standard for use in e-Commerce
✓ Answer: (b)

In asymmetric / public-key cryptography, every participant has a key pair: a private key kept secret by the owner, and a public key freely distributed. Data encrypted with the public key can only be decrypted with the matching private key, and vice versa. RSA is one algorithm that implements this concept, not the definition of it.

Q1(v)In electronic cash payment — I. customer withdraws 'coins' signed by bank, II. bank has DB of issued coins, III. bank has DB of spent coins, IV. bank cannot trace a customer1M
a) I, II
b) I, II, III
c) I, II, III, IV
d) II, III, IV
✓ Answer: (c)

All four statements are true for a proper e-cash (digital cash) system: I — the bank issues digitally signed coins of various denominations; II — issued coins are tracked to prevent over-spending; III — spent coins are recorded to prevent double-spending; IV — blind signature schemes ensure the bank cannot link a spent coin to the original withdrawal (anonymity).

Q1(vi)SET protocol is used for1M
a) Securing message transfer
b) Securing card transaction
c) Securing authentication
d) Securing overall e-Commerce
✓ Answer: (b)

SET (Secure Electronic Transaction) was developed by Visa and MasterCard specifically to secure credit/debit card transactions over open networks. It uses digital certificates to authenticate all parties (cardholder, merchant, payment gateway) and encrypts card details so the merchant never sees the full card number.

Q1(vii)A key is a1M
a) Value
b) Logic
c) Function
d) All of these
✓ Answer: (d)

In cryptography, a key can be viewed as a value (a number/bit string), as logic (it governs the encryption/decryption process), and as a function parameter (the algorithm is a function that takes plaintext and key as input). Hence all three characterisations are valid.

Q1(viii)Which model of e-commerce deals with virtual auction?1M
a) Aggregator model
b) Info-mediary model
c) Community model
d) Brokerage model
✓ Answer: (d)

The Brokerage model brings buyers and sellers together and facilitates transactions — virtual auctions (like eBay) are a classic brokerage model implementation. The broker earns a commission or fee per transaction. Other brokerage variants include buy/sell fulfilment, market exchange, and reverse auctions.

Q1(ix)A/an ________ is a network that links selected resources of a company's intranet with its customers, suppliers or other business partners.1M
a) Internet
b) Extranet
c) VPN
d) Wide area network
✓ Answer: (b)

An Extranet is a controlled private network that extends a company's intranet to select external parties — customers, suppliers, or partners — granting them limited, authenticated access to specific internal resources (catalogues, order systems, inventory) over the internet.

Q1(x)How many types of trade cycle are there in e-commerce?1M
a) One
b) Two
c) Three
d) Four
✓ Answer: (c)

There are three types of trade cycles in e-commerce: (1) Repeat/Commodity trade cycle — for routine, repeat purchases (e.g., regular raw-material orders); (2) Credit/Large order trade cycle — for large, negotiated B2B purchases; (3) Cash/Low-value trade cycle — for small, one-off, low-value purchases paid immediately (e.g., online retail).

Q2What do you mean by knowledge engineering and business process redesign?5 Marks
✓ Answer

Knowledge Engineering (KE)

Knowledge Engineering is the process of capturing, structuring, and embedding human expert knowledge into a computer system (usually an expert system or knowledge base) so that the system can reason and make decisions like an expert. It involves:

  • Knowledge acquisition — eliciting knowledge from domain experts via interviews, observation, documents.
  • Knowledge representation — encoding knowledge using rules (IF–THEN), frames, semantic networks, or ontologies.
  • Knowledge validation — verifying accuracy and completeness of the knowledge base.
  • Inference engine — applying logic to derive new conclusions from stored knowledge.

Example in e-commerce: A product recommendation engine uses KE to encode rules like "IF customer buys a laptop THEN suggest a laptop bag."

Business Process Redesign (BPR)

BPR (also called Business Process Re-engineering) is the fundamental rethinking and radical redesign of business processes to achieve dramatic improvements in cost, quality, speed, and service. Key points:

  • Starts from scratch — questions "why do we do this at all?" rather than just improving existing steps.
  • Focuses on end-to-end processes across departments, not isolated functions.
  • Uses IT as an enabler to eliminate unnecessary steps, reduce paper work, and automate workflows.
  • Aims for order-of-magnitude improvements (e.g., reducing order processing from 5 days to 4 hours).

In ERP context: BPR is done before implementing ERP to align business processes with best practices built into the ERP software.

Q3Explain E-Governance with suitable examples.5 Marks
✓ Answer

E-Governance (Electronic Governance) is the use of information and communication technology (ICT) by government to deliver services, exchange information, and conduct transactions with citizens (G2C), businesses (G2B), and other government agencies (G2G).

Key Objectives

  • Improve efficiency and transparency of government services.
  • Reduce corruption by eliminating human intermediaries.
  • Provide 24×7 access to government services online.
  • Reduce cost and time for citizens and government alike.

Models of E-Governance

  • G2C (Government to Citizen): Online passport application, income-tax filing (Income Tax e-filing portal), driving licence renewal, ration card management.
  • G2B (Government to Business): Online company registration (MCA21), GST filing portal, e-procurement / GeM portal for government purchases.
  • G2G (Government to Government): PFMS (Public Financial Management System) — funds flow between central and state governments.
  • G2E (Government to Employee): Online payroll, leave management, and HR systems for government staff.

Examples in India

  • DigiLocker — citizens store and share official documents digitally.
  • UMANG App — single platform for 1200+ government services.
  • e-Courts — online case status tracking and e-filing of petitions.
  • IRCTC — online railway ticket booking (G2C).
Q4What are the benefits and points of concern in CPD (Collaborative Product Development)?5 Marks
✓ Answer

CPD (Collaborative Product Development) — also referred to as Collaborative Product Commerce — involves multiple firms (manufacturer, suppliers, design partners) jointly designing and developing a product using shared digital tools over a network.

Benefits of CPD

  • Faster time-to-market: Parallel design work by distributed teams cuts development cycles significantly.
  • Cost reduction: Shared resources, fewer physical prototypes, reduced travel for design reviews.
  • Better quality: Expertise from multiple partners leads to more robust designs; problems caught earlier.
  • Global talent access: Companies can tap best-in-class design partners anywhere in the world.
  • Improved supplier integration: Suppliers involved early can suggest design-for-manufacturability improvements.
  • Single version of truth: All partners work on the same shared CAD/PLM data — no version conflicts.

Points of Concern

  • IP (Intellectual Property) risk: Sharing design data with external partners risks leakage of proprietary information.
  • Security: Sensitive design files transmitted over networks must be encrypted and access-controlled.
  • Trust issues: Requires high trust between competing or partially competing partners.
  • Integration complexity: Different partners use different CAD/PLM tools — interoperability is challenging.
  • Legal agreements: Contracts on ownership, liability, and confidentiality need careful drafting.
  • Change management: Coordinating design changes across multiple organisations is complex.
Q5What is BPR? Explain with an example.5 Marks
✓ Answer

BPR (Business Process Re-engineering) is the fundamental rethinking and radical redesign of core business processes to achieve dramatic improvements in performance measures such as cost, quality, service, and speed. Coined by Michael Hammer and James Champy (1993).

Key Characteristics

  • Fundamental: Questions "Why do we do this?" — not just "How can we do this better?"
  • Radical: Discards existing processes and redesigns from scratch.
  • Dramatic: Seeks order-of-magnitude gains (10× improvement), not incremental 5–10%.
  • Process-oriented: Focuses on cross-functional end-to-end processes, not individual departments.

BPR Steps

1. Identify the process to re-engineer
2. Map the AS-IS (current) process
3. Analyse weaknesses, bottlenecks, redundancies
4. Design the TO-BE (future) process using IT as enabler
5. Implement and measure results

Example — Ford Motor Company's Accounts Payable

Ford's old process: When goods arrived, purchasing sent a PO to accounts payable; receiving sent a receipt note; vendor sent an invoice — AP matched all 3 documents (14-step process, 500 staff).

After BPR: AP checks only 2 things — is there an open PO in the database? Did goods arrive? If yes, payment is triggered automatically. Result: 75% reduction in headcount, much faster payment cycles.

Q7a) Digital Signature vs Digital Certificate   b) RSA with P=17, Q=29   c) Firewall and its functions15 Marks
✓ Answer

a) Digital Signature — and how it differs from Digital Certificate

A Digital Signature is an electronic equivalent of a handwritten signature. It is a mathematical scheme that verifies the authenticity and integrity of a digital message or document.

How it works:

Sender side:
  1. Compute hash of message → H(M)
  2. Encrypt hash with Sender's PRIVATE key → Digital Signature
  3. Send: [Message + Digital Signature]

Receiver side:
  1. Decrypt signature using Sender's PUBLIC key → H(M)'
  2. Independently hash the received message → H(M)
  3. If H(M)' == H(M)  → signature is VALID (authentic + unaltered)

Properties: Authentication (proves identity), Integrity (detects tampering), Non-repudiation (sender cannot deny signing).

A Digital Certificate is a document issued by a trusted Certificate Authority (CA) that binds a public key to an identity (person, organisation, server). It contains: owner's name, public key, CA's signature, validity period, serial number.

b) RSA Encryption — P=17, Q=29

Step 1: N = P × Q = 17 × 29 = 493

Step 2: φ(N) = (P−1)(Q−1) = 16 × 28 = 448

Step 3: Choose E such that:
   • 1 < E < 448
   • gcd(E, 448) = 1  (E and φ(N) must be coprime)
   Try E = 3 → gcd(3, 448) = 1  ✓   (448 = 2^6 × 7, 3 shares no factors)
   → E = 3

Step 4: Find D such that  E × D ≡ 1 (mod φ(N))
   i.e.,  3D ≡ 1 (mod 448)
   Using Extended Euclidean Algorithm:
   448 = 149 × 3 + 1
   → 1 = 448 − 149 × 3
   → D ≡ −149 ≡ 448 − 149 = 299 (mod 448)
   → D = 299

Result:
   N = 493,   E = 3 (public key exponent),   D = 299 (private key exponent)
   Public  key  : (E, N) = (3,  493)
   Private key  : (D, N) = (299, 493)

Verification: E × D mod φ(N) = 3 × 299 mod 448 = 897 mod 448 = 1  ✓

c) Firewall — Definition and Functions in e-Commerce

A Firewall is a network security system (hardware, software, or both) that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It establishes a barrier between a trusted internal network and untrusted external networks (internet).

Types: Packet-filtering firewall, Stateful inspection firewall, Application-layer (proxy) firewall, Next-generation firewall (NGFW).

Functions in e-Commerce:

  • Access control: Blocks unauthorised users from accessing internal servers (database, payment server).
  • Packet filtering: Inspects source/destination IP and port — blocks traffic from known malicious IPs.
  • Protecting web servers: Allows only HTTP/HTTPS traffic (port 80/443) to the web server.
  • Preventing DoS/DDoS: Rate-limits or blocks flood attacks targeting the e-commerce site.
  • DMZ enforcement: Places web server in a DMZ — public can reach it, but it cannot reach internal DB directly.
  • Logging and auditing: Records all traffic for forensic analysis and compliance reporting.
  • VPN gateway: Allows secure remote access for administrators and B2B partners.

Q8a) Call Center & Equipment   b) Modes of operation of call center   c) CPE and Denial of Service Attacks15 Marks
✓ Answer

a) Call Center — Definition and Equipment

A Call Center is a centralised office (physical or virtual) where a large number of customer service representatives handle inbound and outbound telephone/digital communications for an organisation. In e-commerce, call centers support order tracking, complaints, returns, and sales.

Equipment required:

  • ACD (Automatic Call Distributor): Routes incoming calls to the next available agent based on rules.
  • IVR (Interactive Voice Response): Automated phone menu — customers self-serve or get routed before reaching an agent.
  • CTI (Computer-Telephony Integration): Links phone system with CRM so caller's details pop up on agent's screen automatically.
  • Agent workstations: PCs with CRM software, order management system, and knowledge base.
  • Headsets: Hands-free communication for agents.
  • LAN / WAN infrastructure: High-speed network connecting all workstations to servers.
  • PBX (Private Branch Exchange): Internal telephone switching system managing extensions.
  • Recording systems: Quality monitoring and compliance — records calls.
  • UPS / Power backup: Ensures 24×7 uptime.

b) Modes of Operation of a Call Center

  • Inbound call center: Handles incoming calls — customer service, tech support, order queries, helpdesk. Agents wait for calls.
  • Outbound call center: Agents proactively call customers — telemarketing, sales campaigns, follow-ups, surveys.
  • Blended call center: Handles both inbound and outbound — agents switch between receiving and making calls based on queue load.
  • Virtual call center: Agents work from home/remote locations connected via internet/VoIP — no physical office needed.
  • Automated / Self-service: IVR handles routine queries (balance inquiry, order status) without a live agent.

c) CPE — Customer Premises Equipment

CPE (Customer Premises Equipment) refers to any terminal or network device located at the customer's location (as opposed to equipment at the service provider's facility). Examples: routers, modems, set-top boxes, IP phones, DSL modems, VoIP adapters.

In e-commerce context, CPE is the hardware the customer owns through which they access the internet and the e-commerce site.

Denial of Service (DoS) Attacks

A DoS attack is a malicious attempt to make a server, network, or service unavailable to its intended users by overwhelming it with a flood of illegitimate requests until it crashes or becomes too slow to respond.

  • DDoS (Distributed DoS): Attack launched from thousands of compromised machines (botnet) simultaneously — much harder to block.
  • SYN Flood: Attacker sends millions of TCP SYN packets without completing the handshake — exhausts server's connection table.
  • HTTP Flood: Overwhelms web server with GET/POST requests.
  • Ping of Death: Sends malformed oversized ICMP packets.

Impact on e-commerce: Site goes offline → lost sales, reputation damage, SLA breaches.

Mitigation: Rate limiting, CDN with DDoS scrubbing, firewalls, IDS/IPS, cloud-based DDoS protection (Cloudflare, AWS Shield).

Q9a) What is EDI?   b) Basic components   c) Role in e-commerce   d) EDIFACT   e) Benefits of EDI15 Marks
✓ Answer

a) EDI — Electronic Data Interchange

EDI is the computer-to-computer exchange of standard business documents (purchase orders, invoices, shipping notices, payment instructions) between organisations in an electronic format, replacing paper-based processes. No human intervention is needed — one company's computer sends a transaction directly into another's.

b) Basic Components of EDI

  • EDI Standards: Define the document structure and data format — ANSI X12 (US), EDIFACT (international), HL7 (healthcare). Both parties must use the same standard.
  • Translation Software: Converts data from the company's internal format (flat file, database) to the standard EDI format and vice versa.
  • Communication Network: The medium for transmission — VAN (Value Added Network), Internet/AS2, FTP/SFTP.
  • VAN (Value Added Network): A third-party network acting as an intermediary "mailbox" — stores and forwards EDI messages securely.
  • Trading Partners: The two (or more) organisations exchanging EDI documents. Both must agree on standards and protocols.
  • Interchange Control: Envelope structure (ISA/IEA in X12) wrapping transactions — ensures correct routing and acknowledgement.

c) Role of EDI in e-Commerce

  • Automated procurement: POs sent automatically from buyer's ERP to supplier's ERP when stock falls below reorder level.
  • Faster order-to-cash cycle: Invoice + payment exchanged electronically — days reduced to hours.
  • Supply chain visibility: ASN (Advance Ship Notice) lets buyer prepare for receiving before goods arrive.
  • B2B integration: EDI is the backbone of B2B e-commerce between large retailers and their suppliers (Walmart-supplier EDI mandate).
  • Reduces errors: No manual re-keying → fewer data entry errors → fewer disputes.

d) EDIFACT

EDIFACT (Electronic Data Interchange for Administration, Commerce and Transport) is the international EDI standard developed by the United Nations (UN/EDIFACT). It is used globally, especially in Europe and Asia, for cross-border trade. Key features:

  • Hierarchical message structure: Interchange → Functional Group → Message → Segment → Data Element.
  • Standard message types: ORDERS (purchase order), INVOIC (invoice), DESADV (despatch advice), REMADV (remittance advice).
  • Character-based, human-readable syntax using delimiters.
  • Maintained by UNECE (United Nations Economic Commission for Europe).
EDIFACT Message structure:
UNB (Interchange header)
  UNG (Functional group header)
    UNH (Message header) → INVOIC
      BGM, DTM, NAD, LIN, MOA ...
    UNT (Message trailer)
  UNE (Functional group trailer)
UNZ (Interchange trailer)

e) Benefits of EDI

  • Speed: Documents exchanged in minutes vs days for paper/post.
  • Accuracy: Eliminates manual data entry errors — data flows system to system.
  • Cost reduction: Lower paper, postage, storage, and labour costs.
  • Better inventory management: Real-time data enables JIT (Just-In-Time) ordering.
  • Improved cash flow: Faster invoicing and payment.
  • Competitive advantage: Mandatory for doing business with major retailers/manufacturers.
  • Audit trail: All transactions logged electronically — easier compliance and dispute resolution.
Q10a) Define ERP and information integration   b) Reasons for ERP growth and advantages15 Marks
✓ Answer

a) ERP — Definition and Information Integration

ERP (Enterprise Resource Planning) is an integrated software system that manages and automates core business processes — finance, HR, manufacturing, supply chain, procurement, sales — using a single unified database. All departments share the same data in real time.

How ERP integrates information:

  • Single database: All modules (Finance, Sales, Inventory, HR) write to and read from one central database — eliminates data silos.
  • Real-time updates: When sales records an order, inventory and finance are updated instantly — no batch uploads.
  • Cross-functional workflows: A sales order automatically triggers inventory check → production planning → procurement → shipping → invoice generation.
  • Standardised processes: Best-practice workflows are built in — enforces consistency across locations/departments.
  • Reporting & analytics: Management gets organisation-wide dashboards from one source of truth.

b) Reasons for ERP Market Growth

  • Globalisation: Companies operating in multiple countries need integrated systems to manage cross-border operations.
  • Y2K readiness: Late 1990s Y2K compliance push drove massive ERP adoption as companies replaced legacy systems.
  • Competitive pressure: Companies needed real-time data and lean operations to compete.
  • Internet & e-commerce integration: ERP needed to connect with online stores, portals, and EDI.
  • Cloud ERP: SaaS ERP (SAP S/4HANA Cloud, Oracle NetSuite) lowered cost of entry for SMEs.
  • Regulatory compliance: Finance regulations (Sarbanes-Oxley, GST) require integrated, auditable systems.
  • Merger & acquisition activity: Acquired companies need to integrate onto a single platform.

Advantages of ERP System

  • Elimination of data redundancy: Single database means no duplicate data entry.
  • Improved visibility: Real-time dashboards give management instant insight across all departments.
  • Streamlined processes: Automated workflows reduce manual steps and bottlenecks.
  • Better customer service: Sales reps can instantly check inventory, delivery schedules, and order status.
  • Cost reduction: Lower inventory carrying costs, better supplier negotiations, reduced overtime.
  • Scalability: New modules and users can be added as the business grows.
  • Regulatory compliance: Built-in controls for financial reporting (GAAP, IFRS) and audit trails.
  • Supply chain optimisation: Integrated procurement and inventory management reduces stockouts and over-ordering.
Q11Short notes on any three: a) EAI   b) SCE Framework   c) BAAN   d) SAP   e) VPN15 Marks
✓ Answer

a) EAI — Enterprise Application Integration

EAI is the use of middleware software and architectures to integrate disparate enterprise applications (ERP, CRM, SCM, legacy systems) so they can share data and business processes seamlessly — without custom point-to-point coding.

  • Middleware approaches: Message brokers, ESB (Enterprise Service Bus), API gateways.
  • Integration patterns: Hub-and-spoke, peer-to-peer, publish/subscribe.
  • Why needed: Large enterprises run 50–200+ applications; EAI creates a unified data flow between them.
  • Benefits: Reduced integration cost, real-time data sync, process automation across systems.
  • Example: When a CRM records a new sale, EAI automatically triggers ERP inventory deduction and logistics scheduling.

b) SCE Framework — Supply Chain Execution

SCE (Supply Chain Execution) refers to the set of applications and processes that manage the physical flow of goods from manufacturer to end customer — the "execution" layer of supply chain management.

  • Components: WMS (Warehouse Management System), TMS (Transportation Management System), OMS (Order Management System), manufacturing execution.
  • vs SCP (Supply Chain Planning): SCP forecasts demand and plans; SCE executes the actual movement of goods.
  • Key functions: Receiving, put-away, pick-pack-ship, carrier selection, track & trace, returns management.
  • Integration: SCE connects ERP (order data) with physical warehouse and carrier systems in real time.

c) BAAN

BAAN (now Infor LN) is a major ERP software package originally developed by Dutch company Baan Company NV. It was particularly strong in discrete manufacturing industries.

  • Modules: Finance, Manufacturing, Project, Procurement, Sales, Distribution, Service.
  • Known for its strength in engineer-to-order (ETO) and project manufacturing (aerospace, defence, heavy machinery).
  • Uses a proprietary 4GL language called Tools for customisation.
  • Competed directly with SAP and Oracle ERP in the late 1990s.
  • Acquired by Invensys, then SSA Global, and now Infor — rebranded as Infor LN.

d) SAP

SAP (Systems, Applications, and Products in Data Processing) is the world's largest ERP software company, founded in 1972 in Germany. Its flagship product SAP S/4HANA is used by over 400,000 companies globally.

  • Core modules: FI (Financial Accounting), CO (Controlling), MM (Materials Management), SD (Sales & Distribution), PP (Production Planning), HR (Human Resources), PM (Plant Maintenance).
  • SAP R/3: Client-server based ERP (1992) — dominant for 3 decades.
  • SAP S/4HANA: Modern cloud/on-premise ERP built on SAP HANA in-memory database.
  • Industry solutions: SAP for Retail, SAP for Banking, SAP for Pharma.
  • Integration: SAP integrates with e-commerce platforms (Magento, Shopify) via SAP Commerce Cloud.

e) VPN — Virtual Private Network

A VPN creates an encrypted, private tunnel over a public network (internet) — allowing remote users or branch offices to securely access a private corporate network as if they were physically connected to it.

  • How it works: Client software encrypts all traffic → sends through encrypted tunnel → VPN server decrypts → forwards to destination on private network.
  • Protocols: IPSec, SSL/TLS, OpenVPN, WireGuard, PPTP, L2TP.
  • Types: Remote-access VPN (employee connects from home), Site-to-site VPN (two office LANs connected securely).
  • e-Commerce use: Secures communication between e-commerce server and back-office ERP; allows remote admin access; enables B2B EDI over internet securely.
  • Benefits: Data confidentiality, authentication, integrity; bypasses geographic restrictions; cost-effective alternative to leased lines.
