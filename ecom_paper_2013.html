Q1(i)EDI requires1M
a) Representation of common business documents in computer readable form
b) Data entry operators by receivers
c) Special value added networks
d) Special hardware at co-operating business premises
✓ Answer: (a)

EDI's core requirement is converting standard business documents (invoices, POs, shipping notices) into a machine-readable, standardised electronic format so computers can exchange them directly. EDI eliminates manual re-keying by ensuring both sender's and receiver's systems use the same format. No special hardware is required — EDI runs over standard internet connections (AS2) or VANs.

Q1(ii)By security in e-Commerce we mean — I. protecting data, II. preventing disasters, III. authenticating messages, IV. protecting messages from unauthorised reading1M
a) I, III
b) II, III
c) III, IV
d) I, III, IV
✓ Answer: (d) — I, III, IV

e-Commerce security covers: I — data protection (preventing unauthorised access to customer and transaction data), III — authentication (verifying who sent a message and that it wasn't altered), and IV — message confidentiality (encryption prevents interception). Statement II (preventing disasters) is business continuity/disaster recovery — outside the scope of e-commerce security per se.

Q1(iii)A firewall may be implemented in1M
a) Routers which connect Intranet to internet
b) Bridges used in an Intranet
c) Expensive modem
d) User's application programs
✓ Answer: (a)

Firewalls are implemented in routers/gateways at the network boundary between the trusted internal network (intranet) and the untrusted internet. This is the architectural choke point where all external traffic passes — giving maximum inspection and filtering capability. All packets entering or leaving the network pass through this single point.

Q1(iv)Public key encryption method is a system1M
a) Which uses a set of public keys one for each participant
b) In which each person has two keys — a private key (known to him only) and a public key (publicised)
c) Which uses the RSA coding system
d) Which is a standard for use in e-Commerce
✓ Answer: (b)

Public key (asymmetric) cryptography uses a key pair: a private key that only the owner knows, and a public key freely distributed to everyone. Data encrypted with one key can only be decrypted with the other. RSA is one algorithm implementing this concept — but the definition of public key encryption is the key-pair mechanism described in (b).

Q1(v)In electronic cash payment — I. customer withdraws 'coins' signed by bank, II. bank has DB of issued coins, III. bank has DB of spent coins, IV. bank cannot trace a customer1M
a) I, II
b) I, II, III
c) I, II, III, IV
d) II, III, IV
✓ Answer: (c) — All four

All four statements are true for a complete e-cash system: I — bank issues blind-signed coins; II — issued coin DB prevents over-issuance; III — spent coin DB prevents double-spending; IV — blind signature scheme ensures the bank cannot link a coin to the original withdrawal, preserving customer anonymity (like physical cash).

Q1(vi)SET protocol is used for1M
a) Securing message transfer
b) Securing card transaction
c) Securing authentication
d) Securing overall e-Commerce
✓ Answer: (b)

SET (Secure Electronic Transaction) was designed specifically by Visa/MasterCard to secure payment card transactions over the internet. It protects cardholder data using dual-signature (merchant sees order but not card details; bank sees card but not order), digital certificates for all parties, and RSA/DES encryption.

Q1(vii)A key is a1M
a) Value
b) Logic
c) Function
d) All of these
✓ Answer: (d)

A cryptographic key can be described as a value (a number/bit string), as logic (it controls the behaviour of the encryption algorithm), and as an input to a function (the encryption algorithm is a function taking plaintext and key). All three characterisations are valid.

Q1(viii)Which model of e-commerce deals with virtual auction?1M
a) Aggregator model
b) Community model
c) Info-mediary model
d) Brokerage model
✓ Answer: (d) — Brokerage model

The Brokerage model brings buyers and sellers together and facilitates transactions — earning a fee/commission per transaction. Virtual auctions (eBay, Sotheby's online, government e-auctions) are classic brokerage model implementations. The broker provides the marketplace; buyers and sellers transact through it. Sub-types include: buy/sell fulfilment, auction brokers, transaction brokers, reverse auctions.

Q1(ix)A/An ________ is a network that links selected resources of a company's internet with its customers, suppliers or other business partners.1M
a) Internet
b) Extranet
c) VPN
d) Wide area network
✓ Answer: (b) — Extranet

An Extranet is a controlled, private extension of a company's intranet that gives selected external parties — customers, suppliers, partners — limited, authenticated access to specific internal resources. It sits between fully public (internet) and fully private (intranet). Uses: supplier portals, customer order tracking, dealer networks, partner collaboration.

Q1(x)How many types of trade cycle are there in e-commerce?1M
a) One
b) Two
c) Three
d) Four
✓ Answer: (c) — Three

Three trade cycle types: (1) Repeat/Commodity — routine, repeat purchases (regular raw materials); (2) Credit/Large order — large negotiated B2B purchases on agreed credit terms; (3) Cash/Low-value — small, one-off immediate-payment purchases (online retail). Each requires different EDI message sets and process flows.

Q2What do you mean by knowledge engineering and business process redesign?5 Marks
✓ Answer

See 2008-09 paper Q2 for the complete answer. Summary:

Knowledge Engineering (KE): Capturing expert domain knowledge and encoding it into computer systems (expert systems, AI) through knowledge acquisition, representation (rules, frames, ontologies), validation, and inference engine design. Enables machines to reason like human experts. In e-commerce: product recommendation engines, fraud detection rules, pricing algorithms.

Business Process Redesign (BPR): Fundamental, radical rethinking of business processes — not incremental improvement, but a complete redesign from scratch to achieve dramatic (10×) improvements in cost, quality, and speed. Steps: identify process → map as-is → analyse waste → design to-be (with IT as enabler) → implement. Classic example: Ford Motors reduced AP staff by 75% through BPR. In ERP context: BPR must happen before ERP implementation — redesign processes to match ERP's best practices rather than customising ERP around broken processes.

Q3Explain E-Governance with suitable examples.5 Marks
✓ Answer

See 2008-09 paper Q3 for the complete answer. Summary:

e-Governance is the use of ICT by government to deliver services to citizens (G2C), businesses (G2B), other agencies (G2G), and employees (G2E). Goals: efficiency, transparency, accessibility, reduced corruption.

Indian examples:

  • G2C: Income Tax e-Filing portal, Passport Seva Kendra online, DigiLocker (digital document storage), IRCTC (rail bookings), e-Courts (case status, e-filing).
  • G2B: GeM (Government e-Marketplace) — government procurement from businesses; GSTN — GST registration and filing; MCA21 — company registration.
  • G2G: PFMS (Public Financial Management System) — fund tracking across central and state government; NICNET connecting all government offices.
  • G2E: HRMS (Human Resource Management System) for government employees — payroll, leave, transfer.

How it works: Online portal → citizen authenticates (Aadhaar/PAN/OTP) → fills digital form → backend systems process request → status updates sent via SMS/email → fees paid online → certificate/service delivered digitally. No physical visit needed.

Q4What are the benefits and points of concern in CPD (Collaborative Product Development)?5 Marks
✓ Answer

See 2008-09 paper Q4 for the complete answer. Summary:

CPD Benefits: Faster time-to-market (parallel design); cost reduction (shared resources, fewer prototypes); better quality (multi-partner expertise); global talent access; improved supplier integration (design-for-manufacturability); single version of truth via shared PLM/CAD data.

Points of Concern: IP theft risk (sensitive design shared externally); security of design files in transit; trust issues between partner companies; tool interoperability (different CAD standards); legal agreements on ownership and liability; change management complexity across organisations.

Q5What is BPR? Explain with an example.5 Marks
✓ Answer

See 2008-09 paper Q5 for the complete answer. Summary:

BPR (Business Process Re-engineering) — Hammer & Champy (1993) — is the fundamental rethinking and radical redesign of business processes to achieve dramatic improvements (not incremental). Key: questions "why do we do this at all?" not "how can we do it better?"

Steps: Identify process → map AS-IS → analyse waste/bottlenecks → design TO-BE (enabled by IT) → implement → measure results.

Example — Ford Motors Accounts Payable: Old process: 500 staff matching 3 documents (PO, goods receipt, vendor invoice). After BPR: AP queries just 2 things in the database (open PO? goods received?) → if yes, auto-pay. Result: 75% staff reduction, much faster payment cycles.

Another example: An e-commerce company redesigning its returns process — from 14-step manual approval (7 days) to an automated self-service return portal (same day refund trigger) using BPR + system automation.

Q7a) Digital Signature vs Digital Certificate   b) RSA: P=17, Q=29   c) Firewall & functions15 Marks
✓ Answer

See 2008-09 paper Q7 for the complete detailed answer. This question is identical. Key points:

a) Digital Signature vs Digital Certificate

Digital Signature: Cryptographic proof attached to a message — sender encrypts a hash of the message with their private key. Receiver decrypts with sender's public key and verifies the hash. Provides: authentication, integrity, non-repudiation.

Digital Certificate: CA-issued document binding a public key to an identity. Contains: owner name, public key, CA's signature, validity dates. Enables trust — you trust a website because its certificate was signed by a CA your browser trusts.

AspectDigital SignatureDigital Certificate
WhatEncrypted hash proving message authenticityDocument binding a public key to an identity
Created byMessage sender using private keyCertificate Authority (CA)
PurposeVerify message origin and integrityVerify public key ownership
Used inEmail signing, document signing, SETHTTPS/TLS, SET, code signing

b) RSA Calculation — P=17, Q=29

N  = P × Q = 17 × 29 = 493
φ(N) = (P−1)(Q−1) = 16 × 28 = 448
E: gcd(E, 448) = 1 → E = 3  (448 = 2^6 × 7; 3 is coprime)
D: 3D ≡ 1 (mod 448)
   Extended Euclidean: 448 = 149×3 + 1
   → D = 448 − 149 = 299

Public key:  (E=3,   N=493)
Private key: (D=299, N=493)
Verify:  3 × 299 = 897 = 2×448 + 1 → 897 mod 448 = 1 ✓

Encrypt M:   C = M³ mod 493
Decrypt C:   M = C²⁹⁹ mod 493

c) Firewall — Definition and Functions in e-Commerce

A firewall monitors and filters network traffic based on security rules — placed at the intranet-internet boundary. Functions: packet filtering (block/allow by IP/port), DMZ creation (isolate web server from internal DB), DoS protection, IP blacklisting, application inspection (WAF blocks SQL injection/XSS), VPN termination, and audit logging. See 2008-09 Q7(c) and 2010-11 Q9(d) for full detail.

Q8a) Call Center & equipment   b) Modes of operation   c) CPE & Denial of Service Attacks15 Marks
✓ Answer

See 2008-09 paper Q8 for the complete answer. This question is identical. Key points:

a) Call Center & Equipment

Centralised facility handling customer communications. Equipment: ACD (routes calls), IVR (automated menus), CTI (links phone to CRM — screen pop), agent PCs with CRM software, headsets, PBX, LAN, call recording system, UPS backup.

b) Modes of Operation

  • Inbound: Handles incoming customer calls — support, queries, complaints.
  • Outbound: Agents proactively call customers — sales, surveys, follow-ups.
  • Blended: Both inbound and outbound — agents switch based on queue.
  • Virtual: Remote agents connected via internet/VoIP — no physical office.
  • Automated/Self-service: IVR handles routine queries without live agent.

c) CPE & Denial of Service Attacks

CPE (Customer Premises Equipment): Hardware at the customer's location through which they access services — routers, modems, IP phones, set-top boxes, DSL modems.

DoS Attack: Malicious flood of requests overwhelms a server/network, making it unavailable. DDoS (Distributed) uses thousands of compromised machines (botnet). Types: SYN flood, HTTP flood, UDP flood, Ping of Death. In e-commerce: site goes offline → lost revenue, reputation damage. Mitigation: rate limiting, CDN scrubbing, cloud DDoS protection (Cloudflare, AWS Shield), firewalls, IDS/IPS.

Q9a) What is EDI?   b) Basic components   c) Role in e-commerce   d) EDIFACT   e) Benefits of EDI15 Marks
✓ Answer

See 2008-09 paper Q9 for the complete answer. This question is identical. Key summary:

a) EDI

Computer-to-computer exchange of standardised business documents (POs, invoices, ASNs, payment instructions) in machine-readable format between trading partners — no manual intervention required.

b) Basic Components

EDI Standards (X12/EDIFACT), Translation Software (converts internal ↔ EDI format), Communication Network (VAN / AS2 / SFTP), VAN (store-and-forward mailbox), Trading Partner Agreements, Functional Acknowledgements.

c) Role in e-Commerce

Automates B2B procurement; enables real-time inventory updates via ASNs; required by major retailers; powers order-to-cash and procure-to-pay automation; provides scale for high-volume e-commerce operations.

d) EDIFACT

UN/EDIFACT — international EDI standard. Hierarchical: UNB/UNZ (interchange) → UNG/UNE (group) → UNH/UNT (message). Key messages: ORDERS, INVOIC, DESADV, REMADV, CONTRL. Uses delimiter syntax (+ : '). Used internationally especially Europe and cross-border trade. Maintained by UNECE.

e) Benefits

Speed (minutes vs days), accuracy (no re-keying errors), cost (near-zero per transaction vs ₹1500+ for paper), automation (triggers downstream processes), JIT inventory support, audit trail, competitive necessity for retail suppliers.

Q10a) Define ERP & information integration   b) Reasons for ERP growth & advantages15 Marks
✓ Answer

See 2008-09 paper Q10 for the complete answer. This question is identical. Key summary:

a) ERP & Information Integration

ERP integrates all business functions through a single unified database. Integration mechanism: all modules (FI, MM, SD, PP, HR) share one database → sales order automatically updates inventory, triggers production plan, creates accounting entry, and generates customer invoice — in real time, without manual data transfer. Single source of truth eliminates silos.

b) Reasons for ERP Market Growth

Globalisation, Y2K remediation, competitive pressure, internet/e-commerce growth, cloud ERP (SaaS reduces barrier to entry), regulatory compliance (GST, SOX, GDPR), M&A consolidation, industry-specific solutions.

Advantages of ERP

Single data source, real-time visibility, process standardisation, reduced IT costs, faster order fulfilment, better customer service, automated financial close, supply chain optimisation, regulatory compliance, scalability.

Q11Short notes on any three: a) EAI   b) SCE Framework   c) BAAN   d) SAP   e) VPN15 Marks
✓ Answer

See 2008-09 paper Q11 for the complete answers. These are identical questions. Summary:

a) EAI — Enterprise Application Integration

Middleware/architecture that connects disparate enterprise applications (ERP, CRM, SCM, legacy) enabling seamless data and process flow. Approaches: ESB (Enterprise Service Bus), message broker, API gateway. Hub-and-spoke or publish-subscribe topology. Eliminates point-to-point spaghetti integrations. Example: CRM new sale → EAI triggers ERP inventory reduction + logistics scheduling automatically.

b) SCE Framework

Supply Chain Execution — the operational layer managing physical goods movement. Core systems: WMS (warehouse), TMS (transport), OMS (orders), MES (manufacturing floor). Bridges planning (SCP) with physical execution. KPIs: fill rate, OTIF, warehouse throughput, freight cost/unit.

c) BAAN

Dutch ERP vendor (Baan NV) — strong in discrete/engineer-to-order manufacturing (aerospace, defence, heavy machinery). Used proprietary 4GL "Tools" language. Acquired by Invensys → SSA Global → Infor → now marketed as Infor LN. Modules: Finance, Manufacturing, Project, Procurement, Service.

d) SAP

World's largest ERP vendor (Germany, 1972). SAP S/4HANA is current flagship — in-memory database, cloud/on-premise. Core modules: FI, CO, MM, SD, PP, HR, QM, PM. Industry solutions: SAP for Retail, Pharma, Banking. SAP Commerce Cloud for e-commerce integration. Used by 400,000+ companies globally.

e) VPN — Virtual Private Network

Creates an encrypted tunnel over the public internet — allows secure private communication. Protocols: IPSec, SSL/TLS, OpenVPN, WireGuard. Types: remote-access (employee from home), site-to-site (branch offices). e-Commerce use: secure admin access, B2B EDI over internet, remote staff accessing ERP. Benefits: confidentiality, authentication, integrity, cost-effective alternative to leased lines.
