Q1(I)What is SVG and why is it used?1M
✓ Answer

SVG (Scalable Vector Graphics) is an XML-based format for describing 2D vector graphics. Unlike raster images (JPG/PNG), SVG images are defined by mathematical paths and scale to any size without losing quality.

Why used: Perfect for logos, icons, charts, and illustrations. Fully styleable with CSS, scriptable with JavaScript, accessible (text-based), small file size for simple graphics, and renders crisply on retina/4K screens.

Q1(III)Difference between variables in which the chomp function works in Perl.1M
✓ Answer

chomp() in Perl removes the trailing newline character (\n) from a string. It works on:

  • Scalar variable ($var): Removes one trailing newline from that single string.
  • Array variable (@arr): Removes trailing newline from every element in the array.
my $str = "Hello\n";
chomp($str);          # $str is now "Hello"

my @lines = ("a\n", "b\n", "c\n");
chomp(@lines);        # @lines is now ("a","b","c")

Key difference: chop() removes the LAST character regardless; chomp() only removes newline.

Q1(IV)What is Multiprogramming?1M
✓ Answer

Multiprogramming is an OS technique where multiple programs are loaded into memory simultaneously, and the CPU is switched between them to keep it busy. When one program is waiting for I/O, the CPU executes another program.

Goal: Maximise CPU utilisation and system throughput. The CPU never sits idle waiting for slow I/O operations. Differs from multitasking (time-sharing) which switches rapidly to give illusion of simultaneous execution.

✓ Answer

To make a VoIP call you need:

  • Hardware: Broadband internet connection, microphone, speakers/headset.
  • Software/App: A VoIP application — Skype, Zoom, WhatsApp, Google Meet, or a SIP client.

Process: Open the app → enter the recipient's number/username → click Call → your voice is digitised, compressed (codec), packetised (RTP/UDP), transmitted over internet → reconstructed at receiver's end.

Q1(VII)What is Encapsulation Technique?1M
✓ Answer

In networking, encapsulation is the process of wrapping data with protocol-specific headers (and sometimes trailers) at each layer of the network model as it travels down the stack before transmission.

Application  → Data
Transport    → [TCP/UDP Header] + Data         = Segment
Network      → [IP Header] + Segment           = Packet
Data Link    → [Frame Header] + Packet + [FCS] = Frame
Physical     → Bits (electrical/optical signal)

At the receiver, de-encapsulation strips headers layer by layer going up the stack.

Q1(IX)What is the use of CORS?1M
✓ Answer

CORS (Cross-Origin Resource Sharing) is a browser security mechanism that controls how web pages can make HTTP requests to a different domain than the one that served them (cross-origin requests).

Browsers enforce the Same-Origin Policy by default (blocks cross-origin AJAX). CORS allows servers to explicitly permit cross-origin access using HTTP headers:

Access-Control-Allow-Origin: https://myapp.com
Access-Control-Allow-Methods: GET, POST
Access-Control-Allow-Headers: Content-Type

Used when a frontend (React on port 3000) calls a backend API on a different domain/port.

Q1(XII)What is the use of grouping in CSS3?1M
✓ Answer

Grouping selectors in CSS allows multiple selectors to share the same style rules, reducing code repetition:

/* Without grouping — repetitive */
h1 { color: blue; font-family: Arial; }
h2 { color: blue; font-family: Arial; }
p  { color: blue; font-family: Arial; }

/* With grouping — clean and DRY */

Reduces file size, improves maintainability, and makes stylesheets easier to read.

Q3How to create a circular progress bar using SVG?5 Marks
✓ Answer

A circular progress bar uses SVG <circle> with stroke-dasharray and stroke-dashoffset to control what percentage of the circle is drawn.

<!DOCTYPE html>
<html>
<head>
<style>
  .progress-ring { transform: rotate(-90deg); }
  .progress-ring__bg {
    fill: none; stroke: #e8e8f0; stroke-width: 10;
  }
  .progress-ring__circle {
    fill: none; stroke: #6c63ff; stroke-width: 10;
    stroke-linecap: round;
    stroke-dasharray: 251.2;   /* 2 * π * r = 2 * 3.14 * 40 */
    stroke-dashoffset: 62.8;   /* offset = dasharray * (1 - progress/100) */
    transition: stroke-dashoffset 0.5s ease;
  }
  .progress-text {
    font-size: 1.4rem; font-weight: 700; fill: #6c63ff;
  }
</style>
</head>
<body>
<svg width="120" height="120" viewBox="0 0 120 120">
  <circle class="progress-ring__bg" cx="60" cy="60" r="40"/>
  <circle class="progress-ring__circle" id="ring" cx="60" cy="60" r="40"/>
  <text x="60" y="65" text-anchor="middle" class="progress-text"
        transform="rotate(90,60,60)">75%</text>
</svg>

<script>
  function setProgress(percent) {
    const r = 40;
    const circumference = 2 * Math.PI * r;   // 251.2
    const offset = circumference * (1 - percent / 100);
    document.getElementById("ring").style.strokeDashoffset = offset;
  }
  setProgress(75);  // show 75%
</script>
</body>
</html>

Key formula: strokeDashoffset = circumference × (1 − progress/100). At 0% offset the full circle shows; increasing offset hides more of it.

Q6How to create Image Stack Illusion using HTML and CSS?5 Marks
✓ Answer

An image stack illusion uses CSS transform and box-shadow to make a single image look like a stack of photos.

<!DOCTYPE html>
<html>
<head>
<style>
  body { display: flex; justify-content: center;
         align-items: center; height: 100vh; background: #f0f0f0; }

  .stack { position: relative; width: 220px; height: 160px; }

  /* Bottom layers — decorative rotated copies via pseudo-elements */
  .stack::before,
  .stack::after {
    content: "";
    position: absolute;
    width: 220px; height: 160px;
    background: white;
    border: 1px solid #ddd;
    border-radius: 4px;
    box-shadow: 0 2px 8px rgba(0,0,0,0.15);
  }
  .stack::before {
    transform: rotate(-5deg);
    top: 6px; left: -6px;
    z-index: 1;
  }
  .stack::after {
    transform: rotate(3deg);
    top: 3px; left: 3px;
    z-index: 2;
  }

  /* Top (actual) image */
  .stack img {
    position: relative;
    width: 220px; height: 160px;
    object-fit: cover;
    border-radius: 4px;
    border: 1px solid #ccc;
    box-shadow: 0 4px 16px rgba(0,0,0,0.2);
    z-index: 3;
  }

  /* Hover effect — fan out */
  .stack:hover::before { transform: rotate(-10deg) translateX(-10px); }
  .stack:hover::after  { transform: rotate(6deg)  translateX(6px); }
  .stack::before, .stack::after,
  .stack img { transition: transform 0.3s ease; }
</style>
</head>
<body>
  <div class="stack">
    <img src="photo.jpg" alt="Stacked photo">
  </div>
</body>
</html>

Key techniques: relative/absolute positioning, CSS pseudo-elements (::before, ::after), rotate transform, z-index layering, and transition for smooth hover animation.

Q7Write about Arrays in JavaScript. [15]15 Marks
✓ Answer

1. Creating Arrays

const arr1 = [1, 2, 3, 4, 5];              // Array literal (preferred)
const arr2 = new Array(5);                  // Array of 5 empty slots
const arr3 = new Array(10, 20, 30);         // [10, 20, 30]
const mixed = ["Alice", 25, true, null, {role:"dev"}]; // mixed types

2. Accessing and Modifying

console.log(arr1[0]);        // 1  (zero-indexed)
console.log(arr1.length);    // 5
arr1[0] = 99;                // modify element
console.log(arr1);           // [99, 2, 3, 4, 5]

3. Adding and Removing Elements

arr1.push(6);          // add to END        → [99,2,3,4,5,6]
arr1.pop();            // remove from END   → [99,2,3,4,5]
arr1.unshift(0);       // add to FRONT      → [0,99,2,3,4,5]
arr1.shift();          // remove from FRONT → [99,2,3,4,5]
arr1.splice(1, 2);     // remove 2 at index 1 → [99,4,5]
arr1.splice(1, 0, 10, 20); // insert at index 1 → [99,10,20,4,5]

4. Searching and Checking

const nums = [10, 20, 30, 20, 40];
nums.indexOf(20);           // 1 (first occurrence)
nums.lastIndexOf(20);       // 3
nums.includes(30);          // true
nums.find(x => x > 25);    // 30 (first match)
nums.findIndex(x => x > 25); // 2

5. Iteration Methods

const data = [1, 2, 3, 4, 5];

// forEach — execute for each element
data.forEach((val, i) => console.log(i, val));

// map — transform each element, returns new array
const doubled = data.map(x => x * 2);    // [2,4,6,8,10]

// filter — keep elements matching condition
const evens = data.filter(x => x % 2 === 0); // [2,4]

// reduce — accumulate to single value
const sum = data.reduce((acc, x) => acc + x, 0); // 15

// every / some
data.every(x => x > 0);    // true (all positive)
data.some(x => x > 4);     // true (at least one > 4)

6. Sorting and Reversing

const letters = ["banana", "apple", "cherry"];
letters.sort();                      // ["apple","banana","cherry"]
letters.reverse();                   // ["cherry","banana","apple"]

const nums2 = [30, 5, 100, 2];
nums2.sort((a, b) => a - b);        // [2, 5, 30, 100] ascending
nums2.sort((a, b) => b - a);        // [100, 30, 5, 2] descending

7. Copying, Merging, Slicing

const a = [1, 2, 3];
const b = [4, 5, 6];

const merged = a.concat(b);          // [1,2,3,4,5,6]
const copy   = [...a];               // spread — shallow copy
const slice  = a.slice(1, 3);        // [2,3] (index 1 to 2)
const flat   = [[1,2],[3,4]].flat(); // [1,2,3,4]

8. Destructuring and Spread

const [first, second, ...rest] = [10, 20, 30, 40, 50];
// first=10, second=20, rest=[30,40,50]

// Swap variables without temp
let x = 1, y = 2;
[x, y] = [y, x];   // x=2, y=1

9. Multidimensional Arrays

const matrix = [[1,2,3],[4,5,6],[7,8,9]];
console.log(matrix[1][2]);   // 6
matrix.forEach(row => console.log(row.join(" ")));

Q10Write about Functions in JavaScript. [15]15 Marks
✓ Answer

1. Function Declaration (Hoisted)

function greet(name) {
    return "Hello, " + name + "!";
}
console.log(greet("Alice")); // Hello, Alice!
// Can be called BEFORE declaration (hoisted)

2. Function Expression (Not hoisted)

const add = function(a, b) { return a + b; };
console.log(add(3, 4)); // 7

3. Arrow Functions (ES6)

// Single expression — implicit return
const square = x => x * x;
console.log(square(5)); // 25

// Multiple params
const multiply = (a, b) => a * b;

// Block body
const factorial = n => {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
};
console.log(factorial(5)); // 120

4. Default Parameters

function power(base, exp = 2) { return Math.pow(base, exp); }
console.log(power(3));    // 9  (exp defaults to 2)
console.log(power(2, 10)); // 1024

5. Rest Parameters & Spread

function sum(...nums) {
    return nums.reduce((acc, n) => acc + n, 0);
}
console.log(sum(1, 2, 3, 4, 5)); // 15

const nums = [1, 2, 3];
console.log(Math.max(...nums));   // 3 (spread)

6. Closures

function makeCounter() {
    let count = 0;
    return {
        increment: () => ++count,
        decrement: () => --count,
        value:     () => count
    };
}
const counter = makeCounter();
counter.increment(); counter.increment();
console.log(counter.value()); // 2

7. Higher-Order Functions

// Function that takes a function as argument
function applyTwice(fn, x) { return fn(fn(x)); }
console.log(applyTwice(x => x * 2, 3)); // 12

// Function that returns a function
function multiplier(factor) {
    return num => num * factor;
}
const triple = multiplier(3);
console.log(triple(7)); // 21

8. IIFE (Immediately Invoked Function Expression)

(function() {
    const secret = "I run immediately!";
    console.log(secret);
})();
// secret is NOT accessible outside

9. Async Functions (ES8)

    try {
        const res = await fetch(`/api/users/${id}`);
        const data = await res.json();
        return data;

        console.error("Error:", err);
    }

function* range(start, end) {
    for (let i = start; i <= end; i++) yield i;

Q11Which Protocols Describe VoIP Connections? Explain in detail. [15]15 Marks
✓ Answer

1. SIP — Session Initiation Protocol (5 marks)

The most widely used VoIP signalling protocol. Handles call setup, modification, and teardown.

Caller                  SIP Proxy               Callee
  │── INVITE ─────────────────────────────────►│
  │◄── 100 Trying ──────────────────────────────│
  │◄── 180 Ringing ─────────────────────────────│
  │◄── 200 OK (SDP media params) ───────────────│
  │── ACK ──────────────────────────────────────►│

  │◄── 200 OK ──────────────────────────────────│

SIP messages: INVITE, ACK, BYE, CANCEL, REGISTER, OPTIONS. SDP (Session Description Protocol) negotiates codec, IP, port.

2. RTP — Real-time Transport Protocol

Carries the actual voice/video data over UDP. Provides: sequence numbers (ordering), timestamps (sync/jitter), SSRC (stream ID), payload type (codec). Works with RTCP for QoS monitoring.

3. RTCP — RTP Control Protocol

Companion to RTP. Sends periodic control packets: SR (Sender Report) with stats, RR (Receiver Report) with loss/jitter. Allows both sides to monitor call quality and adapt (e.g., switch codec).

4. H.323 — ITU-T Standard

Older but still used enterprise VoIP standard. Uses: H.225 (call signalling), H.245 (media control), RAS (Registration, Admission, Status). More complex than SIP but more mature for enterprise PBX systems.

5. MGCP — Media Gateway Control Protocol

Controls media gateways (devices connecting PSTN to IP) from a centralised call agent. The gateway is "dumb" — the call agent (softswitch) does all the intelligence. Used by telephone carriers.

Complete VoIP Call Flow

1. REGISTER:  SIP phone registers with SIP server
2. INVITE:    Caller sends SIP INVITE to SIP server
3. ROUTE:     SIP proxy routes to callee's SIP address
4. NEGOTIATE: SDP in INVITE/200 OK negotiates codec & ports
5. ACK:       3-way handshake completes — call established
6. MEDIA:     RTP packets flow UDP between caller & callee
7. QOS:       RTCP monitors packet loss, jitter, delay
8. BYE:       Either party sends SIP BYE to end call
