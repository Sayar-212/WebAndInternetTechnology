a) Web server
b) Application server
c) Web browser
d) Database server
✓ Answer: (a) Web server

Cookies are created by the web server and sent to the browser via the Set-Cookie HTTP response header. The browser then stores the cookie and sends it back with every subsequent request to the same domain using the Cookie request header.

SSL/TLS sits between the Application layer and the Transport layer in the TCP/IP stack. It intercepts data from apps (HTTP, FTP) and encrypts it before passing it down to TCP.

Q1(viii) In ________ the IP header of the original packet is also encrypted. 1 Mark
a) only tunnel mode
b) only transport mode
c) both tunnel mode and transport mode
d) none of these
✓ Answer: (a) Only Tunnel Mode

In IPSec Tunnel Mode, the entire original IP packet (header + payload) is encrypted and a new outer IP header is added. In Transport Mode, only the payload is encrypted — the original IP header remains in clear text.

NAT maps private IP addresses (e.g., 192.168.x.x) to a public IP address for communication with the internet. It conserves IPv4 address space and provides a layer of security by hiding the internal network topology.

Q2 a) What is associative array in PERL?   b) Write a program to find length of a string in PERL. 2+3 = 5
✓ Answer

a) Associative Array in Perl (2 marks)

An associative array (called a Hash in Perl) is a collection of key-value pairs. Unlike regular arrays that use numeric indices, hashes use string keys. They are declared using the % sigil.

# Declaring a hash
my %student = (
    'name' => 'Alice',
    'roll' => 101,
    'dept' => 'CSE'
);

# Accessing a value
print $student{'name'};   # Output: Alice

# Adding a new key-value pair
$student{'year'} = 3;

# Iterating over hash
while (my ($key, $val) = each %student) {
    print "$key => $val\n";
}

Q3 a) Purpose of TTL field in IPv4   b) How salt makes dictionary attack difficult   c) Components of VPN 2+2+1 = 5
✓ Answer

a) Time to Live (TTL) field in IPv4 (2 marks)

The TTL field is an 8-bit field in the IPv4 header. It specifies the maximum number of hops (routers) a packet can traverse before being discarded.

  • Each router that forwards the packet decrements TTL by 1.
  • When TTL reaches 0, the router discards the packet and sends an ICMP Time Exceeded message back to the sender.
  • Purpose: Prevents packets from looping infinitely in the network due to routing errors.
  • Common default values: Windows = 128, Linux = 64.

b) How Salt makes Dictionary Attack difficult (2 marks)

A salt is a random value appended to a password before hashing it.

  • Without salt: hash("password") = X — attacker precomputes a rainbow table of common passwords.
  • With salt: hash("password" + "r4nD0m$alt") = Y — even if two users have the same password, their hashes differ.
  • The attacker must recompute the hash for every possible password + every unique salt — making precomputed rainbow tables useless.

c) Components of VPN (1 mark)

  • VPN Client software
  • VPN Server / Gateway
  • Tunneling Protocol (IPSec / SSL / PPTP / L2TP)
  • Authentication Server
  • Encryption / Decryption module
Q4 a) Advantages of JavaScript over HTML   b) What is DOM? Explain briefly. 2+3 = 5
✓ Answer

a) Advantages of JavaScript over HTML (2 marks)

  • Dynamic behavior: JS can change page content, styles, and structure without reloading.
  • Client-side validation: Validates form inputs before sending to server, saving bandwidth.
  • Event handling: Responds to user actions (clicks, keypresses, hover) in real-time.
  • DOM Manipulation: Can create, modify, or delete HTML elements dynamically.
  • HTML is static markup; JS adds logic and interactivity.

b) DOM — Document Object Model (3 marks)

The DOM is a programming interface (API) provided by the browser that represents an HTML or XML document as a tree of nodes. Each element, attribute, and text is a node in this tree.

HTML Document:
<html>
  <body>
    <h1 id="title">Hello</h1>
    <p>World</p>
  </body>
</html>

DOM Tree:
Document
└── html
    └── body
        ├── h1 (#title) → "Hello"
        └── p → "World"

JavaScript uses the DOM to dynamically access and modify the document:

// Change text content
document.getElementById("title").textContent = "Hi there!";

// Create a new element
let p = document.createElement("p");
p.textContent = "New paragraph";
document.body.appendChild(p);
Q5 Design a firewall to block all traffic from 10.0.0.0/24 except port 23 and 80 of TCP. 5 Marks
✓ Answer

A packet-filtering firewall uses rules processed top to bottom. We allow exceptions first, then deny everything else from that subnet.

Firewall Rule Table (ACL)

Rule#Source IPProtocolDest PortActionReason
110.0.0.0/24TCP80ALLOWAllow HTTP
210.0.0.0/24TCP23ALLOWAllow Telnet
310.0.0.0/24ANYANYDENYBlock all else from subnet
4ANYANYANYALLOWPermit rest of world

iptables Implementation (Linux)

# Allow TCP port 80 (HTTP) from 10.0.0.0/24
iptables -A INPUT -s 10.0.0.0/24 -p tcp --dport 80 -j ACCEPT

# Allow TCP port 23 (Telnet) from 10.0.0.0/24
iptables -A INPUT -s 10.0.0.0/24 -p tcp --dport 23 -j ACCEPT

# Block ALL other traffic from 10.0.0.0/24
iptables -A INPUT -s 10.0.0.0/24 -j DROP

# Allow everything else
iptables -A INPUT -j ACCEPT

Key principle: Rules are evaluated in order — specific ALLOW rules come before the blanket DENY rule.

  • Version (4b): Always 6 for IPv6
  • Traffic Class (8b): QoS / priority (like TOS in IPv4)
  • Flow Label (20b): Labels packets of same flow for QoS treatment
  • Payload Length (16b): Length of data after the header
  • Next Header (8b): Identifies the type of header that follows
  • Hop Limit (8b): Replaces TTL — decremented at each hop
  • Source / Destination Address: 128-bit each

Next Header field purpose: It works like the Protocol field in IPv4. It identifies what comes after the IPv6 header — could be TCP (6), UDP (17), ICMPv6 (58), or an IPv6 Extension Header.

b) Java RMI (3 marks)

RMI (Remote Method Invocation) is a Java API that allows a program running in one JVM to invoke methods of an object running in another JVM — even on a different machine over the network.

Why RMI is useful:

  • Enables distributed computing — split application logic across multiple machines.
  • Uses Java's type safety and object-oriented principles across network boundaries.
  • Handles marshalling/unmarshalling (serialization) of arguments automatically.
  • Foundation for enterprise applications (EJBs, distributed services).

d) Multicast Addressing + Transport Gateway (3 marks)

Multicast Addressing: A single packet is sent to a group of interested receivers (one-to-many). Uses Class D addresses (224.0.0.0 – 239.255.255.255). Routers replicate packets only on paths to group members. More efficient than broadcast (doesn't bother non-members) and unicast (one copy per receiver).

Transport Gateway: A device that connects two networks using different transport protocols (e.g., TCP vs. ISO TP4). It receives a segment from one network, terminates the transport connection, re-encapsulates the data in the format of the destination network, and forwards it. Acts as a translator at the Transport Layer (Layer 4).

XML (eXtensible Markup Language) is a markup language designed to store and transport data. It is platform-independent, human-readable, and self-descriptive. Unlike HTML, tags in XML are user-defined.

<?xml version="1.0" encoding="UTF-8"?>   <!-- XML Declaration -->
<root>                                     <!-- Root element (mandatory) -->
  <element attribute="value">             <!-- Element with attribute -->

  <empty-element />                        <!-- Self-closing element -->

Rules: One root element, all tags closed, case-sensitive, attributes quoted, proper nesting required.

Q11 Short Notes on any THREE: a) Java Socket   b) Packet Filtering   c) Image (HTML)   d) Java Applet   e) Web Crawler 3 × 5 = 15
✓ Answer

a) Java Socket (5 marks)

A socket is one endpoint of a two-way communication link between two programs over a network. Java provides java.net.Socket (client) and java.net.ServerSocket (server).

// SERVER SIDE
import java.net.*; import java.io.*;
public class Server {
    public static void main(String[] args) throws Exception {
        ServerSocket ss = new ServerSocket(5000); // listen on port 5000
        System.out.println("Waiting for client...");
        Socket s = ss.accept();                   // blocks until client connects
        BufferedReader in = new BufferedReader(
            new InputStreamReader(s.getInputStream()));
        PrintWriter out = new PrintWriter(s.getOutputStream(), true);
        System.out.println("Client says: " + in.readLine());
        out.println("Hello from Server!");
        s.close(); ss.close();
    }
}

// CLIENT SIDE
public class Client {
    public static void main(String[] args) throws Exception {
        Socket s = new Socket("localhost", 5000);  // connect to server
        PrintWriter out = new PrintWriter(s.getOutputStream(), true);
        BufferedReader in = new BufferedReader(
            new InputStreamReader(s.getInputStream()));
        out.println("Hello from Client!");
        System.out.println("Server says: " + in.readLine());
        s.close();
    }
}

b) Packet Filtering (5 marks)

Packet filtering is the most basic form of firewall operation. It examines each packet at the network layer (Layer 3) and decides whether to ALLOW or DENY it based on:

  • Source IP address
  • Destination IP address
  • Protocol (TCP, UDP, ICMP)
  • Source / Destination port number
  • Packet direction (inbound / outbound)

Advantages: Fast, low overhead, transparent to users.
Disadvantages: Cannot inspect payload content, vulnerable to IP spoofing, no state tracking (stateless).

d) Java Applet (5 marks)

A Java Applet is a small Java program embedded within an HTML web page and executed by the browser's JVM plugin. It runs inside a sandbox (restricted environment).

import java.applet.*;
import java.awt.*;
public class HelloApplet extends Applet {
    public void paint(Graphics g) {
        g.setColor(Color.BLUE);
        g.setFont(new Font("Arial", Font.BOLD, 24));
        g.drawString("Hello from Java Applet!", 50, 100);
    }
}
// HTML to embed:
// <applet code="HelloApplet.class" width="400" height="200"></applet>

Lifecycle: init() → start() → paint() → stop() → destroy()
Restrictions: Cannot access local file system, restricted network access (same origin), cannot run system commands.

e) Web Crawler (5 marks)

A Web Crawler (also called a spider or bot) is an automated program that systematically browses the World Wide Web to index web pages for search engines.

Working Process:

  • Starts with a list of seed URLs.
  • Downloads each page's HTML content.
  • Extracts all hyperlinks from the page.
  • Adds unvisited links to a queue.
  • Processes each URL in the queue (BFS or DFS).
  • Stores/indexes page content for the search engine.
  • Respects robots.txt — pages that don't want to be indexed.

Examples: Googlebot, Bingbot, Slurp (Yahoo).
Challenges: Duplicate detection, handling dynamic pages (JavaScript-rendered), rate limiting to avoid overloading servers.
